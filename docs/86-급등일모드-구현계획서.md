## 1. 개요

KOSPI 급등일(+3%↑)에 MTF threshold와 ML buy threshold를 동적으로 낮춰
시스템이 급등 전환을 포착할 수 있도록 함.

### 현재 병목

```
8개 후보 → MTF(5개 차단, 0.67) → 3개 → ML(모두 60% 미달) → 0개 매수
```

### 목표

```
8개 후보 → MTF(급등일 0.34, 1~2개 차단) → 6~7개 → ML(급등일 50%, 일부 통과) → 1~2개 매수
```

## 2. 수정 방안

### 2.1 config 필드 추가

`core/config.py` `StrategyConfig`:

```python
# ── 급등일 모드 (P-86) ──
surge_day_min_correction: float = 15.0    # 장중보정 점수 ≥ 이 값 → 급등일 모드
surge_day_mtf_threshold: float = 0.34     # 급등일 MTF 임계값 (3TF 중 1개)
surge_day_buy_threshold: float = 0.50     # 급등일 ML 매수 임계값 (정상 0.60)
surge_day_position_multiplier: float = 1.5  # 급등일 포지션 크기 배수
```

### 2.2 entry_planner.py 수정

`plan_entries()`에서 장중보정 점수로 급등일 감지 → MTF/ML 임계값 override:

```python
# 장중보정 점수로 급등일 감지
intraday_correction = abs(regime_result.evidence.get("intraday_correction", 0))
is_surge_day = intraday_correction >= self._strategy.surge_day_min_correction
```

**MTF 필터** — 급등일이면 threshold 0.34로 override:

```python
mtf_threshold = None
if is_surge_day:
    mtf_threshold = self._strategy.surge_day_mtf_threshold
```

**ML 예측** — 급등일이면 buy threshold 50%로 완화:

```python
effective_buy_threshold = (
    self._strategy.surge_day_buy_threshold
    if is_surge_day
    else self._strategy.buy_threshold
)
```

## 3. 리스크 평가

| 항목 | 리스크 | 내용 |
|---|---|---|
| **MTF 0.34** | 🟡 **중간** | 분봉만 상승(0.33)도 통과 → 무분별한 진입 가능 |
| **ML 50%** | 🟡 **중간** | 신뢰도 낮은 예측(50~59%)으로 매수 실행 |
| **position 1.5x** | 🟡 **중간** | 손실 시 평소보다 1.5배 손실 |
| **방어막** | 🟢 **안전** | min_hold 20분 동안 stop_loss는 정상 작동. 급등일 반대 급락에도 20분 내 손절 가능 |
| **false positive** | 🟡 KOSPI가 +3% 올랐다가 -2%로 마감 → 급등일 모드 진입 후 손실 | min_hold+stop_loss가 1차 방어, 일일 loss limit이 2차 방어 |

**총평: 🟡 Medium — 리스크가 있지만 방어막(min_hold+stop_loss+daily_loss_limit)이 3중으로 하방 보호.**

## 4. 테스트 방법

1. syntax check 3개 파일
2. surge_day_min_correction을 0.0으로 설정 → 항상 급등일 모드 → MTF 통과율 확인
3. 실제 KOSPI 등락률로 급등일 감지 확인
4. 복원 후 서버 재시작
