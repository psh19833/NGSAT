## P0 — main.py model.update_minute_model() AttributeError

### 증상
분봉ML 자동 재학습 시 `main.py:455`에서 `model.update_minute_model(minute_model)` 호출.  
`model`은 `PriceRiseModel` 객체이나 `update_minute_model`은 `MLInference`에만 존재.  
→ `AttributeError: 'PriceRiseModel' object has no attribute 'update_minute_model'`  
→ 분봉ML 재학습이 항상 실패하고 `except` 블록에서 조용히 삼켜짐

### 원인
- `model` 변수 출처: `train_from_price_data()` → `PriceRiseModel` (`ml/training/trainer.py:101`)
- 분봉ML 모델 교체 대상: `orchestrator._inference` → `MLInference` (`ml/inference.py:79`)
- `update_minute_model()`: `MLInference`의 메서드 (`ml/inference.py:298`)

### 수정方案
```python
# AS-IS (main.py:455)
model.update_minute_model(minute_model)

# TO-BE
orchestrator._inference.update_minute_model(minute_model)
```

### 리스크 평가
| 항목 | 내용 |
|---|---|
| **🟡 변경 영향 범위** | 분봉ML 자동 재학습 코드만 영향. 사이클 전체에는 영향 없음 |
| **🔴 실패 시** | save_model()도 같이 실행되므로, 구 모델 유지 + 새 모델 파일만 저장 실패 |
| **🟢 복원 방법** | `git checkout -- main.py`로 즉시 복원 가능 |
| **🔴 런타임 위험** | 장 마감 후(15:30 이후)에만 실행되므로 장중 영향 없음 |
| **총평** | 1줄 변경. `orchestrator._inference`는 `MLInference`로 초기화 검증 완료. **안전** |

### 검증 방법
```python
# orchestrator._inference 타입 확인
assert hasattr(orchestrator, '_inference')
assert hasattr(orchestrator._inference, 'update_minute_model')
```

`bull_t`/`bear_t`만 별도로 `load_config()` 조회 — **일관성 없음**

### 수정方案

```python
# AS-IS (entry_planner.py:147-151)
from core.config import load_config

bear_t = _cfg.strategy.regime_bear_threshold

# TO-BE
bull_t = getattr(self._strategy, 'regime_bull_threshold', 65)

```

| **총평** | **안전**. 단, `self._strategy` 값이 사이클 간에 갱신되는지 확인 필요 |

## P2 — repository.py get_daily_pnl() 타입힌트 불일치

### 증상
```python
def get_daily_pnl(self) -> list[dict]:   # 타입힌트
    ...
    return result, equity_curve           # 실제: tuple
```
호출자(`api.py`)에서 `result, equity_curve = repo.get_daily_pnl()`로 언패킹하므로 **기능 정상**.  
타입힌트만 불일치 → IDE 경고 + 정적 분석 오탐지 원인.

### 수정方案 (2가지 옵션)

**Option A (추천) — 타입힌트만 수정**
```python
def get_daily_pnl(self) -> tuple[list[dict], list[dict]]:
```
최소 변경, 호출자 영향 없음.

**Option B — 메서드 분리**
```python
def get_daily_pnl(self) -> list[dict]: ...
def get_equity_curve(self) -> list[dict]: ...
```
SRP 관점 우수하나, `equity_curve`가 `get_daily_pnl` 내부에서 계산되므로 중복 계산 발생.  
리팩토링 규모 대비 효과가 작아 Option A 추천.

### 리스크 평가
| 항목 | 내용 |
|---|---|
| **🟢 변경 영향** | 타입힌트만 변경. 런타임 동작 0% 변화 |
| **🟢 복원** | 1줄 변경 |
| **총평** | **리스크 없음** |
