# ML 모델 자동 선택 기능 기획서

> **문서:** docs/17-ML-모델-자동-선택-기획서.md
> **작성일:** 2026-06-29
> **상태:** 기획 (리뷰 필요)

## 1. 문제 정의

현재 사용자가 직접 AI 모델 종류(Gradient Boosting / XGBoost / LightGBM 등)를 선택해야 한다.
비개발자 입장에서 어떤 모델이 지금 시장에 가장 적합한지 알기 어렵다.

**목표:** 매 재학습 사이클마다 5개 모델을 모두 학습시켜 가장 AUC가 높은 모델로 **자동 교체**한다.

## 2. 현재 구조

### auto_retrain() 흐름

```
auto_retrain()
  │
  ├─ 데이터 준비 (build_training_dataset)
  │
  ├─ [20회마다] Optuna 튜닝 (같은 모델 종류 유지)
  │
  ├─ 1개 모델만 학습 (현재 설정된 model_type)
  │
  └─ AUC 비교 → 기존보다 좋으면 교체, 나쁘면 유지
```

### 지원 모델 (5종)

| 모델 | 패키지 | 현재 상태 |
|------|--------|:---------:|
| Logistic Regression | sklearn | 내장 |
| Random Forest | sklearn | 내장 |
| Gradient Boosting | sklearn | 내장 |
| XGBoost | xgboost | 옵션 (미설치 시 fallback) |
| LightGBM | lightgbm | 옵션 (미설치 시 fallback) |

## 3. 변경 대상

### 3.1 trainer.py — auto_retrain() → auto_select_retrain()

**변경 전:**
```python
new_model = PriceRiseModel(self.model_type, ...)
new_result = new_model.train(X, y)
# AUC 비교 → 교체 or 유지
```

**변경 후:**
```python
best_model_type = self.model_type   # 기본값
best_auc = getattr(self, '_last_auc', 0)

for model_type in ALL_MODELS:        # 5개 전부 시도
    if not _is_available(model_type):
        continue
    m = PriceRiseModel(model_type, ...)
    r = m.train(X, y)
    if r.success and r.auc > best_auc:
        best_auc = r.auc
        best_model = m
        best_model_type = model_type

# 최종: best_model로 교체
self._model = best_model._model
self.model_type = best_model_type    # ← 자동 전환!
```

### 3.2 core/config.py
- `ml_auto_select_model: bool = False` (기본 OFF, 사용자 선택)
- True일 때만 5개 모델 전부 학습
- False면 기존처럼 단일 모델 학습

### 3.3 dashboard/backend/api.py
- CONFIG_FIELD_MAP: `NGSAT_ML_AUTO_SELECT_MODEL`
- StrategyUpdateRequest: `ml_auto_select_model`

### 3.4 dashboard/StrategyConfigPanel.jsx
- "⑧ ML 학습 기간" 섹션에 토글 추가
- `자동 모델 선택 [●══○]` — 켜면 5개 중 최적 모델 자동 선정
- 현재 선택된 모델 종류 표시 (예: `현재 모델: XGBoost (AUC 0.684)`)

### 3.5 dashboard/api.py — config 응답
- GET /api/strategy/config 응답에 `current_model_type` + `current_auc` 포함
- 대시보드 상단에 현재 모델 정보 표시

## 4. 리스크 분석

| # | 리스크 | 등급 | 설명 | 대응 |
|---|--------|:----:|------|------|
| R1 | **학습 시간 5배 증가** | 🟡 MED | 5개 모델 순차 학습, auto_retrain이 1초→5초로 증가 | 병렬 학습(concurrent.futures) 또는 학습 시간 제한(timeout=30s) |
| R2 | **XGBoost/LightGBM 미설치** | 🟢 LOW | 패키지 없으면 fallback, sklearn 3종만 비교 | try/import로 graceful fallback |
| R3 | **모델 종류 잦은 변경** | 🟡 MED | 매 사이클마다 다른 모델로 바뀌면 예측 일관성 저하 | "최근 N회 중 최다 선택" 방식 도입 (예: 최근 5회 중 3회 이상 1등) |
| R4 | **AUC 편향** | 🟢 LOW | 일부 모델이 항상 높은 AUC를 보일 수 있음 | 다양한 시장 레짐(강세/약세/중립)에서 평가 |
| R5 | **메모리 사용량** | 🟢 LOW | 5개 모델 동시 로드 (각 ~100MB) → ~500MB | 순차 학습 후 1개만 메모리 유지 |
| R6 | **사용자 혼란** | 🟢 LOW | "무슨 모델 쓰고 있는지 모르겠어요" | 대시보드 상단에 현재 모델명+성능 표시 |

### 종합 리스크 등급: 🟢 LOW-MEDIUM

> R1(학습 시간)과 R3(모델 변경 안정성)만 주의하면 무난히 진행 가능.

## 6. 예상 동작

```
자동 모델 선택 OFF (기존 동작):
  Gradient Boosting → 재학습 → GB 유지

자동 모델 선택 ON:
  Cycle 1: RF(AUC 0.62) | GB(AUC 0.68) | XGB(AUC 0.71) ← XGB 선정
  Cycle 2: RF(0.65) | GB(0.70) | XGB(0.69) ← GB 선정
  Cycle 3: RF(0.63) | GB(0.67) | XGB(0.72) ← XGB 선정
  → 시장 상황에 따라 최적 모델이 자동 변경됨

대시보드 표시:
  ┌───────────────────────────────────────┐
  │ 현재 모델: XGBoost  |  AUC: 0.712    │
  │ 최근 3회: [XGB] [GB] [XGB]           │
  └───────────────────────────────────────┘
```

## 7. 종속성

- XGBoost/LightGBM 미설치 시 sklearn 3종으로만 비교 (자동 fallback)
- 학습 데이터 50개 미만이면 auto-select 생략 (단일 모델만 학습)
- auto_retrain OFF면 auto-select도 동작 안 함

---

*참고: 현재 학습 데이터는 최근 250일(기본) 또는 사용자 지정 구간의 KIS 실데이터. 5개 모델 동일 데이터로 학습.*
