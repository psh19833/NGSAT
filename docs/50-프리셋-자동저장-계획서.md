# 프리셋 자동 전환 영속화 — 수정 계획서

> **문제**: PresetRouter가 자동으로 프리셋을 변경해도 서버 재시작 시 기본값으로 복원됨
> **파일**: `docs/50-프리셋-자동저장-계획서.md`

---

## 분석

### 현재 흐름

```
PresetRouter.select_preset()
  → orchestrator._preset_router.current_preset 업데이트 (메모리 only)
  → 텔레그램 알림 발송
  → ❌ 디스크/DB 저장 없음
  → 서버 재시작 → .env 기본값 또는 ConfigService DB 저장값으로 복원
```

### ConfigService 구조

ConfigService는 DB(`ngsat.db`)에 환경변수를 저장/조회 가능. 대시보드 전략설정에서 변경한 값은 ConfigService에 저장됨.

| 저장소 | 읽기 | 쓰기 | 자동저장 |
|:------|:----:|:----:|:--------:|
| `.env` 파일 | ✅ 초기 로드 | ❌ 수동 | ❌ |
| `ConfigService` DB | ✅ 읽기 | ✅ 대시보드 | ❌ (API 호출 시에만) |
| `StrategyConfig` 메모리 | ✅ 사용 | ✅ 런타임 변경 | ❌ |

PresetRouter가 프리셋을 변경할 때 ConfigService DB에도 저장하여, 서버 재시작 시 복원.

**수정 항목**:

| # | 항목 | 파일 | 작업량 | 리스크 |
|:-:|------|------|:------:|:-----:|
| 1 | `ConfigService.save_preset()` 메서드 추가 | `core/config.py` | ~10줄 | 🟢 ZERO |
| 2 | PresetRouter 전환 시 ConfigService 호출 | `live/orchestrator.py` | ~5줄 | 🟢 LOW |
| 3 | 서버 시작 시 ConfigService에서 프리셋 복원 | `main.py` | ~5줄 | 🟢 LOW |
| 4 | 대시보드 전략설정 저장 시 ConfigService 동기화 | `dashboard/backend/api.py` | ~3줄 | 🟢 ZERO |

**동작 방식**:
```
PresetRouter 전환 →
  ① orchestrator._preset_router.current_preset = "단타형" (메모리)
  ② ConfigService.save("preset_name", "단타형") (DB 저장)
  ...
서버 재시작 →
  ① ConfigService.load("preset_name") → "단타형"
  ② PresetRouter.set_preset("단타형")
  ③ StrategyConfig에 프리셋 값 적용
```

## 권장: A안 (ConfigService)

### 상세 구현

**1. ConfigService.save_config() 메서드 추가** (`core/config.py`)

```python
class ConfigService:
    def save_config(self, key: str, value: str) -> None:
        """설정값을 DB에 저장 (upsert)."""
        with self._session() as session:
            existing = session.query(ConfigEntry).filter(
                ConfigEntry.key == key
            ).first()
            if existing:
                existing.value = value
            else:
                session.add(ConfigEntry(key=key, value=value))
            session.commit()
    
    def load_config(self, key: str, default: str = "") -> str:
        """DB에서 설정값 조회."""
        with self._session() as session:
            entry = session.query(ConfigEntry).filter(
                ConfigEntry.key == key
            ).first()
            return entry.value if entry else default
```

**2. PresetRouter 전환 시 저장** (`live/orchestrator.py`)

```python
# Step 4b: Auto preset selection
new_preset = self._preset_router.select_preset(regime_result, atr_pct=vol)
if new_preset:
    logger.info(f"프리셋 자동 전환: {self._preset_router.current_preset} → {new_preset}")
    result.preset_change = new_preset
    # ConfigService에 저장 (재시작 복원용)
    try:
        from core.config import ConfigService
        cs = ConfigService()
        cs.save_config("preset_name", new_preset)
    except Exception:
        pass
```

**3. 서버 시작 시 복원** (`main.py`)

```python
# ConfigService에서 마지막 프리셋 복원
try:
    from core.config import ConfigService
    cs = ConfigService()
    saved_preset = cs.load_config("preset_name", "")
    if saved_preset and hasattr(orchestrator, '_preset_router'):
        router = orchestrator._preset_router
        if hasattr(router, 'set_preset'):
            router.set_preset(saved_preset)
            logger.info(f"프리셋 복원: {saved_preset}")
except Exception:
    pass
```

| 항목 | 리스크 | 대응 |
|:----|:-----:|:------|
| ConfigService save 실패 | 🟢 ZERO | try/except로 무시 → 기존 동작 유지 |

| ConfigEntry 테이블 없음 | 🟢 ZERO | ConfigService는 기존 DB 스키마 사용 |
