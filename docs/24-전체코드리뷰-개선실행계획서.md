### P1-3. TR-1: ATR 포지션 사이징 100배 단위 오류

| 항목 | 내용 |
|------|------|
| 파일 | `live/orchestrator.py:325` |
| 리스크 | MEDIUM — 수정 후 포지션 크기 급증 (0.3% → 10%), 테스트 필수 |
| 소요 | 10분 (1줄) + 30분 (테스트) |

**수정 내용:**
```python
# Before (line 325):
vol_pct = max(vol * 100, 0.5)  # vol은 0~1 범위, 백분율로 변환

# After:
vol_pct = max(vol, 0.5)  # vol은 이미 백분율 (std/mean*100)
```

**리스크 검토:**
- 수정 전: vol=1.5% → vol_pct=150 → position=0.3% (현재 상태)

- **위험**: 포지션 크기가 33배 증가. 실거래 전 반드시 백테스트로 검증
- **완화**: 수정 후 첫 거래 시 `max_position_size`를 임시로 3%로 제한, 점진적 증가

**검증:**
- `tests/test_live/test_orchestrator.py`에 ATR 사이징 단위 테스트
