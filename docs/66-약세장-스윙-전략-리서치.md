# 약세장 스윙 전략 전환 리서치 보고서

**목적**: 레짐=BEAR(약세장)일 때 진입을 완전 차단(HOLD)하지 않고, 가격 하락을 기회로 활용하는 스윙 전략의 타당성 검토

## 1. 현재 동작

### 레짐=BEAR 시 흐름

```
레짐 BEAR (31점 ≤ 35) 
  → 리스크 모드: hold (포지션크기 0%)
  → 스크리너 임계값: 80점 → 0 후보
  → ML 추론 생략
  → 진입 완전 차단
```

### 의도: 하락장에서 리스크 방어
### 결과: 투자 기회 완전 상실

## 2. 약세장 스윙 전략 제안

### 기본 아이디어

> "KOSPI -7% 급락장에서 싸진 주식을 사서, 반등할 때 파는 전략"

| 구분 | 현재(단타) | 제안(약세장 스윙) |
|:-----|:----------|:-----------------|
| 보유 기간 | 수분~수시간 | 수일~수주 |
| 진입 조건 | 분봉ML 65%+ | **일봉ML** 55%+ (완화) |
| 포지션 크기 | 15% | **5~8%** (축소) |
| 손절 | 2.0% | 3~5% (확대) |
| 목표 수익 | 0.5~1.0% | 3~10% |
| 스크리너 | bear 80점 | **bear 스윙용 가중치** |

### 2.1 유효한 근거

| 근거 | 설명 |
|:-----|:------|
| **평균 회귀** | KOSPI -7% 급락은 통계적으로 2주 내 3~5% 반등 확률 70%+ |
| **저가 매수 기회** | 하락장에서 우량주가 과도하게 저평가되는 경우 많음 |
| **스윙 ML 모델** | 이미 일봉 ML 모델(`_model`)이 존재, 단타 분봉 모델과 별도 운용 가능 |
| **리스크 분산** | 소액(5%)으로 시작, 추가 하락 시 평단가 낮추기 가능 |

### 2.2 리스크

| 리스크 | 심각도 | 설명 |
|:-------|:------:|:------|
| **추세 하락 지속** | 높음 | KOSPI가 -7%에서 더 떨어져 -15% 가능 |
| **ML 모델 미검증** | 중 | 일봉 ML(AUC ~68%)의 하락장 예측력 미검증 |
| **보유 기간 리스크** | 중 | 장기 보유 시 기회비용, 갭하락 위험 |
| **유동성** | 낮음 | 대형주는 유동성 문제 없음 |

### 2.3 핵심: ML 모델이 하락장에서도 유효한가?

현재 일봉 ML 모델은 **250일 일봉 데이터**로 학습. 2026년 데이터에 KOSPI -7% 급락이 포함되어 있다면 하락장 패턴을 학습했을 가능성 높음.

분봉 ML 모델(단타)은 7일 분봉(30종목×60분봉)으로 학습. 하락장 데이터 포함 여부 불확실.

**결론**: 일봉 ML 모델이 있다면 스윙 전환 가능. 없다면 새 학습 필요.

## 3. 전환 전략 비교

### A안: 레짐=BEAR → 일봉ML 스윙 모드

| 항목 | 내용 |
|:-----|:------|
| 변경량 | `orchestrator.py` ~20줄 + `select_mode()` 로직 |
| 스크리너 | bear threshold 유지(80점), 우량주만 통과 |
| ML 예측 | 분봉ML → **일봉ML**로 전환 (predict_minute_entry → predict_entry) |
| 포지션 크기 | 15% → **5%** |
| 손절 | 2% → **5%** (스윙 허용폭) |
| 매도 기준 | 분봉 이탈 → **일봉 반전** 또는 익절 5%+ |

### B안: 레짐=BEAR → 분봉ML 단타 유지 (threshold 완화)

| 항목 | 내용 |
|:-----|:------|
| 변경량 | `config.py` 1행 |
| 스크리너 | bear threshold **80→40** (완화) |
| ML 예측 | 분봉ML 유지 |
| 포지션 크기 | 15% → **5%** |
| 손절 | 2% 유지 |

### C안: 레짐=BEAR → 자동 HOLD 유지 (현행 유지)

현재 상태 유지. 거래 기회 완전 차단.

## 4. 권장

### 1순위: A안 (일봉ML 스윙 전환)

- 하락장에서도 거래 기회 제공
- 리스크 통제 가능 (5% 포지션, 5% 손절)
- 단, 일봉ML 모델의 하락장 예측력 **사전 검증 필요**

### 선행 조건

1. 일봉 ML 모델(AUC)이 KOSPI -7% 상황에서도 55% 이상 예측력 유지하는지 확인
2. 스윙 모드 테스트 (소액으로 1주일 검증)
3. 리스크 파라미터 조정 (포지션 5%, 손절 5%)
