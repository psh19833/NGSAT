# 06 · 레짐·모드 전환 고도화 기획서

> 시장 상태 판단 및 매매 모드 자동 전환 정교화  
> 작성: 2026-06-26

---

## 1. 현수준 분석

### 1.1 현재 로직 (strategy/mode_selector.py)

```
강세장(BULL)   → 항상 스윙
중립장(NEUTRAL) → ATR ≥ 1.5%: 단타, ATR ≤ 0.5%: 홀드, 그 외: 스윙
약세장(BEAR)   → 항상 홀드
```

### 1.2 문제점

| 문제 | 설명 |
|------|------|
| **상태 수 부족** | 3가지 레짐 × 2가지 변동성 = 6개 상태를 단순화 |
| **강세장 획일화** | 강세장이면 무조건 스윙 — 추세 강도, 변동성 수준 무시 |
| **중립장 과잉 단순화** | 변동성만 보고 결정 — 방향성, 모멘텀, 거래량 미고려 |
| **임계값 고정** | ATR 1.5% / 0.5% — 시장 상황에 무관하게 동일 |
| **확률 정보 미사용** | 레짐 점수(0~100)의 신뢰도를 반영하지 않음 |

### 1.3 실제 결과 (2026-06-26 오전)

- 레짐: 강세장(76점)
- 모드: 스윙 (항상)
- 매매: ML 예측 확률 50~52%로 매수 임계(55%) 미달 → 0건
- **강세장인데도 매수가 한 건도 안 됨** — "강세=스윙=매수"라는 단순 논리의 한계

## 3. 개선 방안

### 3.1 레짐 평가 → 2차원 스코어로 확장

**기존:** 1차원 점수(0~100) → 3개 구간

**개선:** 2차원 평면 (추세 강도 × 변동성)

```
      고변동성
        │
   약세 │  강세
   ·단타│  ·단타+스윙
────────┼────────→ 추세 강도
   약세 │  강세
   ·홀드│  ·스윙
        │
      저변동성
```

- 추세 강도 축: 현재 레짐 점수(0~100) 그대로 사용
- 변동성 축: ATR 3분위(저/중/고)로 세분화

### 3.2 모드 결정 → 가중치 점수제

**기존:** if-else 조건문 5개

**개선:** 각 모드(스윙/단타/홀드)에 점수를 매기고 최고점 선택

```
mode_score[스윙]  = (추세강도 × 0.6) + (저변동성 × 0.4)
mode_score[단타]  = (중립성   × 0.5) + (고변동성 × 0.5)
mode_score[홀드]  = (약세성   × 0.7) + (리스크 × 0.3)
```

- 장점: 애매한 상황에서도 자연스러운 전환
- 전략설정에서 가중치 조정 가능

### 3.3 변동성 단계 세분화

**기존:** ATR ≥ 1.5% (고) / ≤ 0.5% (저) → 2단계

**개선:** ATR 백분위 기반 3단계

| 단계 | ATR 기준 | 예시 |
|------|---------|------|
| 저변동 | 하위 30% | 하루 등락 ±1% 내외 |
| 중변동 | 30~70% | 하루 등락 ±2% 내외 |
| 고변동 | 상위 30% | 하루 등락 ±3%+ |

### 3.4 동적 임계값

**기존:** ATR 임계값 고정 (1.5%, 0.5%)

**개선:** 최근 60일 ATR의 이동평균 + 표준편차 기반

```
HIGH_ATR = mean_ATR_60d + 1.0 × std_ATR_60d
LOW_ATR  = mean_ATR_60d - 0.5 × std_ATR_60d
```

- 시장 변동성 수준에 자동 적응
- 2024년처럼 고변동 장세에서도 유연하게 대응
