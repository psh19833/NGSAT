## 1. 문제

MTF 필터가 3TF 중 2개 일치(고정 0.67)를 요구하여,  
bear 장(12점)에서 주봉/일봉 하락 + 분봉 반등인 모든 종목을 차단 중.

| 레짐 | 주봉 | 일봉 | 분봉 | 현재(고정0.67) | 필요 |
|---|---|---|---|---|---|
| bear(12점) | 하락 | 하락 | 상승 | ❌ 차단 | ✅ 통과 필요 |
| neutral | 하락 | 상승 | 상승 | ✅ 통과 | ✅ 통과 |
| bull | 상승 | 상승 | 상승 | ✅ 통과 | ✅ 통과 |

---

## 2. 수정 방안

### 2.1 config 필드 추가

`core/config.py` `StrategyConfig`:
```python
# ── 다중 TF 필터 (P-83) ──
mtf_bear_threshold: float = 0.50   # bear: 2TF 중 1개
mtf_neutral_threshold: float = 0.67  # neutral: 3TF 중 2개
mtf_bull_threshold: float = 0.67    # bull: 3TF 중 2개
```

### 2.2 entry_planner.py MTF 필터 수정

`_check_multitf_alignment()`에서 `self._strategy`로 임계값 조회:

```python
# 레짐별 임계값 (strategy config → DB config로 동적 전환)
regime = self._get_current_regime()  # or parameter
thresholds = {
    "bear": self._strategy.mtf_bear_threshold,
    "neutral": self._strategy.mtf_neutral_threshold,
    "bull": self._strategy.mtf_bull_threshold,
}
mtf_threshold = thresholds.get(regime, 0.5)
aligned = score >= mtf_threshold
```

### 2.3 regime 정보 전달

`_check_multitf_alignment()`가 regime 정보를 받아야 함.
- `plan_entries()`가 `regime_result`를 이미 가지고 있음
- `_evaluate_and_execute()`로 전달 → MTF 필터에 주입

## 4. 리스크 평가

| 항목 | 리스크 | 내용 |
|---|---|---|
| **bear threshold=0.50** | 🟡 **중간** | 2TF 중 1개만 일치해도 통과. 분봉 단독 상승 시 무분별한 진입 가능성 |
| **DB config 우선** | 🟢 안전 | 기본값=0.50/0.67/0.67. 대시보드에서 실시간 조정 가능 |
| **기존 로직 영향** | 🟢 없음 | 임계값만 동적으로 변경. 필터 자체는 동일 |
| **분봉 위험** | 🟡 bear에서 분봉만 보고 진입 시 주봉 하락 리스크 존재 | 단기 청산(min_hold 20분)과 stop_loss가 2차 방어 |

**총평: 🟡 Medium — bear threshold 0.50이 다소 느슨하나, min_hold 20분 + stop_loss가 하방 방어. 기본값으로 합리적이며 DB config로 즉시 조정 가능.**
