## 1. 문제 증상

```bash
npm run build
```
```
[vite:esbuild] Transform failed with 1 error:
214|      </ErrorBoundary>
error: Unexpected end of file
```

---

## 2. 실제 원인 (2026-06-28 18:30 최종 확인)

**`App.jsx`에서 `App()` 함수를 닫는 `}`가 1개 누락되었습니다.**

### 상세

Toast 및 ErrorBoundary 컴포넌트를 추가하는 과정에서, 함수의 마지막 부분을 수정할 때 실수로 `}`를 지웠습니다.

```javascript
// 원래 코드 (정상)
  </div>
)
}            ← App() 함수 닫는 중괄호 ✅

// 수정 후 코드 (오류)
    </ErrorBoundary>
)            ← }가 없음! ❌
```

중괄호 하나가 없었기 때문에:
- ESBuild 파서가 `export default function App() {` 에서 시작된 함수 몸체를 끝까지 닫지 못함
- 파일 끝(EOF)에 도달할 때까지 `}`를 찾지 못해 "Unexpected end of file" 오류 발생
- 오류가 `</ErrorBoundary>` 줄에서 발생한 것은 **그 줄 이후에 닫힘이 없어서**지, `ErrorBoundary` 자체와는 무관

### 잘못된 이전 결론 (정정)

**"esbuild 0.25.12 버그"라고 단정한 것은 잘못된 분석이었습니다.** 오류 위치(`</ErrorBoundary>` 줄)만 보고 esbuild 버전 문제로 오해했으나, 실제로는 **코드 문법 오류**였습니다. Root esbuild(0.28.1)로 직접 파싱해도 동일한 오류가 발생하는 것을 확인했습니다.

## 5. 교훈

1. **"esbuild 버그"라고 의심하기 전에 코드 문법부터 확인할 것**
2. `}` 하나 빠져도 빌드가 안 될 수 있음
3. `npx esbuild src/App.jsx` 로 직접 파싱 테스트하면 원인 파악이 빠름
4. 원인이 불확실하면 "esbuild 버그"로 단정하지 말고 가능성을 열어둘 것
