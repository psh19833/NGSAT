# 스크리너 상세 점수 표시 — 구현 계획서

**목적**: 대시보드에서 스크리너 통과 종목 클릭 시 각 지표별 점수 기여도를 시각적으로 표시

---

## 1. 현재 상태

### 백엔드 → 프론트엔드 데이터 흐름

```
orchestrator.py run_cycle()
  → _last_diagnosis["screened"] = [{code, name, score, reason}, ...]
    → GET /api/diagnosis
      → DiagnosisPanel.jsx (screened.map)
```

### 현재 전송되는 데이터

```json
{
  "code": "018260",
  "name": "삼성에스디에스",
  "score": 70.7,
  "reason": "점수 70.7/100 | RSI 68.9 | MFI 48.1 | ..."
}
```

### 현재 표시

```
📊 스크리너 (1종목 통과)
  삼성에스디에스(018260)          70.7점
```

## 2. 변경 사항

### 2.1 백엔드: `indicators` 필드 추가

**파일**: `live/orchestrator.py` (L424~427)

**변경**:
```python
# 현재
result.screened = [
    {"code": c.code, "name": c.name, "score": round(c.score, 1),
     "reason": c.reason}
    for c in screen_result.candidates
]

# 변경
result.screened = [
    {"code": c.code, "name": c.name, "score": round(c.score, 1),
     "reason": c.reason, "indicators": c.indicators}
    for c in screen_result.candidates
]
```

**영향**: 없음 (`indicators`는 ScreenCandidate에 이미 존재)

### 2.2 프론트엔드: 클릭 시 상세 패널 표시

**파일**: `dashboard/frontend/src/components/DiagnosisPanel.jsx`

**변경**: 행 클릭 시 접이식 상세 패널 추가

```jsx
// 현재
{screened.map((s, i) => (
  <div key={i} className="...">
    <span>{s.name}({s.code})</span>
    <span>{s.score}점</span>
  </div>
))}

// 변경 (useState로 expanded 관리)
const [expandedStock, setExpandedStock] = useState(null)

{screened.map((s, i) => (
  <>
    <div key={i} className="... cursor-pointer"
         onClick={() => setExpandedStock(expandedStock === i ? null : i)}>
      <span>{s.name}({s.code})</span>
      <span>{s.score}점</span>
      {/* 화살표 아이콘 */}
    </div>
    {expandedStock === i && (
      <div className="ml-4 pl-2 border-l-2 border-ngsat-border text-xs space-y-1">
        {/* 지표별 점수 바 */}
      </div>
    )}
  </>
))}
```

### 2.3 점수 시각화 포맷

각 지표를 **점수 바 + 수치**로 표시:

```
▼ 삼성에스디에스(018260) — 70.7점
  RSI 68.9 ━━━━━━━━━━━━━━━━ 90%  (가중치 20%)
  MFI 48.1 ━━━━━━━━━━━━━    60%  (가중치 15%)
  ADX DI   ━━━━━━━━━        50%  (가중치 10%)
  OBV      ━━━━━━━━━━━━━━   80%  (가중치 10%)  
  MA 정렬  ━━━━━━━━━━━━━━━━ 90%  (가중치 10%)
  거래량   ━━━━━━━━━        55%  (가중치 10%)
  패턴     ━━━━━━━━━━━━━━   70%  (가중치 15%)
  캔들     ━━━━━━━━━━━      60%  (가중치 10%)
  ─────────────────────────────────
  최종 점수 = 각 (점수 × 가중치) 합 / 가중치 총합 = 70.7점
```

### 2.4 필요한 indicator 필드

백엔드에서 전송되는 `indicators` dict에는 다음 값이 포함되어 있음:

| 키 | 설명 | 표시 |
|:---|:-----|:------|
| `rsi` | RSI 값 | `RSI {val}` |
| `mfi` | MFI 값 | `MFI {val}` |
| `adx` | ADX 값 | `ADX {val}` |
| `di_plus` | DI+ | `DI+{val}` |
| `di_minus` | DI- | `DI-{val}` |
| `obv_slope` | OBV 기울기 | `OBV {val:+.2f}` |
| `ma5` | 5일 MA | `MA5 {val:.0f}` |
| `ma20` | 20일 MA | `MA20 {val:.0f}` |
| `volume_ratio` | 거래량 비율 | `거래량 {val:.1f}` |
| `atr_pct` | 변동성 | `ATR {val:.2f}%` |
| `rs` | 상대강도 | `RS {val:.2f}` |
| `stochastic_k` | 스토캐스틱 K | `K={val:.0f}` |

## 3. 리스크 분석

| 구분 | 리스크 | 확률 | 대응 |
|:-----|:-------|:----:|:------|
| 백엔드 | `indicators` 키 누락 시 FE 오류 | 낮음 | FE에서 `s.indicators?.rsi ?? '?'` 옵셔널 체이닝 |
| 프론트엔드 | 긴 리스트에서 여러 개 펼치면 UX 저하 | 중 | 한 번에 1개만 펼치기 (accordion) |
| 프론트엔드 | 지표 이름이 영어 → 사용자 이해도 | 중 | 한글 레이블 매핑 테이블 사용 |
