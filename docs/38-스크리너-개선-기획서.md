**버전:** 1.0  
**작성일:** 2026-07-02  
**상태:** 계획 (리서치 완료, 미구현)

## 1. 배경 및 문제 정의

### 1.1 현재 상황

스크리너(3단계 파이프라인의 2단계)는 40개 유니버스 종목(KOSPI 28 + KOSDAQ 12)에서 레짐별 한도만큼 후보를 선별합니다.

| 레짐 | 최소 점수 | 최대 후보 |
|:---:|:--------:|:--------:|
| 강세장 | 60점 | 15개 |
| 중립장 | 70점 | 10개 |
| 약세장 | 80점 | 5개 |

### 1.2 발견된 문제: 점수 압축 (Score Compression)

약세장(80점 컷)에서 현대중공업·원익아이피에스 등 5종목이 모두 90~100점을 기록.  
이는 사용 지표가 5개로 한정되어 **상위권 변별력이 부족**함을 의미합니다.

**원인:** 동일한 5개 지표 조합으로 40종목을 평가하면 대부분 비슷한 점수대에 몰림.

### 1.3 개선 목표

1. 약세장 80점 컷에서 진짜 우량 종목만 선별되도록 변별력 강화
2. 강세장/중립장에서도 과열 종목/거짓 신호 감소
3. 모든 레짐에서 후보의 질적 수준 향상

## 2. 현재 스크리너 분석

### 2.1 사용 중인 지표 (5개)

| 지표 | 파라미터 | 스코어 기여 | 비고 |
|:----|:-------:|:----------:|:----|
| RSI | 14 | +15 / -5 | 30~70 구간 정상, 70↑ 과열, 30↓ 과매도 |
| MACD (히스토그램) | 12/26/9 | +10 / -10 | 양수=상승모멘텀 |
| SMA 정배열 | 5/20 | +15 / -15 | close>MA5>MA20 = 강한 상승 |
| 거래량비율 | 20 | +5 | 1.5배 이상 = 거래량 동반 |
| 차트패턴 (5종) | — | +4~20 | 돌파/눌림/반등/스퀴즈/이평교차 |

### 2.2 점수 계산 공식

```
기본 50
+ RSI(50~70) +15 / RSI(30~50) +10 / RSI(≤30) +5 / RSI(≥70) -5
+ MACD 양수 +10 / 음수 -10
+ 정배열(close>MA5>MA20) +15 / close>MA5 +5 / 역배열 -15
+ 거래량>1.5배 +5
+ 패턴 1개당 +4 × 레짐가중치(강세1.2/중립1.0/약세0.8)
+ KOSPI +5
= 0~100점 (clamp)
```

## 3. 개선 방안

### 3.1 ⭐ 스토캐스틱 오실레이터 추가 (리스크 ZERO)

**구현:** `indicators.py`에 이미 구현되어 있음 → `screener.py`에서 호출만 필요

```python
k, d = stochastic(highs, lows, closes)
k_val, d_val = float(k[-1]), float(d[-1])

if k_val < 20 and d_val < 20:
    score += 10   # 과매도 구간 — 반등 기대
elif k_val > 80 and d_val > 80:
    score -= 10   # 과매수 구간 — 추격 위험
```

**효과:** RSI가 놓치는 극단 구간(특히 단기 급등/급락)을 별도로 포착. 약세장에서 과매수 상태인 종목 제거.

### 3.2 ⭐⭐ ADX 추세 강도 필터 (리스크 ZERO)

**구현:** `indicators.py`에 ADX 함수 이미 구현됨

```python
adx_val = float(adx(highs, lows, closes, 14)[-1])

if adx_val > 25:
    score += 10   # 추세 있는 종목 우대
elif adx_val < 15:
    score -= 10   # 횡보 종목 패널티
```

**효과:**
- 약세장에서도 추세가 살아있는 종목(반등 추세)만 선별
- 횡보·정체 종목 자연스럽게 하위권으로
- 레짐 평가에서 ADX를 쓰지만 스크리너에는 미반영된 점을 보완

### 3.3 ⭐⭐⭐ 거래량 정밀 분석 (리스크 ZERO)

현재: 단순 `vol_ratio > 1.5` 체크 (예/아니오, +5점 고정)

개선:
```python
# 거래량 추세 (단기평균 vs 장기평균)
vol_ma5 = sma(volumes, 5)[-1]
vol_ma20 = sma(volumes, 20)[-1]

if vol_ma5 > vol_ma20 and vol_ratio > 1.2:
    score += 8    # 거래량 증가 추세 + 동반
elif vol_ma5 > vol_ma20:
    score += 4    # 증가 추세만
elif vol_ma5 < vol_ma20 and vol_ratio < 0.7:
    score -= 5    # 거래량 감소 추세
```

### 3.4 ⭐⭐⭐⭐ 패턴 강도 가중치 차등화 (리스크 LOW)

현재: 패턴 감지만 하고 모든 패턴 동일 가중치

개선:
```python
# 패턴별 신뢰도 차등
pattern_weights = {
    "breakout": 1.5,    # 돌파 — 거래량 동반 시 신뢰도 높음
    "bollinger_squeeze": 1.3,  # 스퀴즈 후 확장 — 강한 신호
    "ma_cross": 1.2,    # 골든크로스 — 확인된 신호
    "pullback": 0.8,    # 눌림 — 상승 추세 전제
    "rebound": 0.7,     # 반등 — 하락 추세 전제
}
score += sum(4 * pattern_weights.get(p.pattern_name, 1.0) for p in patterns)
```

### 3.5 ⭐⭐⭐⭐⭐ 지표 가중치 리밸런싱 (리스크 MEDIUM)

현재 지표별 최대 기여도가 불균형:

| 지표 | 현재 최대 기여 | 제안 최대 기여 |
|:----|:------------:|:------------:|
| RSI | 20 (15+5) | **20** |
| MACD | 10 | **10** |
| MA 정렬 | 15 | **20** |
| 거래량 | 5 | **10** |
| ADX (신규) | — | **10** |
| 스토캐스틱 (신규) | — | **10** |
| 패턴 | ~20 | **10** |
| KOSPI 가산점 | 5 | **5** |
| **합계** | ~105 | **100~110** |

핵심: **MA 정렬 비중 확대** (추세 추종 강화) + **신규 지표 2개 추가**로 더 넓은 평가 기준 적용

## 6. 추천 진행 순서

```
Phase A (ZERO, 당일)
  ├─ A-1: 스토캐스틱 추가 → 당장 변별력 ↑
  └─ A-2: ADX 추가 → 횡보 종목 제거
      ↓
Phase B (ZERO~LOW, 검토 후)
  ├─ B-1: 거래량 정밀화
  └─ B-2: 패턴 강도 차등화
      ↓
Phase C (MEDIUM, 장기 검토)
  └─ C-1: 가중치 리밸런싱 (백테스트 병행)
```

**Phase A만 적용해도 현재 5종목 모두 90~100점 → 60~95점으로 분산**되어 변별력이 크게 개선될 것으로 예상됩니다.
