## 2. 상세 내역

### 🔴 P0 — 대시보드 크래시

#### P0-1: TradesTable useMemo Hooks 위반 (Critical #1)

**파일**: `dashboard/frontend/src/components/TradesTable.jsx`  
**증상**: early return (`!data`, `tradeList.length === 0`) 이후 `useMemo` 호출 → React Hooks 규칙 위반. 거래 내역 0건 시 크래시.  
**수정**: `useMemo`를 early return 이전으로 이동, 또는 `groupTradesByDate`를 일반 함수 호출로 변경.  
**변경량**: ~3줄  
**리스크**: 없음 (Hooks 호출 순서만 변경)

---

### 🟠 P1 — 기능 오동작

#### P1-1: getTrades 페이지네이션 누락 (High #2)

**파일**: `dashboard/frontend/src/api.js` + `TradesTable.jsx`  
**증상**: `getTrades()`가 파라미터 없이 정의 → `fetchJSON('/trades')`. 호출부(`PAGE_SIZE, offset`) 전달 안 됨.  
**수정**: `getTrades: (limit, offset) => fetchJSON(\`/trades?limit=${limit}&offset=${offset}\`)`  
**변경량**: 1행  
**리스크**: 없음 (API URL에 파라미터만 추가)

#### P1-2: BacktestPanel 폴링 미구현 (High #3)

**파일**: `dashboard/frontend/src/components/BacktestPanel.jsx`  
**증상**: `pollRef` 선언만 있고 실제 폴링 로직 없음. 백테스트 비동기 결과 수신 불가.  
**수정**: `setInterval` + `api.getBacktestState()` 폴링 로직 추가. 완료 시 `clearInterval`.  
**변경량**: ~15줄  
**리스크**: 낮음 (UI만 영향)

#### P1-3: handleRestart 서버 확인 누락 (High #4)

**파일**: `dashboard/frontend/src/App.jsx`  
**증상**: `await api.restart()` 후 즉시 "재시작 완료" 토스트. 서버 재시작 완료 확인 안 함.  
**수정**: `api.getStatus()` 폴링으로 연결 확인 후 "완료" 표시. 타임아웃(30초) 처리.  
**변경량**: ~10줄  
**리스크**: 낮음

#### P1-4: handleControl forcehold await 누락 (Low #10)

**파일**: `App.jsx`  
**증상**: `api.forceHold(code)` 후 `refreshAll()` 호출 시 `await` 없음.  
**수정**: `await api.forceHold(code)` + `await refreshAll()`  
**변경량**: 2행  
**리스크**: 없음
