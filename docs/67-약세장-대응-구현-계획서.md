### 1.2 신규 모듈: `live/position_sizer.py`

```python
# live/position_sizer.py — 포지션 크기 동적 계산
def calc_position_size(regime_score: float, regime: str, atr_pct: float, config) -> float:
    """Return position size as fraction of total capital (0.0 ~ 0.15)."""
```

**이유**: 포지션 사이징 로직이 orchestrator.py의 mode_selection과 risk 관리 사이에 끼어들면 모듈 비대화. 별도 모듈로 분리하면 테스트/수정/독립적 검증 가능.

섹터 가중치 + 듀얼 모멘텀 점수를 **screener.py에서 분리**

**이유**: 섹터 점수를 `scorer.py`에 넣으면 scorer.py가 비대해짐. 별도 모듈로 분리하면 섹터 목록만 수정하면 됨.

## 2. 상세 구현

### 2.1 Kelly 포지셔닝 (`live/position_sizer.py`)

```python
def calc_position_size(
    regime_score: float,
    regime: str,
    atr_pct: float,
    max_position_pct: float = 0.15,
) -> float:
    """레짐 점수 기반 동적 포지션 크기 (0.0 ~ max_position_pct)

    - BEAR + 점수 ≤ 15: 2%
    - BEAR + 점수 ≤ 25: 5%  
    - BEAR + 점수 ≤ 35: 8%
    - NEUTRAL: 10~15% (ATR 비례)
    - BULL: 15%
    """
```

### 2.2 섹터 로테이션 (`strategy/sector_weights.py`)

```python
MOMENTUM_BONUS = {
```

### 2.3 통합 (`scorer.py`)

```python
def compute_total_score(
    indicator_scores: dict[str, float],
    regime: str,
    sector_bonus: float = 0.0,      # 섹터 가산점
    momentum_bonus: float = 0.0,    # 모멘텀 가산점
) -> float:
    score = weighted_sum / total_weight
    score += sector_bonus        # 점수에 직접 가산
    score += momentum_bonus       # 점수에 직접 가산
    return max(0.0, min(100.0, score))
```

## 3. 유지보수 관점

### 3.1 모듈 분리로 얻는 이점

| 관점 | 분리 전 (orchestrator.py 1개) | 분리 후 (3개 모듈) |
|:-----|:------------------------------|:--------------------|
| **SRP** | 포지션+리스크+모드=1개 | **position_sizer / risk / mode** 각각 분리 |
| **테스트** | orchestrator.py 단위 테스트 불가 | position_sizer 단독 테스트 가능 |
| **수정** | 포지션 로직 수정 → orchestrator 전체 영향 | position_sizer.py만 수정 |
| **가독성** | 1100줄 집중 | 각 50~150줄, 명확 |

### 3.2 유지보수 원칙

1. **Scorer.py는 200줄 이하 유지** → 새 지표 추가 시 scorer.py가 아닌 indicators.py에 추가
2. **Sector_weights.py는 설정 파일처럼** → 값만 변경, 로직 없음
3. **Position_sizer.py는 pure function** → side effect 없음, 입력→출력만

### 3.3 코드 길이 제한

| 모듈 | 현재 | 목표 | 초과 시 대책 |
|:-----|:----:|:----:|:------------|
| `orchestrator.py` | 1097 | **< 1100** | run_cycle() 분할 (prepare/fetch/eval/execute/report) |
| `screener.py` | 395 | **< 400** | scorer.py로 추가 분리 |
| `scorer.py` | 180 | **< 220** | 유지 |
| `risk.py` | 150 | **< 200** | 유지 |
| `indicators.py` | 555 | **< 600** | 유지 |
| **🆕 position_sizer.py** | — | **< 80** | 분할 불필요 |
| **🆕 sector_weights.py** | — | **< 60** | 분할 불필요 |
