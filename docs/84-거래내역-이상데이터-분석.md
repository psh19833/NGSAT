## 1. 발견된 이상 항목

### 1.1 created_at 표시 오류 (API에서 N/A)

**증상**: 대시보드 거래내역에서 모든 거래의 created_at이 "N/A"로 표시됨.  
**원인**: TradeRecord의 `created_at` 컬럼 저장 포맷이 이원화되어 있음.

| 포맷 | 예시 | 해당 레코드 |
|---|---|---|
| ISO 8601 (T) | `2026-06-26T10:19:08.375846` | id=1 (최초 1건) |
| SQL datetime | `2026-07-14 10:44:06.397242` | id=2~85 (나머지 80건) |

`core/models.py:55`: `default=datetime.now` — Python datetime의 기본 str 포맷이 SQL datetime 형식이라 ISO 파싱에서 실패.  
**API serializer가 ISO 포맷만 지원**하여 SQL datetime 형식은 N/A로 표시.

## 2. 수정 방안

### 2.1 created_at 포맷 통일 (P0)

**원인**: `core/models.py`에서 `default=datetime.now` 사용.  
`datetime.now()`는 SQLAlchemy DateTime 컬럼에 저장될 때 `str(datetime.now())` = `YYYY-MM-DD HH:MM:SS.mmmmmm` 포맷으로 저장됨.

**수정**: `datetime.utcnow` → ISO 포맷 또는 `func.now()` 사용

```python
# AS-IS
created_at: Mapped[datetime] = mapped_column(DateTime, default=datetime.now)

# TO-BE (2가지 옵션)
# Option A: SQLAlchemy func.now() — DB 레벨에서 관리
created_at: Mapped[datetime] = mapped_column(DateTime, server_default=func.now())

# Option B: UTC+9 ISO 포맷 함수
def _kst_now():
    return datetime.now(timezone(timedelta(hours=9)))

created_at: Mapped[datetime] = mapped_column(DateTime, default=_kst_now)
```

**리스크**: 🟢 낮음. 기존 레코드의 포맷은 그대로 유지되지만, 신규 레코드부터 통일된 포맷으로 저장됨.

### 2.2 API serializer 수정 (P0)

created_at을 파싱할 때 ISO 포맷과 SQL datetime 포맷을 모두 지원하도록 수정:

```python
# api.py serializer에서
if isinstance(created_at, str):
    try:
        # ISO 8601
        dt = datetime.fromisoformat(created_at)
    except ValueError:
        # SQL datetime format: '2026-07-14 10:44:06.397242'
        dt = datetime.strptime(created_at, "%Y-%m-%d %H:%M:%S.%f")
    created_at = dt.isoformat()
```

**리스크**: 🟢 낮음. 문자열 파싱만 추가.

### 2.3 execute_sell 실패 시 record_sell 방지 (P1)

`live/exit_manager.py:_execute_exit()`에서 `exec_result.success`가 False면 record_sell을 skip하도록 수정:

```python
# 현재 (문제): exec_result.success 여부와 무관하게 record_sell 호출
self._trade_recorder.record_sell(exec_result, position, sell_price, ...)
if exec_result.success:
    return {"sold": True}
return {"error": f"매도 실패 ..."}

# 수정: 실패 시 record_sell skip
if exec_result.success:
    self._trade_recorder.record_sell(exec_result, position, sell_price, ...)
    return {"sold": True}
return {"error": f"매도 실패 ..."}
```

**리스크**: 🟡 중간. `record_sell`이 실패 sell도 기록하지 못하게 되어 거래내역 누락 가능성.  
하지만 실패한 sell을 기록하는 건 의미 없음.
