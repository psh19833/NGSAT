# 레짐 평가 리서치 — 장중보정 후 enum 전환

## 현재 구조

```
레짐 평가 (5사이클마다) 
  → evaluate_regime() → RegimeResult(regime=NEUTRAL, score=46)
  → 장중보정 → score=31으로 변경, BUT regime=NEUTRAL 유지
  → 스크리너에 regime=NEUTRAL 전달 → threshold=30
```

## 접근법 분석

### A안: 장중보정 후 레짐 enum 즉시 재평가 (권장)

orchestrator.py에서 장중보정 후 보정된 점수로 bear/bull threshold 재확인.

```python
# After correction
if corrected_score <= bear_t:
    regime_result.regime = MarketRegime.BEAR
elif corrected_score >= bull_t:
    regime_result.regime = MarketRegime.BULL
```

**변경량**: orchestrator.py 5줄  
**리스크**: 낮음 — hysteresis 없이 바로 전환되나, 장중 등락 반복 시 깜빡임 가능성  
**유지보수**: ✅ 단순, 이해 쉬움

### B안: 레짐 평가 주기 단축

현재 5사이클 보호 → 2~3사이클로 단축.

```python
# orchestrator.py L328
if self._cycle_count < 3 and self._last_regime is not None:
```

**변경량**: orchestrator.py 1행  
**리스크**: 중 — 초기 안정화 시간 단축으로 잘못된 레짐 가능성  
**유지보수**: ✅ 단순하나 효과 제한적 (보정 자체를 반영하는 것은 아님)

### C안: 스크리너가 보정 점수 직접 사용

`screener.py`가 `regime_result.regime` 대신 `regime_result.score`(보정 후)를 받아 threshold를 동적 결정.

**변경량**: screener.py + orchestrator.py ~15줄  
**리스크**: 중 — screener 로직 변경, 기존 동작 영향 가능  
**유지보수**: ❌ 복잡, 이해 어려움
