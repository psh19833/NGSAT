# 버그 리포트: 장 종료 후 시장 레짐 평가 이상

> 일자: 2026-06-30
> 상태: 확인됨 (미수정)

---

## 1. 증상

정규장 종료 후(15:30 이후 또는 주말) 대시보드에서 **"매매 시작" 버튼을 누르면 시장 레짐이 평가됨**. 장이 열리지 않았는데도 레짐 점수와 강세/중립/약세 판정이 표시됨.

## 2. 원인 분석

### 2.1 실행 흐름

`main.py` 메인 루프 → `orchestrator.run_cycle(index_prices, universe)`:

```
Step 1: 계좌 조회
Step 2: 일일 손실 체크
Step 3: 레짐 평가 ← 항상 실행됨 (market hours 무관)
Step 4A: 모드 선택 ← 항상 실행됨 (market hours 무관)
Step 4B: 스크리닝 ← 항상 실행됨 (market hours 무관)
Step 5: ML 예측 + 매수 실행
  → 344: market_open = is_market_hours()
  → 345: if not market_open: "장 운영 시간 아님 — 신규 진입 생략"
  →     (레짐 평가는 이미 끝남)
Step 6: 청산 체크
```

**Step 3(레짐 평가)과 Step 4A(모드 선택), Step 4B(스크리닝)는 `market_open` 체크보다 먼저 실행**됩니다. 장이 닫혀도 무조건 실행됩니다.

### 2.2 사용되는 데이터

레짐 평가에 사용되는 `index_prices`는 `data_provider.refresh_prices()`가 반환한 값입니다. 장 종료 후에는 **KIS API가 마지막 거래일의 데이터를 반환**합니다.

즉, 토요일 오후에 평가된 레짐은 **금요일 장 마감 기준**의 데이터로 평가된 것입니다.

### 2.3 `is_market_hours` 함수의 위치

`is_market_hours()` 함수는 `core/types.py`에 정의되어 있고, 오케스트레이터에서 import해서 사용합니다. 그런데 이 함수는 **Step 6(매수 실행)**에서만 호출되고, **Step 3(레짐 평가)보다 앞서 호출되지 않습니다.**

## 3. 영향

| 영향 | 설명 |
|------|------|
| **레짐 점수 오해** | 장 종료 후에도 레짐 점수가 표시되지만, 이는 **마지막 거래일의 데이터** — 현재 시장 상태가 아님 |
| **모드 선택 오작동** | 장 종료 후에도 스윙/단타/관망 모드가 선택됨 (실제로는 무의미) |
| **대시보드 표시 혼란** | "알 수 없음"이 아닌 레짐 점수가 표시되어 사용자가 현재 시장 상태로 오인 |

## 4. 권장 수정 방안

### A안 (권장): 레짐 평가를 market_open 뒤로 이동

`run_cycle()`에서 **가장 먼저** `is_market_hours()` 체크하고, 장이 아니면 레짐 평가/스크리닝/ML 전부 스킵.

```python
market_open = is_market_hours()
if not market_open:
    # 장 종료: 청산만 실행, 레짐/스크리닝/진입 생략
    result.reason = "장 운영 시간 아님 — 청산만 실행"
    # Step 7: 청산만 진행
    return await self._run_exit_only(current_positions, ...)
```

### B안: 레짐 평가 스킵 + 대시보드 표시 제한

장이 아닐 때는 레짐 평가를 건너뛰고, 대시보드에 "장 종료 — 마지막 평가: 2026-06-30 15:30" 같이 표시.

### C안: 레짐 평가 시점에 market_open 플래그 저장

레짐 평가는 하되, 평가 결과에 `evaluated_at_market_hours: bool` 필드를 추가해서 대시보드에서 장중 데이터인지 구분.

---

**우선순위**: LOW — 장 종료 후 매매시작 버튼을 누를 일이 거의 없고, 청산 자체는 정상 동작함. 다만 데이터 정합성 측면에서 개선 필요.
