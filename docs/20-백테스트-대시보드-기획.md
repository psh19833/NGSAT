# 대시보드 백테스트 탭 — 기획 및 리스크 검토

## 제안: 대시보드 8번째 탭으로 "백테스트" 추가

```
현재 탭: 운영요약 | 계좌 | 포지션 | 거래내역 | 운영제어 | 진단현황 | 전략설정
추가:                                          + 백테스트
```

---

## 화면 구성

### ① 백테스트 실행 폼 (좌측)
- **기간**: 시작일 ~ 종료일 (달력 선택)
- **초기 자본**: 입력 (기본 1,000만원)
- **ML 모델**: 현재 모델 / 없이(지표만) 선택
- **실행 버튼**: 🚀 백테스트 실행

### ② 결과 요약 (우측 상단)
```
수익률: +12.3%  |  승률: 58.7%  |  MDD: -8.2%  |  거래: 47회
초기: 10,000,000 → 최종: 11,230,000  |  데이터: KIS 실데이터 38종목
```

### ③ 자산 추이 차트 (중간)
- recharts sparkline (EquityChart 재사용)
- 붉은색 구간 = 손실, 초록색 = 이익

### ④ 거래 내역 (하단)
- TradesTable 재사용 (백테스트 거래내역 표시)
- 행 클릭 시 근거 확인

### ⑤ 이전 결과 목록 (사이드 패널)
- 지금까지 실행한 백테스트 결과 리스트
- 클릭하면 해당 결과 로드

## 리스크 검토

| 리스크 | 등급 | 해결 |
|--------|------|------|
| 백테스트 중 대시보드 느려짐 | 🟡 MEDIUM | `run_in_executor()` 별도 스레드 |
| 실행 중 실수로 재실행 | 🟢 LOW | 버튼 비활성화 |
| 결과가 합성데이터인지 실제인지 혼란 | 🟡 MEDIUM | 결과 상단에 출처 명확 표시 |
| 프론트 빌드 불가 (esbuild) | 🔴 HIGH | 현재 esbuild 이슈로 빌드 안 됨 |
| 많은 데이터로 프론트 느려짐 | 🟢 LOW | 페이지네이션 재사용 |

## 가장 큰 문제: esbuild (프론트 빌드)

백테스트 탭을 만들려면 **React 코드를 수정하고 빌드**해야 합니다. 현재 esbuild 이슈로 `npm run build`가 안 되는 상태라, 새 탭을 추가해도 대시보드에 반영할 수 없습니다.

**해결 순서**:
1. 먼저 esbuild 이슈 해결 (ESBUILD_BINARY_PATH 등)
2. 그 다음 백테스트 탭 개발
3. 빌드 → 배포
