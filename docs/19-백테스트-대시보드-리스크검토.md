# 대시보드 백테스트 기능 추가 — 리스크 검토 보고서

> 요청: 대시보드에서 백테스트를 실행할 수 있도록 기능 추가
> 현재: `python main.py --backtest` 터미널 전용

---

## 제안 기능

대시보드에 **백테스트 탭** 추가:
1. 백테스트 실행 폼 (기간·종목·설정 선택)
2. 실행 버튼 + 진행률 표시
3. 결과 요약 (수익률·승률·MDD·거래횟수)
4. 이전 결과 히스토리

---

## 리스크 검토

### 1. RealDataProvider vs 합성 데이터 (MEDIUM)

**현재**: `run_backtest()`은 KIS 실데이터 로드 시도 → 실패 시 합성 데이터 fallback
**문제**: 대시보드에서 실행하면 사용자는 "이 결과가 진짜야? 가짜야?"를 알 수 없음

**완화**: 결과 화면에 "KIS 실데이터 기반" / "합성 데이터 기반" 명확히 표시

---

### 2. CPU/메모리 폭주 (MEDIUM)

**현재**: 백테스트는 단일 스레드에서 실행되며, 40종목 × 250일 데이터를 처리
**문제**: 대시보드와 같은 프로세스에서 실행되면 API 응답이 느려지거나 타임아웃 발생 가능
- 40종목 백테스트: 약 3~10초 소요
- 그동안 대시보드 API 응답 불가

**완화**: 
- `asyncio.to_thread()` 또는 `run_in_executor()`로 별도 스레드에서 실행
- 진행률을 shared state로 업데이트 → 프론트에서 폴링
- 실행 중에는 대시보드 응답 유지

---

### 3. 동시 실행 (LOW)

**문제**: 사용자가 실수로 백테스트를 두 번 연속 실행하면 자원 경합
**완화**: 실행 중에는 버튼 비활성화 + "실행 중..." 표시

---

### 4. 결과 저장 (LOW)

**문제**: 백테스트 결과를 어디에 저장할지 미정
**옵션**:
- A: 메모리에만 보관 (프로세스 재시작 시 사라짐) — 간단
- B: DB에 저장 (히스토리 열람 가능) — 복잡

**추천**: Phase 1은 메모리만, Phase 2에서 DB 저장 추가

---

### 5. 백테스트 설정 UI (LOW)

**문제**: StrategyConfig 30+개 필드를 모두 백테스트에도 적용해야 정확
**해결**: 현재 대시보드 전략설정 값을 그대로 사용. 백테스트 전용 설정 UI는 추후
