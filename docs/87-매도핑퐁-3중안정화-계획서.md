## 1. 문제

### 현재 매도 파이프라인

```
분봉ML 예측 → sell 확률 ≤ 30%? → YES → execute_exit()
```

문제: 매수 후 10초, 분봉 1~2개만 추가돼도 확률이 60%→14%로 급변 → 즉시 청산.

### 신한지주 사례

```
매수: 109,100원 (분봉ML 60%↑, 좋은 신호)
매도: 108,100원 (10초 후, ML 14% ≤ 30%)
PnL: -1,000원 (-0.9%) — 거래세만 0.18%×216,400 = 390원 손실
```

---

## 2. 수정 방안 (3중 안정화)

### ① Sell threshold 30%→20%

ML 청산 예측 확률 기준을 낮춰서 "팔아야 한다"는 기준을 더 까다롭게.

```python
# ml/inference.py predict_minute_exit
# return MLPrediction(..., action=SELL if prob <= 0.30)
#                                                     0.30 → 0.20
```

### ② 최소 PnL 조건 (abs ≥ 1%)

ML이 SELL을 예측해도, 수익/손실이 1% 미만이면 청산 보류.

```python
# exit_manager.py _check_single_exit ML exit section
if exit_pred and exit_pred.action == DecisionAction.SELL:
    if abs(position.profit_loss_pct) < 1.0:
        # 의미 없는 미세 청산 방지
        continue  # 청산 안 함
```

### ③ 분봉 bars 30→60

분봉 데이터를 30개→60개(5시간→10시간)로 늘려 안정적인 패턴 인식.

```python
# exit_manager.py _fetch_minute_prices
# bars = broker._minute_builder.get_bars(code, 60)  # 30→60
```

## 4. 리스크 평가

| 리스크 | 심각도 | 내용 | 방어 |
|---|---|---|---|
| **진짜 하락 놓침** | 🟡 중간 | threshold 20% + PnL 1% 조건으로 청산이 늦어짐 | stop_loss(1.5%)가 별도로 하방 방어. 분봉 60bars가 더 정확한 하락 감지 |
| **급락 손실 확대** | 🟡 중간 | 1% 손실까지 참다가 더 큰 손실 | min_hold 20분 + stop_loss 1.5% + daily_loss 3% 3중 방어 |
| **수익 청산 지연** | 🟢 낮음 | 1% 이상 수익에서만 청산 → 더 큰 수익 가능성 ↑ | 오히려 수익률에 긍정적 |
