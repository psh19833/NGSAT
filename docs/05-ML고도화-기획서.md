## 1. 현수준 분석

### 1.1 사용 모델 (3종)

| 모델 | 라이브러리 | 용도 |
|------|-----------|------|
| RandomForest | sklearn | 주력 모델 |
| HistGradientBoosting | sklearn | 부스팅 계열 |
| LogisticRegression | sklearn | 빠른 베이스라인 |

- XGBoost / LightGBM: 코드에 업그레이드 경로 주석만 존재, pip 미설치
- 분봉 ML 모델(`ml/features/minute_builder.py`) 구조는 있으나 학습 파이프 미가동

### 1.2 피처 (20종)

```
rsi_14, macd_line, macd_signal, macd_histogram,
ma5/20/60_distance_pct, bollinger_position, bollinger_width,
atr_pct, volume_ratio_20, stoch_k, stoch_d,
price_change_1d/5d/10d/20d, volatility_20d,
obv_momentum, zscore_20, higher_high, lower_low,
gap_pct, intraday_volatility, institution_net_buy
```

- 모두 기술적 지표 기반 (가격·거래량·변동성)
- 재무 데이터, 시장 데이터, 업종 정보 미사용

### 1.3 학습 파이프라인

| 항목 | 내용 |
|------|------|
| 타겟 정의 | N일 뒤 +threshold% 상승 여부 (이진 분류) |
| forward_days | 스윙 3일, 단타 60분 (설정 가능) |
| forward_threshold | 2.0% |
| 교차검증 | TimeSeriesSplit (시계열 순서 보존) |
| 특성 중요도 | permutation_importance |
| 모델 저장 | joblib (.pkl) → `models/trained/` |

### 1.4 추론 파이프라인

- `MLInference`: 일봉 모델(스윙) + 분봉 모델(단타) 지원
- 진입 예측(`predict_entry`) + 청산 예측(`predict_exit`) 분리
- 모드별 라우팅: 스윙=일봉, 단타=분봉 (orchestrator에서 분기)
- buy_threshold(0.55~0.75), sell_threshold(0.30~0.40) 로 진입·청산 결정

### 1.5 현 성능 (KIS 실데이터 945샘플 기준)

| 지표 | 값 |
|------|-----|
| 정확도 | 60.8% |
| 정밀도 | 47.6% |
| 재현율 | 42.3% |
| F1 | 44.8% |
| AUC | 66.5% |
| 양성비율 | 37.8% |

- 클래스 불균형 존재 (양성 37.8%)
- 하이퍼파라미터: 전부 sklearn 기본값 (튜닝 0회)
- AUC 66.5% → 동전 던지기(50%)보다는 낫지만 개선 여지 큼

### 1.6 한계 요약

1. 하이퍼파라미터 튜닝 없음 — 기본값만 사용
2. 재무 데이터(실적·밸류에이션) 미사용
3. 단타 ML 모델 학습 파이프라인 미완성
4. 정기적 재학습 자동화 없음 (수동)
5. 모델 간 비교 실험 기능 없음

## 2. 오픈소스 벤치마킹

6개 주요 오픈소스 퀀트/ML 트레이딩 프로젝트 분석 결과.

| 프로젝트 | 스타 | 유형 | NGSAT 적용 포인트 |
|---------|------|------|-------------------|
| **freqtrade** | 39.9k | 암호화폐 ML 봇 | ★★★ FreqAI 자동재학습 파이프라인 + Hyperopt 파라미터 최적화 |
| **FinRL** | 10k | 강화학습 트레이딩 | ★★ RL 주문실행 최적화 (분할매수·매도 타이밍) |
| **ml4trading** | 17.8k | ML 교과서 (150개 노트북) | ★★★ 피처 엔지니어링·특성선택·모델평가 기법 |
| **qlib** | 16k | AI 퀀트 플랫폼 | ★★ XGBoost/LightGBM/LSTM 등 18종 모델 벤치마크 |
| **vnpy** | 27k | 중국 퀀트 플랫폼 | ★ 멀티전략 동시운영 아키텍처 |
| **TensorTrade** | 5.2k | RL 거래 에이전트 | ★ RL 평가 지표 (Sharpe·MDD 외) |

**핵심 인사이트:**
- freqtrade의 FreqAI: "매일 새로운 데이터로 자동 재학습 → 더 나은 모델이면 교체" 구조가 NGSAT에 가장 직접 적용 가능
- ml4trading: 피처 전처리 파이프라인 150개 노트북에서 바로 실험 → 좋은 것만 NGSAT에 도입
- FinRL: RL을 통한 주문 실행 최적화는 중장기 과제

## 3. 고도화 방안 (3단계)

### 1단계: 모델·튜닝 즉시 개선 (3시간)

#### ① XGBoost + LightGBM 모델 추가
```
pip install xgboost lightgbm
→ trainer.py에 model_type="xgboost", "lightgbm" 추가
→ 기존 코드 그대로 사용 (scikit-learn API 호환)
```
- RF 대비 AUC 5~15% 향상 기대
- qlib·ml4trading에서 기본 벤치마크로 사용

#### ② Optuna 하이퍼파라미터 튜닝
```
pip install optuna
→ trainer.py에 auto_tune() 메서드 추가
→ n_trials=50, timeout=300초
```
- 튜닝 대상: n_estimators, max_depth, learning_rate 등
- 전략설정 UI에 "튜닝 강도" 추가 가능

#### ③ FreqAI 방식 자동 재학습 (freqtrade 참고)
- 매일 장 마감 후 새 KIS 데이터로 재학습
- 기존 모델 vs 신규 모델 백테스트 성능 비교
- 더 나은 모델 자동 선택 → 교체
- 재학습 스케줄: 매일 16시 (장 마감 직후)

**1단계 기대효과: AUC 66% → 72~78%**

### 2단계: 피처·데이터 확장 (1~2주)

#### ④ 재무 데이터 피처 추가
- PER, PBR, ROE, 영업이익률, 부채비율
- KIS API 또는 별도 수집 스크립트

#### ⑤ 시장 데이터 피처 추가
- 업종 지수, 외국인/기관 순매수, 프로그램 매매
- KIS 투자자별 매매동향 API 활용

#### ⑥ 피처 자동 선별
- 20종 → 35~50종 확장
- permutation_importance 하위 30% 자동 제거
- 다중공선성(VIF) 검사

**2단계 기대효과: AUC 78% → 82~85%**

### 3단계: 학습 파이프라인 자동화 (1~2주)

#### ⑦ 단타 ML 완성
- `train_minute_model()` → 실제 KIS 분봉 데이터 학습
- 분봉 피처 20~30종 + 60분 forward target
- 스윙·단타 모델 완전 독립

#### ⑧ 앙상블 (Voting/Stacking)
- RF + XGBoost + LightGBM 소프트 보팅
- 메타 모델(LogisticRegression)로 스태킹

#### ⑨ RL 주문 실행 최적화 (FinRL 참고)
- 대량 주문 분할 실행 타이밍 최적화
- PPO/SAC 알고리즘 적용

**3단계 기대효과: 안정적 AUC 80%+ 유지, 주문 실행 비용 절감**

## 5. 1단계 실행 계획 (오늘 진행)

| 순서 | 작업 | 소요 시간 | 검증 방법 |
|------|------|---------|----------|
| 1 | pip install xgboost lightgbm optuna | 5분 | import 확인 |
| 2 | trainer.py: model_type "xgboost", "lightgbm" 추가 | 15분 | train_test 통과 |
| 3 | trainer.py: auto_tune() Optuna 메서드 추가 | 30분 | 튜닝 실행 + 로그 |
| 4 | trainer.py: auto_retrain() FreqAI 방식 추가 | 30분 | 재학습 실행 |
| 5 | StrategyConfig: ml_model_type, ml_auto_retrain 추가 | 10분 | load_config 확인 |
| 6 | StrategyConfigPanel: 모델 선택 + 자동재학습 UI 추가 | 20분 | UI 렌더링 확인 |
| 7 | KIS 실데이터로 모델 재학습 + 비교 | 20분 | AUC before/after |
| 8 | pytest 전체 회귀 + 커밋 | 30분 | 308 tests 통과 |

**총 예상 시간: 약 3시간**
