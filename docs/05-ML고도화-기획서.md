## 1. 현수준 분석

### 1.1 사용 모델 (3종)

| 모델 | 라이브러리 | 용도 |
|------|-----------|------|
| RandomForest | sklearn | 주력 모델 |
| HistGradientBoosting | sklearn | 부스팅 계열 |
| LogisticRegression | sklearn | 빠른 베이스라인 |

- XGBoost / LightGBM: 코드에 업그레이드 경로 주석만 존재, pip 미설치
- 분봉 ML 모델(`ml/features/minute_builder.py`) 구조는 있으나 학습 파이프 미가동

### 1.2 피처 (20종)

```
rsi_14, macd_line, macd_signal, macd_histogram,
ma5/20/60_distance_pct, bollinger_position, bollinger_width,
atr_pct, volume_ratio_20, stoch_k, stoch_d,
price_change_1d/5d/10d/20d, volatility_20d,
obv_momentum, zscore_20, higher_high, lower_low,
gap_pct, intraday_volatility, institution_net_buy
```

- 모두 기술적 지표 기반 (가격·거래량·변동성)
- 재무 데이터, 시장 데이터, 업종 정보 미사용

### 1.3 학습 파이프라인

| 항목 | 내용 |
|------|------|
| 타겟 정의 | N일 뒤 +threshold% 상승 여부 (이진 분류) |
| forward_days | 스윙 3일, 단타 60분 (설정 가능) |
| forward_threshold | 2.0% |
| 교차검증 | TimeSeriesSplit (시계열 순서 보존) |
| 특성 중요도 | permutation_importance |
| 모델 저장 | joblib (.pkl) → `models/trained/` |

### 1.4 추론 파이프라인

- `MLInference`: 일봉 모델(스윙) + 분봉 모델(단타) 지원
- 진입 예측(`predict_entry`) + 청산 예측(`predict_exit`) 분리
- 모드별 라우팅: 스윙=일봉, 단타=분봉 (orchestrator에서 분기)
- buy_threshold(0.55~0.75), sell_threshold(0.30~0.40) 로 진입·청산 결정

### 1.5 현 성능 (KIS 실데이터 945샘플 기준)

| 지표 | 값 |
|------|-----|
| 정확도 | 60.8% |
| 정밀도 | 47.6% |
| 재현율 | 42.3% |
| F1 | 44.8% |
| AUC | 66.5% |
| 양성비율 | 37.8% |

- 클래스 불균형 존재 (양성 37.8%)
- 하이퍼파라미터: 전부 sklearn 기본값 (튜닝 0회)
- AUC 66.5% → 동전 던지기(50%)보다는 낫지만 개선 여지 큼

### 1.6 한계 요약

1. 하이퍼파라미터 튜닝 없음 — 기본값만 사용
2. 재무 데이터(실적·밸류에이션) 미사용
3. 단타 ML 모델 학습 파이프라인 미완성
4. 정기적 재학습 자동화 없음 (수동)
5. 모델 간 비교 실험 기능 없음

## 2. 고도화 방안

### 1단계: 즉시 개선 (예상 2~3시간)

#### ① XGBoost 모델 추가

```
pip install xgboost
→ trainer.py에 model_type="xgboost" 추가
→ 기존 코드 구조 그대로 활용 (scikit-learn API 호환)
```

- 일반적으로 RF 대비 5~15% AUC 향상
- qlib에서도 기본 벤치마크 모델로 사용

#### ② LightGBM 모델 추가

```
pip install lightgbm
→ trainer.py에 model_type="lightgbm" 추가
```

- XGBoost보다 빠른 학습 속도
- 범주형 변수 처리에 강점

#### ③ Optuna 하이퍼파라미터 튜닝

```
pip install optuna
→ trainer.py에 auto_tune() 메서드 추가
→ n_trials=50, timeout=300초
```

- 튜닝 대상: n_estimators, max_depth, learning_rate, min_samples_split 등
- 전략설정 UI에 "튜닝 강도" (light/heavy) 추가 가능

**1단계 기대효과: AUC 66% → 72~78%**

### 2단계: 피처·데이터 확장 (예상 1~2주)

#### ④ 재무 데이터 피처 추가

- PER, PBR, ROE, 영업이익률, 부채비율
- KIS API `get_stock_info` 또는 별도 수집 스크립트

#### ⑤ 시장 데이터 피처 추가

- 업종 지수, 외국인/기관 순매수, 프로그램 매매
- KIS 투자자별 매매동향 API 활용

#### ⑥ 피처 자동 선별

- 현재 20종 → 35~50종으로 확장 후
- permutation_importance 하위 30% 자동 제거
- 다중공선성(VIF) 검사

**2단계 기대효과: AUC 75% → 78~85%**

### 3단계: 학습 파이프라인 자동화 (예상 1~2주)

#### ⑦ 단타 ML 완성

- `train_minute_model()` → 실제 KIS 분봉 데이터 학습
- 분봉 피처 20~30종 + 60분 forward target
- 스윙·단타 모델 완전 독립

#### ⑧ 앙상블 (Voting/Stacking)

- RF + XGBoost + LightGBM 소프트 보팅
- 메타 모델(LogisticRegression)로 스태킹
- 단일 모델 대비 안정성 ↑, 과적합 ↓

#### ⑨ 정기 재학습 자동화

- 매주 주말 KIS 데이터로 자동 재학습
- 기존 모델 vs 신규 모델 백테스트 성능 비교
- 더 나은 모델을 자동 선택 → 교체

**3단계 기대효과: 안정적인 AUC 80%+ 유지**

## 3. 현수준 vs 고도화 비교

| 항목 | 현재 | 1단계 후 | 2~3단계 후 |
|------|------|---------|-----------|
| 모델 수 | 3종 | 5종 | 5종 + 앙상블 |
| 피처 수 | 20종 | 20종 | 35~50종 |
| 하이퍼파라미터 | 기본값 | Optuna 자동 | Optuna + 스케줄 |
| AUC (추정) | 66.5% | 72~78% | 78~85% |
| 단타 ML | 미완성 | 미완성 | 완성 |
| 재학습 | 수동 | 수동 | 자동 (주1회) |
| 데이터 범위 | 기술적 지표 | 기술적 | 기술+재무+시장 |
| 예상 소요 시간 | — | 2~3시간 | 1~2주 |

## 4. 추천 진행 순서

1. **XGBoost + LightGBM 설치 및 모델 추가**
2. **Optuna 하이퍼파라미터 튜닝**
3. **백테스트로 개선 효과 검증** (before/after 비교)
4. 이후: 피처 확장 → 단타 ML → 앙상블 → 자동화

1단계만 적용해도 AUC 66% → 75% 내외로 개선될 것으로 예상됩니다.
