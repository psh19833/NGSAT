## 2. ATR 포지션 사이징 구조 분석

### 현재 공식 (하드코딩, entry_planner.py L360-366)

```python
target_vol_pct = 1.5               # ← 목표 포트폴리오 변동성 기여도 (%)
min_pct = base_pct * 0.3          # 기본의 30% = 4.5%
max_pct = base_pct * 2.0          # 기본의 200% = 30%
vol_pct = max(ATR, 0.5)           # 현재 변동성 (최소 0.5% 클램프)
adjusted_pct = base_pct × (target_vol_pct / vol_pct)   # 변동성 비례 축소
adjusted_pct = clamp(min_pct, adjusted_pct, max_pct)
```

### 의미

> **"이 종목의 ATR 변동성(5.3%)이 높으니, 포트폴리오 변동성 기여도를 1.5%로 제한하려면 기본 포지션(15%)의 28.3%만 배정하라"**

즉, `ratio = target_vol_pct / vol_pct`이 핵심이며:
- ATR이 target_vol_pct와 같으면 → adjusted_pct = base_pct (100% 배정)
- ATR이 target_vol_pct보다 높으면 → adjusted_pct < base_pct (축소)
- ATR이 target_vol_pct보다 낮으면 → adjusted_pct > base_pct (확대, max_pct 제한)

### 문제점

| 항목 | 현재값 | 문제 |
|:-----|:------:|:-----|
| `target_vol_pct` | **1.5** | 극도로 보수적. ATR=5.3%일 때 ratio=0.283 → 71.7%나 깎음 |
| `min_pct` | **4.5%** | 4.5% × 209만원 = 9.4만원 → 고가주(17만원↑) 1주도 못 삼 |
| P0 로깅 | **없음** | quantity=0 → silent return, 디버깅 불가 |
| log_file | **미생성** | `logs/ngsat.log` 파일 없음 → 전체 로그 추적 불가 |

## 4. 권장 수정안

### ✅ 권장: 시나리오 ② — `target_vol_pct` = 1.5 → **8.0** (1줄 수정)

**코드 변경:** `entry_planner.py L361` 단 1줄

```python

# 변경 후

```

**base_budget_pct(15%)는 DB config로 이미 동적 관리되므로 수정 불필요.**

### 왜 ②인가?

| 기준 | ② target=8.0 | ⑥ base=25% | ④ target=5.0 |
|:-----|:----------:|:---------:|:----------:|
| 수정량 | **1줄** | DB + 코드 2곳 | 1줄 |
| 사용자 의견(50만원) 부합도 | **🟢 47만원** | 🟢 49만원 | 🟡 30만원 |
| ATR=2.0% 안전성 | 🟢 30% clamp | 🟡 50% (과함) | 🟢 30% clamp |
| ATR=8.0% 최소 보장 | 🟢 KB 1주 | 🟢 KB 1주 | 🟢 KB 1주 |
| 포트폴리오 분산도 | 🟢 4종목 (적절) | 🟢 4종목 | 🟢 7종목 |
| base_pct 영향 범위 | **없음** (15% 유지) | 모든 모드 영향 | 없음 |

### ATR 구간별 동작 (② target=8.0)

```
ATR 1~4% → adj=30% (max_pct clamp) → 63만원 → KB 3주
ATR 5%   → adj=24%                   → 50만원 → KB 2주  ← 사용자 의견
ATR 5.3% → adj=22.6%                 → 47만원 → KB 2주  ← 현재 시장
ATR 6%   → adj=20%                   → 42만원 → KB 2주
ATR 7%   → adj=17.1%                 → 36만원 → KB 2주
ATR 8~10% → adj=15~12%              → 31~25만원 → KB 1주
```

**모든 ATR 구간(1~10%)에서 KB 1주 이상 확보. 사용자 의견 50만원에 근접.**

### 정합성 검증: `max_total_exposure_pct = 100%`와의 관계

```
adjusted_pct = 22.6% (ATR=5.3% 기준)
→ 실질 최대 보유 종목 = 100% / 22.6% ≈ 4개
→ max_holdings = 10 이내, 분산 투자 가능
→ 각 종목 예산 대비 KB 1주 비중 = 37% (적절)
```
