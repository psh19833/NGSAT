# 스크리닝 고도화 리서치 보고서

**목적**: 현재 5개 패턴(돌파/눌림/반등/밴드수축/MA교차) + 5개 지표(RSI/MACD/Stochastic/MA정렬/거래량) 기반 스크리닝을 고도화

---

## 1. 현재 상태 분석

### 1.1 사용 중인 지표와 패턴

| 구분 | 항목 | 특징 |
|:-----|:------|:------|
| 지표 | RSI(14) | 과매수/과매도 구간 ±5~15점 |
| 지표 | MACD | 히스토그램 부호만 확인 (±5점) |
| 지표 | Stochastic(14,3) | K/D 20/80 기준 (±10점) |
| 지표 | MA 정렬(5/20/60) | 골든크로스/데드크로스 (±5~20점) |
| 지표 | 거래량 추세 | 5일/20일 MA 비교 (±4~8점) |
| 패턴 | Breakout | 20일 최고가 돌파 |
| 패턴 | Pullback | 상승 후 눌림 |
| 패턴 | Rebound | 하락 후 반등 |
| 패턴 | Bollinger Squeeze | 밴드 수축 후 확장 |
| 패턴 | MA Cross | 5일/20일 교차 |

### 1.2 문제점

| 문제 | 설명 |
|:-----|:------|
| **점수 체계 단순** | 전부 가산점 방식(RSI +15, MACD +5, MA +20...). 지표 간 가중치가 고정되어 있고, 시장 상태에 따라 달라지지 않음 |
| **패턴 중복** | 9개 중 8개가 골든크로스 패턴 — MA 정렬 점수와 사실상 동일한 신호를 중복 계상 |
| **변동성 미반영** | ATR 값을 스크리닝에 사용하지 않음 (ML 학습에는 사용하나 스크리너 점수에는 미반영) |
| **수급 데이터 미활용** | 외국인/기관 순매수, PER/PBR/EPS는 ML 피처에만 있고 스크리너 점수에 미반영 |
| **레짐별 차등 부족** | 강세/중립/약세별 min_score와 max_candidates만 다르고, 지표별 가중치는 동일 |

## 2. 개선 방안

### 2.1 고급 기술 지표 추가

#### ① MFI (Money Flow Index) — ★★★★★

| 항목 | 내용 |
|:-----|:------|
| 설명 | RSI에 거래량을 결합한 지표. 0~100 범위. 80 초과 = 과매수, 20 미만 = 과매도 |
| 현재 RSI가 가격만 보는데, MFI는 **자금 흐름**까지 평가 |
| 구현 | `strategy/indicators.py`에 `mfi(highs, lows, closes, volumes, period=14)` 함수 추가 |
| 스크리너 반영 | MFI < 20 → oversold +10점, MFI > 80 → -5점 (RSI와 차별화) |
| 효과 | 거래량 동반 매집/분산 확인 — 가짜 신호 필터링 |

#### ② OBV (On-Balance Volume) 추세 — ★★★★★

| 항목 | 내용 |
|:-----|:------|
| 설명 | 거래량 누적 기반 모멘텀 지표. OBV 상승 = 매집, 하락 = 분산 |
| OBV의 20일 기울기(regression slope)로 자금 흐름 방향 확인 |
| 구현 | `obv(closes, volumes)` → 20일 선형회귀 기울기 |
| 스크리너 반영 | OBV 기울기 > 0 → +5~10점 (매집 확인), < 0 → -5점 |
| 효과 | 기관/외국인 수급을 간접적으로 추정 가능 (일봉만으로) |

#### ③ ADX + 방향성 지표(DI+/DI-) — ★★★★

| 항목 | 내용 |
|:-----|:------|
| 설명 | 현재는 ADX만 사용. DI+와 DI- 선을 추가하여 **추세 방향과 강도를 동시 평가** |
| DI+ > DI- → 상승 추세 (추가 점수), DI- > DI+ → 하락 추세 |
| 구현 | `adx(highs, lows, closes, period=14)`가 이미 DI+/- 반환하도록 확장 |
| 스크리너 반영 | DI+ > DI- && ADX > 25 → 강한 상승추세 +15점, ADX < 20 → 추세 없음 (관망) |
| 효과 | 골든크로스와 중복되는 패턴을 ADX 강도로 필터링 |

#### ④ 일봉 캔들스틱 패턴 — ★★★★

| 항목 | 내용 |
|:-----|:------|
| 설명 | 일봉 캔들 패턴 인식 — 망치형, 장악형, 별, 관통형 등 |
| (단타의 경우) 분봉 캔들 패턴도 가능 |
| 구현 | `hammer(open, high, low, close)`, `engulfing(prev_open, prev_close, open, close)` 등 |
| 스크리너 반영 | 강세 반전 패턴 → +10점, 약세 반전 패턴 → -5점 |
| 효과 | 9개 중 8개가 골든크로스인 현재 패턴의 다양성 확보 |

#### ⑤ 변동성 기반 스크리닝 — ★★★★

| 항목 | 내용 |
|:-----|:------|
| 설명 | ATR을 스크리너에 직접 반영. 변동성 급증 = 이벤트 발생 |
| ATR 비율(ATR/close) + ATR 5일 변화율 |
| 스크리너 반영 | ATR 급증(>20%) + 방향성 → 추세 추종 기회, ATR 축소 → 횡보 제외 |
| 효과 | 변동성 없는 종목을 자연스럽게 필터링 |

#### ⑥ 상대강도 (RS — Relative Strength) — ★★★

| 항목 | 내용 |
|:-----|:------|
| 설명 | 종목 수익률 / KOSPI 수익률 비율. 1.0보다 크면 시장보다 강함. |
| 20일 RS = stock_return_20d / index_return_20d |
| 스크리너 반영 | RS > 1.2 → +10점 (시장 대비 outperformance), RS < 0.8 → -10점 |
| 효과 | 하락장에서도 강한 종목 선별 가능 (KOSPI 하락 -3.2%인 장에 유효) |

### 2.2 점수 체계 고도화

#### 현재 (단순 가산점)

```
score = 50 + RSI + MACD + MA + Volume + Stochastic + Pattern
```

#### 제안 (가중치 + 정규화)

```
base = 50
indicator_score = (
    RSI * w_rsi +
    MFI * w_mfi +
    ADX * w_adx +
    OBV * w_obv +
    MA_Align * w_ma +
    Volume * w_vol +
    Pattern * w_pattern +
    RS * w_rs +
    Candle * w_candle
) / sum(weights)

score = base + indicator_score * regime_factor
```

| 항목 | 현재 | 제안 |
|:-----|:-----|:------|
| 지표 수 | 5개 | **10개** |
| 패턴 수 | 5개 | **10개+** (캔들 패턴 포함) |
| 가중치 | 고정 | **레짐별 동적** |
| 변동성 | 미반영 | **ATR로 필터링** |
| 수급 | ML 전용 | **스크리너에도 반영** |
| 시장상대비교 | 없음 | **RS(Relative Strength) 추가** |

### 2.4 필터링 강화 (점수 전 단계)

| 필터 | 설명 | 효과 |
|:-----|:------|:------|
| 변동성 최소값 | ATR < 0.5% → 제외 | 횡보장 종목 제거 |
| 거래량 최소값 | 20일 평균 거래량 < 기준 → 제외 | 유동성 확보 |
| MFI 확인 | MFI > 80에서 추가 매수 금지 | 추세 고점 추격 방지 |
| RS 필터 | RS < 0.7 → 제외 | 시장 대비 약세 종목 제외 |
