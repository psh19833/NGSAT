```

                                             │

                                             │
```

| 파일 | 변경 | 내용 |

### 상세 설계

```python
# websocket_client.py — on_price에 3줄만 추가
def on_price(self, code, price, volume, ts):
    # 기존: 가격 캐시 업데이트
    self._price_cache[code] = price
    
    # 신규: MinuteBarBuilder에 tick 전달 (+3줄)
    if self._minute_builder:
        self._minute_builder.feed(
            code, price, price, price, price, volume, ts
        )
```

WebSocket tick은 단일 가격(`price`)만 전달하므로,
- open = high = low = close = 현재가 (단일 tick 기준)
- MinuteBarBuilder._flush()에서 같은 minute 내 여러 tick의 OHLCV를 올바르게 집계

## Phase 2: 일봉 + 분봉 통합 점수 체계

**목표:** 현재 병렬 운영 중인 일봉 점수 + 분봉 점수를 통합 Combined Score 체계로 전환
**근거:** `docs/60-스크리닝-고도화-구현-계획서-v2.md` (scorer.py 분리), `docs/38-스크리너-개선-기획서.md`

### 현재 상태

```
일봉 스크리너 → ScreenCandidate.score (71.6, 67.8, 64.5...)
                                                   (별도, 미통합)
분봉 스크리너 → MinuteScore.score (90, 83, 83...)
```

두 점수가 대시보드에 각각 표시되지만, **매수 판단은 일봉 기준 + ML**만 사용 중.

### 변경 설계

```python
# strategy/combined_scorer.py (신규, ~50줄)
def compute_combined_score(
    daily_score: float,        # 0~100 (일봉)
    minute_score: float,       # 0~100 (분봉)
    regime: MarketRegime,      # 레짐별 가중치
    minute_confidence: float,  # 0~1 (분봉 데이터 충분도)
) -> float:
    """일봉 + 분봉 통합 점수.
    
    레짐별 기본 가중치:
      bull:    일봉 70% + 분봉 30%
      neutral: 일봉 50% + 분봉 50%  (중립장 = 단타 = 분봉 중요)
      bear:    일봉 80% + 분봉 20%  (약세장 = 보수적 = 일봉 우선)
    
    minute_confidence < 0.5 → 분봉 가중치 50% 감소 (데이터 부족 시 안전장치)
    """
    w_daily, w_minute = _regime_weights(regime)
    w_minute *= minute_confidence
    total = (daily_score * w_daily + minute_score * w_minute) / (w_daily + w_minute)
    return round(total, 1)
```

### 예상 효과

| 항목 | 현재 | 통합 후 |
|:-----|:----:|:--------|
| 점수 변화 주기 | 1일(일봉) / 10분(분봉) | **1분(combined)** |
| 레짐별 대응 | 없음 | bull/neutral/bear 별 가중치 |
| 장중 변별력 | 낮음 (점수 고정) | **높음 (분봉 변동 반영)** |
| 데이터 부족 안전장치 | 없음 | **confidence 기반 fallback** |

### 리스크

| 항목 | 등급 | 내용 |
|:-----|:----:|:------|
| 통합 점수로 매수판단 변경 | 🟡 LOW | combined_score는 참고용, 실제 매수는 ML 예측 기반 유지 |
| 분봉 confidence가 항상 낮음 | 🟢 LOW | 장 초반만 해당, 10분 후 정상화 |

**공수:** ~1시간
