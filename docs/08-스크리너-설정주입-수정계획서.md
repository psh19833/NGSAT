# 실거래 스크리너 설정 주입 누락 — 수정 계획서

**작성일**: 2026-06-28  
**상태**: 검토 대기  
**우선순위**: P2 (Phase 2 잔여)

---

## 1. 문제 요약

실거래 경로(`main.py` → `run_live()`)에서 스크리너, 모드선택기, 레짐 평가기의 설정 초기화 함수가 호출되지 않아, 사용자가 `.env`나 대시보드에서 변경한 전략 설정값이 실거래 스크리닝/모드선택/레짐평가에 반영되지 않음.

---

## 2. 검증 결과

### 2.1 누락 확인 — `run_live()`에 init 함수 없음 ✅ (재현)

| 모듈 | run_backtest() | run_live() | 영향 |
|------|---------------|-----------|------|
| `init_screener_config()` | ✅ line 111 | ❌ 없음 | 스크리너 임계값 미반영 |
| `init_mode_selector_config()` | ✅ line 112 | ❌ 없음 | 변동성 ATR 임계값 미반영 |
| `init_regime_config()` | ✅ line 110 | ❌ 없음 | **영향 없음** (하드코딩) |

### 2.2 실제 영향 범위 (정정 필요)

**스크리너** — 실질적 영향 O (`screen_stocks()`가 `_get_screener_config()` 호출):
- `screener_bull_min_score` (기본 60 → 변경 시 미반영)
- `screener_bull_max_candidates` (기본 15)
- `screener_neutral_min_score` (기본 70)
- `screener_neutral_max_candidates` (기본 10)
- `screener_bear_min_score` (기본 80)
- `screener_bear_max_candidates` (기본 5)

**모드선택기** — 실질적 영향 O (`select_mode()`가 `_get_mode_config()` 호출):
- `mode_high_volatility_atr_pct` (기본 1.5)
- `mode_low_volatility_atr_pct` (기본 0.5)

**레짐 평가** — 영향 없음 (`evaluate_regime()`이 하드코딩 상수 사용):
- 가중치 `_WEIGHT_MA_ALIGNMENT=35` 등은 config 미연동
- `init_regime_config()` 호출은 무의미 (단, 코드 일관성을 위해 추가 가능)

### 2.3 KOSPI 가산점 버그

`data/real_data_provider.py:90` — **모든 종목에 `Market.KOSPI` 하드코딩**.
→ 스크리너의 `KOSPI 가산점 +5점`이 모든 종목에 무조건 적용됨.
→ 코스닥 종목을 유니버스에 추가해도 KOSPI로 잘못 분류.

**리스크**: LOW (현재 유니버스는 전부 KOSPI 종목이므로 실제 영향 없음).

### 2.4 데이터 최소 요건 불일치 (30 vs 60)

- 스크리너: `len(price_history) < 30` → 제외
- 오케스트레이터 ML 단계: `len(prices) < 60` → 제외

**영향**: 30~59일 데이터 종목은 스크리닝 통과 후 ML에서 필터링 (불필요 연산).

**리스크**: LOW (크래시 없음, 성능 저하 미미 — 유니버스 30종목 기준).

## 3. 리스크 분석 (SWOT)

| 강점 | 약점 |
|------|------|
| 설정 누락이어도 기본값으로 동작 (시스템 정지 없음) | 사용자가 .env 임계값 변경해도 효과 없음 |
| 수정 범위 최소 (main.py 2~3줄 + init 함수 파라미터 추가) | 모듈 간 설정 주입 방식이 일관되지 않음 (init 함수 + 모듈 변수) |
| 모든 배치 수정은 단위 테스트 보호됨 | KOSPI 가산점 분류는 추가 수정 필요 |

| 기회 | 위협 |
|------|------|
| 2차 리뷰 "2-2 글로벌 config DI"의 전초 작업 | 근본 DI 리팩토링까지 미루면 누락 상태 장기화 |
| screener 성능 최적화(30→60) 병행 가능 | init_regime_config() 무효 상태로 혼란 초래 가능 |

## 4. 권장 작업 목록

### P-a: run_live()에 init 함수 2줄 추가 (리스크 LOW)

**대상**: `main.py:run_live()`

**내용**: 오케스트레이터 생성 직후에 추가:
```python
from strategy.screener import init_screener_config
from strategy.mode_selector import init_mode_selector_config
init_screener_config(config.strategy)
init_mode_selector_config(config.strategy)
```

`init_regime_config()`는 `evaluate_regime()`이 config를 사용하지 않으므로 불필요하나, 코드 일관성을 위해 추가 가능 (무해).

**영향**: 기존 동작 유지. 기존 .env 값이 기본값과 동일하면 변화 없음.

### P-b: KOSPI 가산점 분류 정상화 (리스크 LOW)

**대상**: `data/real_data_provider.py`

**내용**: KOSPI/KOSDAQ 분류 로직 추가 (`_code_to_name`과 병행)

**영향**: 현재 유니버스가 전부 KOSPI이므로 변화 없음. 코스닥 추가 시 정상 작동.

### P-c: 데이터 최소 요건 일관화 (30→60) (리스크 LOW)

**대상**: `strategy/screener.py:148`

**내용**: 
```python
if len(price_history) < 60:  # before: 30
    continue
```

**영향**: 30~59일 데이터 보유 종목이 스크리닝 단계에서 미리 필터링 → 불필요 연산 감소.

### P-d (보류): 레짐 평가 config 연동

**대상**: `strategy/regime.py`

**내용**: `_WEIGHT_MA_ALIGNMENT`, `_WEIGHT_RSI` 등 가중치를 StrategyConfig에서 읽도록 변경.

**리스크**: MEDIUM (레짐 평가 결과가 바뀌면 매매 패턴 전체 영향).

**권장**: 현재는 보류. 2-2 글로벌 config DI 리팩토링에서 범위에 포함.

### P-e (보류): 모듈 변수 → DI 리팩토링

**대상**: `strategy/screener.py`, `strategy/regime.py`, `strategy/mode_selector.py`

**내용**: 전역 모듈 변수 + init 함수 패턴 제거. 오케스트레이터가 config를 직접 전달하도록 개선.

**리스크**: MEDIUM-HIGH (인터페이스 변경, screen_stocks() 시그니처 수정 필요).

**권장**: 보류 — 별도 Phase로 분리.
