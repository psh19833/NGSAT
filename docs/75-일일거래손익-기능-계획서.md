# 일일 거래 손익(P&L) 기능 구현 계획서

**목적:** 체결 기반 일일 손익을 대시보드에서 확인 가능하도록 함

---

## 1. 데이터 구조

### 입력: TradeRecord (DB)
| 필드 | 타입 | 설명 |
|:-----|:-----|:------|
| date | str | 체결일시 |
| code | str | 종목코드 |
| name | str | 종목명 |
| side | str | buy/sell |
| quantity | int | 수량 |
| price | float | 체결단가 |
| amount | float | 체결금액 |
| action | str | buy/sell/stop_loss |

### P&L 계산 로직
```
1. 동일일자 + 동일종목코드의 매수/매도 짝짓기 (선입선출)
2. 매도 체결손익 = (매도단가 - 매수단가) × 수량
3. 일자별 합산
4. 추정 제세금 = 매도금액 × 0.195% (거래세 0.18% + 수수료 0.015%)
```

## 2. Backend

### TradeRepository.get_daily_pnl() — 신규 메서드
- DB 전체 TradeRecord 조회
- 일자별 + 종목별 매수/매도 매칭
- 일별 realized_pnl, fee_estimate, net_pnl, win_rate 반환

### GET /api/pnl/daily — 신규 엔드포인트
```json
{
  "connected": true,
  "daily": [
    {
      "date": "2026-07-09",
      "trade_count": 6,
      "realized_pnl": 1100,
      "fee_estimate": -1640,
      "net_pnl": -540,
      "win_rate": 100.0,
      "trades": [ ... ]
    }
  ],
  "summary": {
    "total_pnl": -1390,
    "total_trades": 37,
    "trade_days": 7,
    "avg_daily_pnl": -199
  }
}
```

## 3. Frontend

### api.js
- `getDailyPnL: () => fetchJSON('/pnl/daily')`

### PnLCard.jsx — 신규 컴포넌트
- 일자별 P&L 테이블 (날짜, 체결건수, 실현손익, 추정세금, 순손익, 승률)
- 전체 요약 (총손익, 총거래일수, 일평균손익)
- 색상: 양수=text-ngsat-green, 음수=text-ngsat-red
- 기존 `pnlColor`, `formatWon`, `formatPercent` 유틸 재사용

### App.jsx
- pnl 탭 추가 + refreshForTab에 api.getDailyPnL 연결
- Sidebar에 탭 항목 추가 (icon: BarChart3)

## 4. 리스크 검토

| 리스크 | 영향 | 대응 |
|:-------|:-----|:------|
| TradeRecord에 수수료/세금 정보 없음 | 추정치만 표시 | 추정세금 별도 표시, 실제와 차이 있을 수 있음 |
| 동일종목 동일일자에 여러 번 매매 | 선입선출로 매칭 | FIFO 방식으로 매칭, 잔여분은 다음날로 이월 |
| 미체결 주문 | P&L에 미포함 | 체결 기준이므로 정상 |
| 서버 재시작 시 P&L 초기화 | DB 유지되므로 영향 없음 | 안전 |
