## 2. 항목별 리스크

### #1 포지션 사이징 [MEDIUM]

| 항목 | 내용 |
|------|------|
| **변경** | main.py: `position_budget_pct=config.strategy.mode_swing_position_size` 전달 |
| **추가 변경** | orchestrator.py: ATR 다이나믹 사이징의 `base_budget_pct` → `self._risk.position_size_pct` |
| **위험** | 현재 단타 모드에서도 10% 고정 사이징 → 수정 후 5%로 줄어들면 **실제 매수 수량이 절반으로 감소** |
| **영향** | 단타 모드 매수 수량 감소 — 리스크 측면에서는 **긍정적**(모드에 맞는 적절한 사이징) |
| **검증** | 단타 모드 사이징 5% × ATR 조정 → 예: 변동성 2% → adjusted = 5% × (1.5/2.0) = 3.75%. 최소 1.5% ~ 최대 10% 범위 내 |
| **롤백** | `position_budget_pct=` 라인 삭제 또는 기본값 0.10으로 복원 |

**판정**: LOW — 기능적 결함 수정이므로 리스크보다 개선 효과가 큼. 단타 모드 진입량이 줄어들어 오히려 안전.
