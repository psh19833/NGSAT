# 백테스트 대시보드 탭 — 기획서

> 작성일: 2026-06-28
> 목표: 대시보드에서 백테스트 실행·진행·결과 조회를 가능하게 함

---

## 1. 개요

**현재**: `python main.py --backtest` — 터미널에서만 실행 가능
**목표**: 대시보드 8번째 탭에서 GUI로 실행

### 사용 흐름

```
1. 전략설정 탭에서 파라미터 조정
2. 백테스트 탭으로 이동
3. 실행 버튼 클릭
4. 5~30초 후 결과 확인
5. 결과 보고 전략 설정으로 돌아가 조정 → 1번 반복
```

## 3. 백엔드 API

### POST /api/backtest/run

요청:
```json
{
  "start_date": "2026-04-01",
  "end_date": "2026-06-26",
  "initial_capital": 10000000,
  "use_ml": true
}
```

응답:
```json
{
  "connected": true,
  "status": "running",
  "job_id": "bt_20260628_001"
}
```

### GET /api/backtest/status

응답:
```json
{
  "connected": true,
  "status": "running",
  "progress_pct": 35,
  "completed_stocks": 13,
  "total_stocks": 38,
  "elapsed_sec": 8
```

## 4. 데이터 흐름

```
사용자: 실행 폼 작성 → [실행] 클릭
  → 프론트: POST /api/backtest/run
    → 백엔드: run_in_executor()로 백테스트 시작
    → 프론트: 1초마다 GET /api/backtest/status 폴링
    → 진행률 표시 (프로그레스 바)
  → 백테스트 완료
    → 프론트: GET /api/backtest/results
    → 결과 요약 + 차트 + 거래내역 표시
```

## 5. 구현 단계

### 1단계 (1일) — 실행 + 결과 요약
- 백엔드: `run_backtest()`를 공유 함수로 분리 (`core/backtest_runner.py`)
- 백엔드: POST /api/backtest/run + GET /api/backtest/status + GET /api/backtest/results
- 프론트: BacktestPanel.jsx (실행 폼 + 결과 요약)
- 프론트: Sidebar.jsx + App.jsx (탭 등록)
- 데이터 출처 표시 (KIS 실데이터 / 합성데이터)

### 2단계 (1일) — 시각화 + 거래내역
- EquityChart 재사용 (자산 추이)

- 이전 결과 목록
