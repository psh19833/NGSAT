# 04 · 하이브리드 매매 타이밍 설계

> 매매 타이밍 방식 확정 및 단계적 재개발 로드맵
> 작성일: 2026-06-25 · 상태: 1단계 착수 승인 대기

---

## 1. 결정 사항 (대표님 확정)

| 항목 | 결정 | 의미 |
|------|------|------|
| 매매 타이밍 | **하이브리드** | 일봉으로 큰 그림(시장·종목), 분봉으로 진입/청산 순간을 정밀하게 |
| 보유 성격 | **C) 혼합** | 시장 상황에 따라 단타 ↔ 스윙을 자동 전환 |
| 개발 순서 | **단계적** | 1차 '스윙+정밀진입' 완성·실거래 검증 → 2차 '단타·자동 전환' 추가 |

비유: 일봉 = 지도/항공사진으로 '어느 동네가 좋은지' 고르기(느리지만 큰 그림 정확). 분봉 = 그 동네에서 '정확히 어느 순간 문을 두드릴지' 잡기(빠르고 정밀). 하이브리드는 둘을 결합한 한국 기관·프로의 정석 멀티타임프레임 구조.

---

## 2. 하이브리드가 NGSAT 3단계 파이프라인에서 작동하는 방식

기존 파이프라인: ① 레짐(시장 분위기) → ② 스크리너(종목 선별) → ③ ML(매매 결정)

| 단계 | 데이터 | 역할 | 변경 |
|------|--------|------|------|
| ① 레짐 | 일봉(지수) | 시장 큰 그림 + **모드 전환 스위치** | 역할 확장 |
| ② 스크리너 | 일봉(종목) | 좋은 후보만 통과 | 유지 |
| ③ ML | **모드별**: 스윙=일봉 / 단타=분봉 | 진입·청산 '순간' 결정 | 이중화 |

①②는 일봉으로 질 높은 후보를 거르고, ③의 진입·청산 타이밍만 분봉으로 정밀화한다.

---

## 3. 혼합(C) — 모드 전환 설계

혼합 = '매매 방식 두 개 + 자동 전환 스위치'.

- **스윙 모드**: 좋은 종목을 며칠~몇 주 보유. 기존 일봉 ML 모델을 살려 개량.
- **단타 모드**: 분봉으로 빠르게 진입·당일 청산. 분봉 ML 모델을 신규 개발.
- **전환 스위치**: 기존 레짐 모듈(`strategy/regime.py`)의 강세/중립/약세 판정을 모드 선택기로 승격.

전환 매핑(초안 — 2단계 설계 시 변동성 지표로 정밀화):

| 레짐 | 모드 | 근거 |
|------|------|------|
| 강세장(추세) | 스윙 | 추세를 며칠 따라가는 것이 유리 |
| 중립장(횡보·변동성) | 단타 | 방향성 약함 → 짧게 먹고 빠짐 |
| 약세장 | 관망/보수적 | 신규 진입 최소화 |

> 보강 예정: '중립장 = 단타'는 1차 매핑일 뿐. 실제로는 변동성(ATR·볼린저 폭)을 직접 측정해 단타 적합 국면을 더 정밀하게 가른다. 2단계 설계에서 확정.

---

## 4. 현재 시스템 vs 목표 — 무엇을 바꿔야 하나

| 영역 | 현재 | 목표 |
|------|------|------|
| 데이터 | 일봉 차트 API만(`endpoints.py`에 분봉 없음) | 분봉 조회 엔드포인트/어댑터 추가 |
| 피처 | 일봉 20종(`ml/features/builder.py`) | 스윙=일봉 유지 + 단타=분봉 피처 신규 |
| ML 모델 | 단일 모델, "5일 뒤 2%↑" 타겟(스윙 전용) | 스윙 모델 개량 + 단타 모델(짧은 분봉 타겟) |
| 추론 | 단일 `MLInference` | 모드 분기(스윙/단타 라우팅) |
| 리스크 | 일일 -5% 중단, 종목 -3%/최대 -5% 동적손절 | 모드별 분리(단타는 타이트·당일청산 강제) |
| 백테스트 | 일봉 기준 | 분봉 처리 지원 |

## 5. 단계적 로드맵

### 1단계 — 스윙 + 정밀진입 (먼저 완성·실거래 검증)
일봉으로 종목을 고르고, 진입·청산 '순간'만 분봉으로 더 유리하게 잡는다. 보유는 며칠 스윙. 기존 일봉 자산을 그대로 살리므로 토대가 빠르게 단단해진다. 이 단계만으로도 실거래 가동 가능.

### 2단계 — 단타 모드 + 자동 전환 (토대 위에 얹기)
분봉 ML 모델과 레짐 기반 모드 전환을 추가해 혼합(C)을 완성한다. 1단계가 검증된 위에서 진행하므로 위험이 작다.

단계적 진행은 **데이터 측면에서도 유리**하다 → 7번 참고.

---

## 6. 1단계 상세 작업 분해 (착수 시 정밀화)

1. **분봉 데이터 경로 추가** — KIS 당일 분봉 엔드포인트(`inquire-time-itemchartprice`) 등록, 어댑터/매퍼/매핑 테스트.
2. **진입 정밀화 로직** — 스크리너 통과 + 스윙 ML이 '매수'로 판단한 종목에 대해, 분봉으로 유리한 진입가/타이밍을 잡는 모듈 추가.
3. **청산 정밀화 로직** — 보유 포지션의 청산 '순간'을 분봉으로 모니터링(손절·익절 트리거 정밀화).
4. **스윙 ML 모델 점검·개량** — 기존 일봉 모델/타겟 재검토, 필요 시 XGBoost로 교체.
5. **리스크 연동** — 스윙 손절/익절 규칙을 분봉 트리거와 연결.
6. **백테스트** — 일봉 후보선별 + 분봉 진입/청산을 함께 검증할 수 있도록 엔진 보강.
7. **테스트 0건 실패 확인** — 각 작업마다 단위·통합 테스트 통과.

---

## 7. KIS 분봉 데이터 제약 및 대응

- **당일 분봉**: KIS API로 실시간 조회 가능(라이브 매매에 충분).
- **과거 분봉**: 보존 기간이 짧아 장기 학습/백테스트용 히스토리 확보가 일봉보다 까다로움.
- **대응**:
  - 1단계(스윙)는 학습이 일봉 기반이라 데이터 문제 없음. 진입/청산만 당일 분봉을 실시간 사용.
  - 2단계(단타) 착수 전, 분봉 데이터 수집기를 먼저 가동해 매일 분봉을 DB에 누적 적재 → 학습 데이터를 미리 쌓는다.
  - 즉 단계적 진행이 단타 모델용 데이터를 모을 시간을 벌어준다.

---

## 8. 모드별 리스크 규칙 (초안)

| 항목 | 스윙 모드 | 단타 모드 |
|------|-----------|-----------|
| 보유 기간 | 며칠~몇 주 | 당일 청산 강제 |
| 종목 손절 | -3% 기준 동적손절(최대 -5%) | 더 타이트(예 -1.5~-2%) |
| 익절 | ML 청산 판단 | 목표 도달 시 즉시 |
| 공통 | 일일 누적 -5% 시 당일 자동 중단 | 동일 |

---

## 9. 검증 원칙

- 모든 매매 판단에 근거 명시(DecisionReason 강제) — 기존 원칙 유지.
- 실거래/백테스트 완전 분리 유지(상호 import 금지, AST 검증).
- 각 작업 단위마다 **테스트 0건 실패** 확인 후 다음 진행.
- 실거래 투입 전 read-only 기능 테스트로 판단·가드 체인만 검증(실주문 경로 미도달).
