## 2. 발견된 문제점 (7가지)

### 문제 1: 핑퐁 (buy→즉시 sell) — 가장 심각

신한지주에서 **07-10 15:02~15:06, 4분간 buy→sell 8회 반복**:
```
buy 110,200 → sell 110,300 → buy 110,300 → sell 110,400 → 
buy 110,400 → sell 110,500 → buy 110,500 → sell 110,400
```

모든 거래의 PnL이 **±0.0%** — 의미 없는 거래만 반복.  
**원인**: 분봉ML이 BUY(60%↑) → 다음 사이클에서 SELL(30%↓) 반복.  
**수정**: ✅ min_hold 20분 (stop_loss 예외 버그도 함께 수정)

### 문제 2: 중복 sell 기록 (execute_sell 실패에도 DB 기록)

SK이노베이션: 동일 가격(102,900)에 sell 2주 × **5회** 기록.  
실제로는 1회만 체결되었을 가능성 높음.  
**원인**: `execute_sell()` 실패해도 `record_sell()`이 DB에 기록.  
**수정**: ✅ record_sell을 exec_result.success 블록 안으로 이동

### 문제 3: 매매시작 버튼 무시 (서버 재시작 시 자동 매매)

`_trading_allowed = True`가 기본값.  
서버 재시작 → 사용자가 매매시작 버튼 안 눌러도 매매 시작.  
**수정**: ✅ `_trading_allowed = False`로 변경, main.py 강제 True 복원 제거

### 문제 4: min_hold의 stop_loss 예외로 핑퐁 재발생 가능

min_hold가 stop_loss 예외를 허용하여 bear 장에서 effective_stop=0.5%로 내려가면 min_hold 무력화.  
**수정**: ✅ min_hold 내 stop_loss 포함 모든 청산 금지

### 문제 5: 전체 82건 중 19건(23%)이 시뮬레이션 데이터

06-26~06-30 한국전력/HLB/실리콘투 19건은 mode='live'지만 실제 KIS 체결이 아닌 테스트 데이터.  
→ 실거래 통계 왜곡.  
**대기**: mode='backtest'로 변경 필요

### 문제 6: ML 모델 예측 정확도

매수 확률 60%↑ 26건 중 실제 수익 본 거래는 거의 없음.  
매도 사유의 94%가 ML 청산 예측(29/31). 손절(stop_loss)은 3건뿐.  
→ 분봉ML이 BUY 신호를 내지만 그 다음 사이클에서 즉시 SELL → 핑퐁.

### 문제 7: KIS 체결 vs DB 기록 불일치

오늘(07-15) 신한지주 109,100→108,100→108,200 거래는 KIS에 체결되었으나 DB 미기록.  
서버 재시작으로 pending_buy_trades 버퍼 소멸.  
**대기**: KIS 체결내역 동기화 로직 필요
