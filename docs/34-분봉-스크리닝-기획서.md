## 1. 배경

### 1.1 현재 문제

현재 스크리너는 **일봉(일별 종가)** 데이터만 사용합니다.

| 항목 | 현재 | 문제 |
|------|------|------|
| 데이터 | 일봉 (1일 1회) | 장중 절대 변경 안 됨 |
| 점수 | MA, RSI, MACD, 패턴 | 동일 종목 동일 점수 반복 |
| ML 예측 | 일봉 피처 27개 | 확률 변화 0.1% 미만 |
| **실시간성** | **없음** | **장중 변화 0** |

### 1.2 기존 분봉 인프라

분봉 기반 코드는 일부 존재합니다:

| 모듈 | 용도 | 상태 |
|------|------|:----:|
| `entry_timing.py` | 분봉 RSI/급등 체크 → 진입 판단 | ✅ 운영 중 |
| `exit_timing.py` | 분봉 청산 신호 감지 | ✅ 운영 중 |
| `ml/inference.py` | 분봉 ML 단타 예측 (predict_minute_entry) | ✅ 운영 중 |
| `ml/features/minute_builder.py` | 분봉 ML 피처 30+종 | ✅ 운영 중 |
| **스크리너** | **분봉 기반 미구현** | **❌ 없음** |

## 2. 상세 설계

### 2.1 분봉 스크리너 (신규)

`strategy/minute_screener.py` — 분봉 데이터로 실시간 종목 선별:

```python
@dataclass
class MinuteScreenCandidate:
    code: str
    name: str
    score: float        # 0~100 실시간 종합 점수
    reasons: list[str]
    indicators: dict    # 분봉 RSI, 거래량 급등, 모멘텀 등

def screen_by_minute(
    universe: list[tuple[StockInfo, list[PriceData]]],  # 일봉용 (종목 리스트)
    minute_data: dict[str, list[PriceData]],             # code → 분봉 데이터
    current_regime: MarketRegime,
    config: StrategyConfig,
) -> list[MinuteScreenCandidate]:
    """분봉 데이터로 실시간 종목 선별.
    
    각 종목당 30초 소요 (40종목 = 20초 병렬).
    점수 구성:
    - 분봉 RSI (30점): 30~70 = 고득점, 70+ = 과열 감점
    - 단기 모멘텀 (25점): 최근 5분 등락률 +1~3% = 고득점
    - 거래량 급등 (20점): 최근 5분 vs 20분 평균 대비
    - 분봉 변동성 (15점): 일정 수준 변동 = 고득점 (너무 낮거나 높으면 감점)
    - 일봉 보조 (10점): 일봉 MA 정렬 상태 (참고용)
    """
```

### 2.2 점수 구성 (100점 만점)

| 지표 | 배점 | 기준 | 변경 주기 |
|------|:----:|------|:--------:|
| 분봉 RSI | 30점 | 30~70 중립 = 30점, 70以上 = 과열 감점 | **매 분** |
| 단기 모멘텀 (5분) | 25점 | +1~3% = 25점, -3% 이하 = 0점 | **매 분** |
| 거래량 급등 (5분/20분) | 20점 | 1.5배 이상 = 20점, 0.5배 이하 = 0점 | **매 분** |
| 분봉 변동성 (ATR) | 15점 | 0.5~2.0% = 15점 | **매 분** |
| 일봉 MA 정렬 | 10점 | MA5 > MA20 = 10점 (일봉 참고) | **1일 1회** |

### 2.3 병렬 처리

40개 종목 분봉을 KIS API로 조회: 약 0.1초 × 40 = 4초
→ `asyncio.gather()`로 병렬 조회 → 4~5초

### 2.4 통합 위치

`orchestrator.py` — `run_cycle()` Step 5 (스크리닝) 다음:

```
Step 4b: 모드 선택
  ↓
Step 5: 일봉 스크리닝 (기존, 15개 후보)
  ↓
Step 5b: 분봉 스크리닝 (신규, 15개 → 순위 재조정)
  ↓
Step 6: ML 예측 (분봉 + 일봉 combined)
  ↓
Step 7: 매수/매도 실행
```

### 2.5 대시보드 진단

`/api/diagnosis`에 분봉 스크리닝 결과 추가:
