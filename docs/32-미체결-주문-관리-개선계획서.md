## 1. 문제 분석

### 1.1 현재 상태

JYP Ent(035900) 사례:
- `entry.limit_price=51,300원`으로 **지정가 매수 주문** 제출
- 시장가격이 51,300원에 도달하지 않아 **미체결 상태**
- 매수 주문은 기록되었지만(position trade_record), 실제 체결되지 않음
- 미체결 주문 취소 로직이 없어 주문이 계속 대기 상태로 남음

### 1.2 주문 유형

| 구분 | 지정가 (현재) | 시장가 |
|------|:------------:|:------:|
| KIS 코드 | `00` | `01` |
| 체결 보장 | 가격 도달 시 | 즉시 체결 (슬리피지 발생) |
| 현재 코드 | `price=entry.limit_price` | `price=None` |

## 2. 개선 설계

### 2.1 시장가 주문 조건 추가

`_refine_entry()`에서 분봉 기반 판단:
- **분봉 RSI가 과열(>70)이 아니고, 급등 위험이 없으면** → `limit_price=None` (시장가)
- **분봉 하락 추세 또는 급등 위험** → `limit_price=현재가` (지정가, 추격 매수 방지)

### 2.2 미체결 주문 취소 프로세스

#### 설계

`orchestrator`에 `_cancel_unfilled_orders()` 메서드 추가:

```python
async def _cancel_unfilled_orders(self):
    """미체결 주문 조회 후 일정 시간 경과 시 취소.

    각 사이클마다 실행:
    1. KIS 미체결 주문 목록 조회 (inquire_unfilled)
    2. 주문 제출 시간과 현재 시간 비교
    3. 설정 시간(초) 초과 시 주문 취소
    4. 취소 결과 기록
    """
```

#### 취소 조건

| 등급 | 제한 시간 | 설명 |
|------|:---------:|------|
| 일반 | **30초** | 지정가 매수 주문은 30초 후 미체결 시 취소 |
| 시장가 | 5초 | 시장가는 거의 즉시 체결되나, 예외 상황 대비 |
| 매도 | 60초 | 매도는 여유 있게 (슬리피지 방지) |

#### KIS API

- **미체결 주문 조회**: `TTTC8036R` (주문번호 조회) 또는 `CTSC9117R` (주문조회)
- **주문 취소**: `TTTC0803U` (이미 `cancel_order`에서 구현됨)

### 2.4 실행 위치

`main.py` trading_loop에서 `run_cycle()` 호출 후 또는 `orchestrator.run_cycle()` 내부에서 실행:

```
while True:
    # run_cycle에서 거래 실행 후
    result = await orchestrator.run_cycle(...)
    
    # 미체결 주문 취소 체크 (매 사이클)
    cancelled = await orchestrator._cancel_unfilled_orders()
```
