## 2. 항목별 상세

### #1 포지션 사이징 이중 관리 [MEDIUM]

**현황**:
- `orchestrator.__init__`: `position_budget_pct: float = 0.10` (기본값 10%)
- `main.py` line 175-185: `TradingOrchestrator()` 호출 시 `position_budget_pct` 인자 미전달 → 항상 10%
- `RiskManager.position_size_pct`: 모드별 값 리턴 (swing 10%, short 5%, hold 0%)

**문제**: 단타 모드(short_term)로 전환돼도 포지션 크기가 5%가 아닌 10%로 유지됨. 리스크 매니저의 모드별 포지션 크기가 실제 매매에 반영되지 않음.

**수정안**:
```python
# main.py
orchestrator = TradingOrchestrator(
    ...
    position_budget_pct=config.strategy.mode_swing_position_size,
)
```

**추가**: ATR 기반 다이나믹 사이징(base_budget_pct)이 mode별 값으로 시작하도록, RiskManager의 position_size_pct를 orchestrator가 참조하도록 개선.

### #3 STRONG_TREND_SCORE 중복 정의 [LOW]

**현황**: `strategy/mode_selector.py` line 49
```python
STRONG_TREND_SCORE = 60.0
```
이 값이 `config.regime_bull_threshold`(기본 65.0)와 별도로 정의되어 있음. threshold를 65→70으로 변경해도 mode_selector의 "강한 추세" 기준은 60으로 남아 불일치 발생 가능.

**수정안**: `config.regime_bull_threshold` 사용으로 변경.
```python
cfg = config or _StrategyConfig()
strong_trend = cfg.regime_bull_threshold - 5  # bull_threshold보다 5 낮게
```

### #4 한국 거래세 미반영 [LOW]

**현황**:
- 손절선 -3%에 매수/매도 수수료(~0.015%×2) + 증권거래세(0.15%) + 농특세(0.0225%) = 약 0.20% 미포함
- 실손실은 -3.20%로 손절선보다 0.20% 더 깊음

**수정안**: 손절선에 0.20%를 더하거나, 로그에 거래세 경고 표시.
- A안: 손절선값에 0.20% 추가 (mode_swing_stop_loss_pct 3.0 → 3.2, mode_short_stop_loss_pct 1.5 → 1.7)
- B안: 별도 파라미터 `transaction_cost_pct: float = 0.20` 추가해서 손절/익절 계산에 반영
- C안: 보류 — 실제 손실이 미미하므로(0.2%) 현행 유지
