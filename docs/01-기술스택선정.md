## 1. 데이터베이스: PostgreSQL

### 왜 PostgreSQL인가?
- **금융 데이터에 적합**: 거래 내역, 계좌 정보 등 돈과 관련된 데이터는 절대 유실되면 안 됨. PostgreSQL은 이를 보장하는 ACID 트랜잭션 지원
- **대용량 처리**: 시장 데이터(수십만 종목의 가격, 거래량 등)를 빠르게 저장하고 검색
- **구독 서비스 대비**: 여러 사용자의 데이터를 동시에 안전하게 관리
- **JSON 지원**: ML 모델의 판단 근거, 복잡한 설정값을 JSON 형태로 유연하게 저장

### 사용 방식
- 시장 데이터, 매매 이력, 판단 근거, 계정 설정 등 모든 데이터를 PostgreSQL에 저장
- Python의 SQLAlchemy 라이브러리로 데이터베이스와 통신

## 2. 백엔드: Python + FastAPI

### 왜 FastAPI인가?
- SAT3에서 이미 검증된 조합
- **비동기 처리**: 여러 종목의 데이터를 동시에 수집하고 분석하는 데 적합
- **실시간 통신**: 대시보드에 실시간으로 매매 현황을 전달 (WebSocket)
- **자동 문서 생성**: API 구조가 자동으로 문서화되어 유지보수 편리

### 사용 방식
- core/: 공통 설정, 데이터 모델
- data/: 데이터 수집 및 DB 저장
- strategy/: 매매 전략 로직
- ml/: 머신러닝 모델
- live/: 실거래 주문 실행 및 리스크 관리
- backtest/: 백테스트 실행 및 성과 분석

## 3. 프론트엔드: React + Vite + TailwindCSS

### 왜 React인가?
- **차트/그래프 생태계**: 주식 차트, 실시간 그래프를 그리는 데 가장 풍부한 라이브러리 (Recharts, Nivo, TradingView Lightweight Charts)
- **대시보드에 최적화**: 트레이딩 대시보드 예제와 컴포넌트가 가장 많음
- **유지보수**: 커뮤니티가 크고, 문제 발생 시 해결책을 찾기 쉬움

### 왜 TailwindCSS인가?
- **빠른 개발**: CSS를 따로 작성하지 않고 HTML 안에서 스타일링
- **일관성**: 디자인이 통일되어 유지보수가 쉬움

### 사용 방식
- dashboard/ 디렉토리에 React 프로젝트 구성
- 대시보드: 운영 요약, 계좌 현황, 보유 포지션, 매매 이력, 판단 근거 표시

## 4. 머신러닝: scikit-learn → XGBoost/LightGBM

### 왜 이 순서인가?
- **1차**: scikit-learn으로 빠르게 베이스라인 모델 구축 (가격 상승 확률 예측)
- **2차**: XGBoost/LightGBM으로 정확도 향상 (그래디언트 부스팅 — 표 형태 데이터에 가장 강력)
- **추후**: 딥러닝(PyTorch)은 필요할 때만 도입

### 사용 방식
- ml/training/: 모델 학습
- ml/inference/: 실시간 추론 (가격 상승 확률 계산)
- ml/features/: 피처 엔지니어링 (기술지표, 차트패턴을 숫자로 변환)
- 학습된 모델은 .pkl/.joblib 파일로 저장 (gitignore 제외)

## 6. 패키지 관리: uv + pyproject.toml

### 왜 uv인가?
- pip보다 10~100배 빠른 패키지 설치
- 의존성 관리가 명확
- 현재 환경에 이미 설치되어 있음

## 7. 전체 기술 스택 요약

| 영역 | 기술 | 이유 |
|------|------|------|
| 언어 | Python 3.12+ | ML/데이터 분석 생태계 최강 |
| 데이터베이스 | PostgreSQL | 금융 데이터 안정성, 구독 서비스 대비 |
| DB 통신 | SQLAlchemy | Python 표준 ORM |
| 백엔드 프레임워크 | FastAPI | 비동기, WebSocket, 자동 문서 |
| 프론트엔드 | React + Vite | 차트/대시보드 생태계 최강 |
| CSS | TailwindCSS | 빠른 개발, 일관성 |
| ML (1차) | scikit-learn | 빠른 베이스라인 구축 |
| ML (2차) | XGBoost / LightGBM | 표 데이터 최강 정확도 |
| 텔레그램 | python-telegram-bot | 알림 + 원격 제어 |
| 패키지 관리 | uv | 속도, 명확한 의존성 관리 |
| 테스트 | pytest | Python 표준 |
| 코드 품질 | ruff + mypy | 린터 + 타입 체크 |
