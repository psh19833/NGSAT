> 작성일: 2026-07-01

## 1. 문제 분석

### 1.1 현재 코드

```javascript
// main component (line 655-658):
for (const [name, p] of Object.entries(PRESETS)) {         // ← 구버전 하드코딩 PRESETS (3개만 있음)
    if (Object.entries(p.values).every(...)) {
        setActivePreset(name)
        break
    }
}

// PresetButtons (line 424-436):
const handleApply = async (name) => {
    ...
    const p = presets[name];
    if (p) onSelect(p.values);   // ← config 값만 업데이트, activePreset 미변경
    ...
}
```

### 1.2 2가지 버그

| # | 문제 | 영향 |
|:-:|------|------|
| ① | `PRESETS` vs `presets` | 활성 프리셋 감지 로직이 구버전 하드코딩 `PRESETS`(3개)를 사용. 새 API 프리셋(8개) 감지 불가 |
| ② | `handleApply`가 `activePreset` 미갱신 | 프리셋 적용 시 `onSelect(preset.values)`로 config 값만 변경, `activePreset` 상태는 그대로 |

### 1.3 결과

- 새 프리셋(단타형, 스윙형 등 5종)은 절대 하이라이트 안 됨
- 구 프리셋(안정형)도 하이라이트 안 됨 (값이 정확히 일치하지 않음)
- 항상 초기값 "균형형"이 하이라이트되어 있음

## 2. 개선 설계

### 2.1 PresetButtons에서 선택 상태 관리

`handleApply`가 성공하면 `onSelect` 콜백에 프리셋 이름도 함께 전달:

```javascript
// PresetButtons handleApply:
if (data.connected) {
    if (p) onSelect(p.values, name);  // ← preset 이름도 함께 전달
    ...
}
```

### 2.2 메인 컴포넌트에서 preset 이름 수신

```javascript
const handlePreset = (values, presetName) => {
    setConfig(values)
    setActivePreset(presetName)  // ← 선택한 프리셋 이름 저장
}
```

### 2.3 API 로드 시 활성 프리셋 감지

```javascript
// 기존: PRESETS 하드코딩 객체 사용
// 개선: API에서 로드한 presets 데이터 사용
const findActivePreset = (cfg, presetsFromApi) => {
    if (!presetsFromApi) return '균형형'
    for (const [name, p] of Object.entries(presetsFromApi)) {
        if (Object.entries(p.values).every(([k, v]) =>
            Math.abs(v - (cfg[k] ?? 0)) < 0.001
        )) return name
    }
    return null  // 일치하는 프리셋 없음
}
```

### 2.4 PresetButtons가 presets 데이터를 부모로 전달

PresetButtons가 API에서 presets를 로드하면, 부모 컴포넌트가 이 데이터를 참조할 수 있도록 `onPresetsLoaded` 콜백 추가:

```javascript
// PresetButtons 내
useEffect(() => {
    fetch('/api/strategy/presets')
        .then(r => r.json())
        .then(d => {
            if (d.connected) {
                setPresets(d.presets)
                onPresetsLoaded?.(d.presets)  // ← 부모에 전달
            }
        })
        .catch(() => {})
}, []);
```
