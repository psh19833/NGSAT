## Part A. 즉시 수정 (Critical, 실거래/운영에 직접 영향)

### A-1. WebSocket on_price 캐시 덮어쓰기 버그 (Critical, 10분)
- **관점**: 트레이더 + 백엔드
- **현황**: `real_data_provider.py` 218행 — `self._universe_cache[i] = (info, [updated])`가 가격 히스토리를 단일 분봉으로 대체
- **영향**: `len(prices) < 60` 조건에서 스크리닝·진입·청산 전부 생략
- **처리**: `[updated]` → `prices[-1] = updated` (마지막 가격만 갱신)
- **리스크**: ZERO — 1줄 수정
- **시간**: 10분

---

### A-2. Graceful Shutdown — SIGTERM/SIGINT 핸들러 (Critical, 1일)
- **관점**: 백엔드
- **현황**: `main.py`에 `asyncio.gather(*tasks)`만 있고, 종료 시그널 핸들러 없음
- **영향**: Ctrl+C/kill로 프로세스 종료 시 KIS WebSocket·DB 세션·텔레그램 폴링이 정리 안 됨. 다음 기동 시 포트 충돌, DB 연결 누수
- **처리**: `asyncio.get_event_loop().add_signal_handler()`로 SIGTERM/SIGINT 핸들러 등록. 모든 태스크 취소 + 리소스 정리
- **리스크**: LOW — 추가 코드, 기존 로직 변경 없음
- **시간**: 1일

---

### A-3. DB 엔진 이중화 — 연결 풀 경합 (Critical, 0.5일)
- **관점**: 백엔드
- **현황**: `data/db.py`(전역 싱글톤)와 `live/orchestrator.py`(독자 생성) — 두 개의 독립 SQLAlchemy 엔진
- **영향**: DB 연결 풀 2배 사용, 트랜잭션 격리 위반, 설정 불일치
- **처리**: `data/db.py`의 `get_engine()`을 orchestrator에서 재사용. sessionmaker 팩토리 주입 방식으로 변경
- **리스크**: MEDIUM — 데이터 접근 패턴 변경. 롤백: git checkout
- **시간**: 0.5일

### A-6. Pickle ML 모델 RCE 방어 (Critical, 0.5일)
