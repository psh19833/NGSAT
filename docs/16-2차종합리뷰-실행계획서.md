### A-5. refresh_prices() 실패 시 사이클 진행 (Critical, 30분)
- **관점**: 백엔드
- **현황**: `refresh_prices()`가 40종목 순회 중 일부 실패해도 조용히 넘어감. 전체 실패해도 사이클 계속 진행
- **영향**: KIS API 장애 시 오래된 데이터로 ML 예측 진행 → 실제가와 괴리된 매매 결정
- **처리**:
```python
# main.py trading_loop() — 수정안
_refresh_failures = 0
if _refresh_counter % 30 == 0:
    try:
        universe, index_prices = await data_provider.refresh_prices()
        _refresh_failures = 0
    except Exception:
        _refresh_failures += 1
        if _refresh_failures >= 3:
            logger.error("연속 3회 데이터 갱신 실패 — 사이클 중단")
            break
```
- **리스크**: LOW — 서킷 브레이커 패턴, 과도한 중단 방지
- **시간**: 30분
