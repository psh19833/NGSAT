### A-2. Error Boundary — 화이트스크린 방어 (Critical, 30분)
- **관점**: 디자이너
- **현황**: 단일 JSX 에러가 전체 대시보드를 하얀 화면(white screen)으로 만듦
- **영향**: API 오류나 데이터 구조 변경 시 사용자에게 아무것도 표시되지 않음
- **처리**: `ErrorBoundary.jsx` 신규 + App.jsx 루트에 적용
```jsx
class ErrorBoundary extends React.Component {
  state = { hasError: false, error: null }
  static getDerivedStateFromError(error) { return { hasError: true, error } }
  render() {
    if (this.state.hasError) {
      return <div className="...">오류 발생: {this.state.error.message}</div>
    }
    return this.props.children
  }
}
```
- **리스크**: ZERO — 신규 컴포넌트 추가, 기존 코드 변경 없음
- **소요시간**: 30분

- **리스크**: LOW — 새 컴포넌트, 기존 handler에 조건만 추가

### A-4. 단타 모드 손절선 1% → 현실적 조정 (Critical, 5분)
- **관점**: 트레이더
- **현황**: `core/config.py` 133행 `mode_short_stop_loss_pct: float = 1.0`
- **영향**: 수수료(0.015%)+세금(0.23%)+슬리피지 포함 시 실질 손실 -1.5~-1.8%. 1% 손절 10회 연속 = -15%, 복구에 +17.6% 필요
- **처리**: 기본값 1.0 → **1.5**로 조정 (대시보드에서 사용자 조정 가능)
```python
mode_short_stop_loss_pct: float = 1.5  # 1.0→1.5
```
- **리스크**: LOW — 실거래 중인 경우 기존 포지션 손절선 영향 있을 수 있음. 프로세스 재시작 후 적용
- **소요시간**: 5분
