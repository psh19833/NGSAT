# 분봉ML AUC 개선 방안 리서치

**작성일:** 2026-07-10

---

## 1. 현재 상태 요약

| 항목 | 현재값 | 문제 |
|:-----|:------:|:-----|
| AUC | **0.452~0.515** | 무작위(0.5) 수준 — 예측력 없음 |
| forward_minutes | **60분** | 너무 긴 예측 기간 |
| forward_threshold | **1.0%** | 60분에 1%는 매우 큰 움직임 |
| 양성비율 | **7%** | 극심한 불균형 (93% 음성) |
| 샘플 수 | **2,160개** | 27종목 × 80샘플 — 매우 부족 |
| class_weight | **없음** | 불균형 대책 없음 |

---

## 2. 근본 원인

### 2.1 예측 기간이 너무 김 (60분)

60분 뒤 1% 상승을 예측하는 것은 분봉 데이터의 SNR(신호대잡음비)이 너무 낮음. 60분 동안 무수한 노이즈가 발생하여 예측 불가능.

**해결:** forward_minutes 60분 → **10~20분**으로 단축

### 2.2 클래스 불균형 (양성 7% vs 음성 93%)

모델이 "항상 하락"만 예측해도 93% 정확도. 양성 샘플이 너무 적어 패턴 학습 불가.

**해결:** class_weight='balanced' 또는 SMOTE 오버샘플링

### 2.3 상승 임계값 1% (너무 높음)

60분에 1% 상승은 연간 수익률로 환산하면 2,000%가 넘는 초고수익. 발생概率이 극히 낮아 학습 불가.

**해결:** forward_threshold 1% → **0.3~0.5%** 로 하향

### 2.4 데이터 부족

2,160개 샘플은 25개 피처를 가진 ML 모델을 학습시키기에 턱없이 부족.

**해결:** 더 많은 종목(40+)과 더 긴 기간(500 bars)의 데이터 활용

---

## 3. 권장 개선안

### A안: 파라미터 튜닝 (1줄 수정)

| 파라미터 | 현재 | 변경 |
|:---------|:----:|:----:|
| forward_minutes | 60분 | **20분** |
| forward_threshold | 1.0% | **0.5%** |
| class_weight | 없음 | **'balanced'** |

**예상 효과:** AUC 0.45 → ~0.60

### B안: A안 + 데이터 증강

- 종목 수: 27개 → **40개** (전체 유니버스)
- 샘플 수: 80/종목 → **200/종목** (더 긴 기간)
- 양성비율: 7% → **~20%** (임계값 하향 효과)

**예상 효과:** AUC ~0.65

### C안: B안 + 모델 앙상블

- 5개 모델(gradient_boosting, xgboost, random_forest, logistic, lightgbm) 학습 후 최고 AUC 자동 선택
- auto_select_model=True 활용 (이미 True)

**예상 효과:** AUC ~0.70
