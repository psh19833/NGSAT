## 1. 개요

### 1.1 배경

초기 기획서(00-기획서.md) Section 6.5에서 정의한 텔레그램 알림 3개 항목 중 2개만 구현되었습니다.

| 알림 항목 | 기획 | 구현 | 상태 |
|----------|:----:|:----:|:----:|
| 거래내역 알림 (매수/매도 체결) | ✅ | ✅ | 정상 |
| 일일 보고 (장 마감 후) | ✅ | ❌ | **코드만 있음, 호출부 없음** |
| 가동상태 이벤트 (시작/중단/오류) | ✅ | ✅ | 정상 |
| **장 시작/종료 메시지** | **❌ (신규)** | ❌ | **미구현** |

### 1.2 목표

1. **장 시작 메시지**: 오전 9시 장 시작 시 텔레그램으로 시장 상황 알림
2. **장 종료 메시지**: 오후 3시 30분 장 종료 시 알림
3. **일일 매매종합보고서**: 장 마감 후 당일 거래 내역 요약 자동 발송

### 1.3 성공 기준

- 장 시작/종료를 감지하여 텔레그램으로 즉시 알림
- 일일 보고서에는 당일 거래 건수, 승률, 손익, 보유 포지션 포함
- 서버 재시작 후에도 정상 동작 (상태 유지)

## 2. 상세 설계

### 2.1 장 시작/종료 감지

#### 현재 상태

`orchestrator.run_cycle()`에서 매 사이클마다 `is_market_hours()`를 호출하지만, **이전 상태를 저장하지 않아** 변화를 감지할 수 없습니다.

#### 설계

`orchestrator`에 MarketSessionTracker 추가:

```python
class MarketSessionTracker:
    """시장 세션 상태 추적 — 장 시작/종료 감지 및 알림 트리거."""

    STATE_CLOSED = "closed"
    STATE_OPEN = "open"

    def __init__(self):
        self._previous_state: str | None = None  # None = 최초 실행
        self._daily_report_sent: bool = False     # 오늘 보고서 전송 여부

    def update(self, is_open: bool) -> dict:
        """
        현재 시장 상태를 업데이트하고, 상태 변화가 있으면 정보 반환.

        Returns:
            {"changed": True, "state": "open", "is_first": False}
            또는 {"changed": False}
        """
        current = self.STATE_OPEN if is_open else self.STATE_CLOSED
        result = {"changed": False}

        if self._previous_state is None:
            # 최초 실행 — 알림 없이 상태만 저장
            self._previous_state = current
            return result

        if self._previous_state != current:
            result["changed"] = True
            result["state"] = current
            # 장 마감 → daily report 전송 필요 플래그
            if current == self.STATE_CLOSED:
                self._daily_report_sent = False
            self._previous_state = current
            return result

        return result

    @property
    def should_send_daily_report(self) -> bool:
        """장 마감 후 최초 1회만 보고서 전송."""
        return (
            self._previous_state == self.STATE_CLOSED
            and not self._daily_report_sent
        )

    def mark_daily_report_sent(self):
        self._daily_report_sent = True
```

#### 통합 위치

`main.py` trading_loop — `run_cycle()` 호출 전후에 상태 체크 로직 추가:

```
while True:
    market_open = is_market_hours()

    # 장 시작/종료 감지
    session_result = session_tracker.update(market_open)
    if session_result["changed"]:
        if session_result["state"] == "open":
            telegram_bot.send_system_event("market_open", ...)
        else:
            telegram_bot.send_system_event("market_close", ...)

    # run_cycle() ...
    result = await orchestrator.run_cycle(index_prices, universe)

    # 장 마감 후 일일 보고서 전송
    if session_tracker.should_send_daily_report:
        report = build_daily_report(trade_repo, account)
        await telegram_bot.send_daily_report(**report)
        session_tracker.mark_daily_report_sent()
```

## 6. 검증 방법

1. 장 시작 전에 서버 실행 → `is_market_hours()`가 False → "장 종료" 상태
