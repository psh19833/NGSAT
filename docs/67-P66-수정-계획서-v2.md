# P-66 수정 계획서 — RS 기반 동적 스크리닝

**기반 문제**: sector_weights.py가 섹터명을 하드코딩 → 시장 상황 변화에 대응 불가
**해결 방향**: 하드코딩 제거 + RS(Relative Strength) 가중치 증대 + bear threshold 완화

---

## 1. 변경 사항

### 1.1 sector_weights.py 제거

`sector_weights.py` 자체를 삭제. `screener.py`의 import 및 호출 코드도 제거.

| 파일 | 변경 |
|:-----|:------|
| `strategy/sector_weights.py` | **삭제** (하드코딩) |
| `strategy/screener.py` | import `get_sector_bonus` 제거, `sector_bonus=` 호출 제거 |

### 1.2 scorer.py — RS 가중치 조정 (핵심)

`_REGIME_WEIGHTS`에서 BEAR 레짐의 RS 가중치를 **5→20**으로 증대.

```python
# 변경 전
"bear": {
    "rsi": 15, "mfi": 20, "adx_di": 10, "obv": 10,
    "ma": 5, "volume": 10, "pattern": 10, "candle": 15, "rs": 5,
}
# 변경 후  
"bear": {
    "rsi": 15, "mfi": 10, "adx_di": 10, "obv": 5,
    "ma": 5, "volume": 10, "pattern": 10, "candle": 5, "rs": 20,
}
```

RS 가중치 5→20, 대신 MFI 20→10, OBV 10→5, Candle 15→5로 조정 (합계 100 유지).

**효과**: BEAR에서 시장 대비 선방한 종목(RS > 1.0)이 스크리너 통과 확률 증가.

### 1.3 bear threshold 완화 (config.py)

| 항목 | 변경 전 | 변경 후 | 사유 |
|:-----|:-------:|:-------:|:------|
| `screener_bear_min_score` | **80** | **50** | 스크리너는 1차 필터, ML이 최종 평가 |
| `screener_bear_max_candidates` | 5 | **8** | 후보 풀 확대 |

**근거**: 
- 스크리너는 1단계 필터일 뿐, **ML이 최종 진입 결정** (buy_threshold 65%)
- 80점은 너무 높아 어떤 종목도 통과 불가 → ML 평가 기회 자체가 없음
- 50점 + RS 가중치(20)로 방어 종목 우선 통과 → ML이 65% 이상일 때만 진입

## 2. 리스크 평가

### 2.1 운영 리스크

| # | 리스크 | 확률 | 영향 | 대응 |
|:-:|:-------|:----:|:----:|:------|
| R1 | **bear threshold 50→나쁜 종목 통과** | 중 | 중 | **ML 65% + Kelly 2~8%로 이중 방어** |
| R2 | **RS 데이터 부족 (신규 상장)** | 낮음 | 낮음 | `relative_strength()`가 데이터 부족 시 1.0 리턴 = 무영향 |
| R3 | **RS 가중치 20으로 과도한 편향** | 중 | 중 | ML이 최종 결정하므로 편향은 제한적 |
| R4 | **threshold 50으로 후보 폭증** | 낮음 | 중 | max_candidates=8로 제한 |

### 2.2 방어 계층 (이중 안전장치)

```
스크리너 (threshold 50 + RS 가중치 20)
  → 최대 8개 후보 선별
    → ML 추론 (buy_threshold 65%)
      → Kelly 포지셔닝 (최대 8% 배분)
        → 손절 3% 자동 청산
```

스크리너가 통과시켜도 **ML이 65% 미만이면 진입 안 함**.  
ML이 매수해도 **Kelly가 최대 8%로 제한**.  
매수해도 **손절 3%로 손실 통제**.

### 2.3 sector_weights.py 제거 리스크

| 항목 | 리스크 | 설명 |
|:-----|:-------|:------|
| import 참조 | 없음 | `screener.py`에서 호출 제거만 하면 됨 |
| scorer.py 호환성 | 없음 | `sector_bonus=` 파라미터 유지 (기본값 0.0, 영향 없음) |
