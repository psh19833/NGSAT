> **상태:** 기획 (리뷰 필요)

## 1. 문제 정의

현재 ML 학습은 "최근 N일" 데이터만 사용한다. 하지만 시장은 반복된다.
사용자가 원하는 것은 **"지금과 비슷한 과거 특정 기간의 데이터로 학습"**한 후,
그 모델로 현재 자동매매를 하는 것이다.

**예시:**
- 지금이 약세장이면 → 2022년 약세장 데이터로 학습
- 지금이 금리 인상기면 → 2022-2023년 금리 인상기 데이터로 학습
- 특정 구간이 좋았으면 → 그 구간만 골라서 학습

## 2. 기술적 현황

### KIS API 지원 확인 ✅
`adapter.py:194` — `get_price_history(code, start, end)` 
- `FID_INPUT_DATE_1`: 시작일 (YYYYMMDD)
- `FID_INPUT_DATE_2`: 종료일 (YYYYMMDD)
→ 이미 임의 날짜 구간 조회 가능

### RealDataProvider (`data/real_data_provider.py`)
- 현: `load()`에서 `end = now, start = now - timedelta(days=250)` 하드코딩
- `fetch_code(code, adapter, start, end)`: 이미 start/end 파라미터 존재
- `_fetch_index()`: start/end 파라미터 추가 필요

### 대시보드 API (`dashboard/backend/api.py`)
- `GET /api/strategy/config` → `asdict(cfg)` 반환 (필드 자동 포함)
- `PUT /api/strategy/config` → Pydantic 검증 후 setattr + DB 저장
- ml_training_days 이미 추가 완료

## 3. 변경 대상

### 3.1 StrategyConfig (`core/config.py`)
```
ml_training_start_date: str | None = None   # YYYY-MM-DD
ml_training_end_date: str | None = None     # YYYY-MM-DD (기본: 오늘)
```

### 3.2 RealDataProvider (`data/real_data_provider.py`)
- `__init__`에 `start_date`/`end_date` 옵션 파라미터 추가
- `load()`에서 날짜가 있으면 해당 구간, 없으면 기존 days 방식
- `_fetch_index()`에 날짜 전달
- `fetch_code()`는 이미 start/end 지원

### 3.3 ConfigService (`core/config_service.py`)
- 날짜 문자열 저장/복원 (이미 문자열 지원)

### 3.4 대시보드 API (`dashboard/backend/api.py`)
- CONFIG_FIELD_MAP: `NGSAT_ML_TRAINING_START_DATE`, `NGSAT_ML_TRAINING_END_DATE`
- StrategyUpdateRequest: `ml_training_start_date` / `ml_training_end_date`

### 3.5 대시보드 UI (`StrategyConfigPanel.jsx`)
- "⑧ ML 학습 기간" 섹션에 날짜 선택기(date input) 2개 추가
- 시작일, 종료일 각각 `<input type="date">`
- 현재 "학습 기간(일)" 슬라이더는 보조 모드로 유지

### 3.6 main.py
- `run_backtest()`: RealDataProvider에 start_date/end_date 전달
- `run_live()`: trading_loop provider에도 전달

## 4. 작업 범위 및 리스크

### 선택지

| 선택지 | 범위 | 예상 시간 | 리스크 |
|--------|------|:---------:|:------:|
| **A 🥇** | 날짜 선택 UI + RealDataProvider + Config | **3시간** | LOW — KIS API가 이미 지원 |
| B | A + "유사 구간 추천" (레짐 기반) | 6시간 | MEDIUM — 추천 로직 설계 필요 |
| C | A + 학습된 모델별 성능 비교 탭 | 8시간 | MEDIUM |

**추천: A안**

### 리스크
1. **KIS API 데이터 가용성:** KIS는 최대 10년 치 데이터 제공. 2015년 이전은 조회 불가.
2. **데이터 포인트 부족:** 기간이 너무 짧으면(10일 미만) 학습 불가 → 최소 30일 체크 필요
3. **과거 데이터와 현재 시장 구조 차이:** 상장폐지/액면분할 등 이슈 → KIS 수정주가(adj_prc) 사용 중이므로 영향 적음

## 5. 사용 예시

```
1. 대시보드 전략 설정 → ⑧ ML 학습 기간
2. "과거 데이터 사용" 체크박스 ON
3. 시작일: 2022-01-01  (금리 인상기)
   종료일: 2023-06-30
4. 저장 → 다음 학습 사이클에 2022-2023 데이터로 학습
5. 학습된 모델로 현재 시장 매매
```

## 6. 추후 확장 가능

- **유사 구간 추천:** 레짐 점수 기반으로 현재와 가장 비슷했던 과거 구간 자동 추천
- **멀티 모델:** 같은 기간 여러 모델(gradient_boosting/xgboost) 동시 학습 후 최적 선택
- **구간 비교 모드:** 여러 구간(A:2022, B:2020, C:2018) 각각 학습 → 성능 비교표
