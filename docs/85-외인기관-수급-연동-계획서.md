## 1. 개요

KIS API `inquire_investor`를 통해 종목별 외국인/기관 순매수 데이터를 조회하여
스크리닝 점수에 가산점으로 반영.

### 현재 상태
- ✅ `data/adapters/kis/adapter.py` → `get_investor_data(code)` — **이미 구현됨**
- ✅ `data/adapters/kis/mapper.py` → `parse_investor_data()` — **이미 구현됨**
- ✅ 엔드포인트 `inquire_investor` 등록 완료

### 2.1 score_investor_flow() 함수

```python
def score_investor_flow(investor_data: dict | None) -> float:
    """외인/기관 순매수 데이터 기반 점수 (0~100).

    Args:
        investor_data: get_investor_data() 결과 dict.
            foreign_net_buy_amt, institution_net_buy_amt

    Returns:
        0~100 점수. 데이터 없으면 50.0 (중립).
    """
    if not investor_data:
        return 50.0

    foreign_amt = investor_data.get("foreign_net_buy_amt", 0.0)
    inst_amt = investor_data.get("institution_net_buy_amt", 0.0)
    total = foreign_amt + inst_amt

    # 1억 이상 순매수 = 100점, -1억 이상 순매도 = 0점
    # 0 = 50점 (중립)
    score = 50.0 + (total / 100_000_000) * 50.0
    return max(0.0, min(100.0, score))
```

### 2.2 스크리너 연동

`_evaluate_single_stock()`의 thresholds에 `investor_data` 전달:

```python
# screener.py _evaluate_single_stock() 내부
investor_data = thresholds.get("investor_data", {})
investor_score = score_investor_flow(investor_data.get(stock.code))
indicator_scores["investor"] = investor_score
```

### 2.3 entry_planner 데이터 조회

`plan_entries()`에서 스크리닝 후 후보 종목에 대해 수급 데이터 조회:

```python
# Step 5: Screen stocks (기존)
screen_result = screen_stocks(...)

# Step 5a: 수급 데이터 조회 (신규)
investor_data = {}
for c in screen_result.candidates[:10]:  # 최대 10개
    try:
        data = await broker.get_investor_data(c.code)
        if data:
            investor_data[c.code] = data
    except Exception:
        pass

# Step 5로 돌아가서 thresholds에 investor_data 추가
# → screen_stocks 재호출 또는 기존 screen_result에 점수 반영
```

## 3. 리스크 평가

| 항목 | 리스크 | 내용 |
|---|---|---|
| **API 호출 증가** | 🟡 **중간** | 사이클당 후보 수×1회 = 최대 10회 추가 API 호출. Rate Limit 부담 |
| **스크리너 구조 변경** | 🟢 낮음 | thresholds dict에 investor_data만 추가. sync/async 경계 처리 필요 |
| **데이터 무결성** | 🟢 낮음 | API 실패 시 0 반환 → score 50.0 중립. 점수 왜곡 없음 |
| **기존 점수 영향** | 🟢 낮음 | 가중치 5~10으로 기존 점수 대비 영향 제한적 |

**총평: 🟡 Medium — API 호출 증가가 주 리스크. 후보 10개로 제한하여 Rate Limit 통제.**
