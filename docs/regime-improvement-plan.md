# 시장 레짐 평가 개선 — 실행 계획서 (A안)

## 개요
현재 5개 기술지표 가중합산 방식 유지 + ADX(추세강도) 신규 추가
가중치 및 지표 미세 조정으로 레짐 판단 정확도 향상

---

## 변경 사항 요약

| 항목 | 현재 | 변경 |
|------|------|------|
| MA 정렬 | 35% | 30% (-5%p) |
| ADX (신규) | — | 5% (신규) |
| RSI | 20% | 20% (유지) |
| 볼린저밴드 | 20% | 20% (유지) |
| 5일 등락률 | 15% | 15% (유지) |
| 거래량 추세 | 10% | 15% (+5%p) |

## 리스크 검토

### 리스크 #1 — MA 정렬 35→30% 감소로 추세 감지 약화
**영향:** MEDIUM
- MA 정렬이 현재 레짐 점수에 가장 큰 영향을 미치는 지표
- 5%p 감소로 극단적(강한 강세/약세) 상황에서 점수 변동 둔화 가능
- **대응:** ADX가 추세강도를 보완하므로 순손실 아님. 기존 점수와의 비교를 위해 1주일 dry-run 병행 필요

### 리스크 #2 — ADX가 MA 정렬과 중복
**영향:** LOW
- MA 정렬 = 방향 + 정렬 상태 (가격 > MA5 > MA20 > MA60 등)
- ADX = 추세 강도 (방향 무관, 0~100)
- 두 지표의 성격이 달라 중복 위험 낮음. 오히려 상호 보완적

### 리스크 #3 — ADX 파라미터(14)의 한국장 적합성
**영향:** LOW-MEDIUM
- ADX 14는 미국 시장 기준이 일반적
- 한국 KOSPI는 미국보다 변동성 큰 편으로, ADX 10~14가 무난
- **대응:** ADX(10) / ADX(14) 두 가지를 비교하는 테스트 포함

### 리스크 #4 — 거래량 비중 10→15% 상향
**영향:** MEDIUM
- 한국 시장은 장 마감 동시호가(10분)에 하루 거래량의 15~20% 집중
- 프로그램 매매(차익거래)가 거래량을 인위적으로 왜곡
- **대응:** 거래량 지표에 20일 이동평균 대비 비율 사용 (현재와 동일)하여 단기 왜곡 방어

### 리스크 #5 — 레짐 판정 변경으로 매매 전략 영향
**영향:** HIGH
- 점수 체계 변경으로 다음과 같은 상황 발생 가능:
  - 기존 Bull(스윙) → 변경 후 Neutral(단타/관망) → 진입 기회 손실
  - 기존 Neutral → 변경 후 Bull → 과도한 진입
- **대응:** 
  - **반드시 dry-run(read-only) 1주일 병행 운영**
  - 변경 전/후 점수 차이를 대시보드에 시각화
  - 실제 매매에는 기존 레짐 사용, 신규 레짐은 참고용으로만 표시
  - 1주일 후 점수 분포 안정성 확인 후 전환

### 리스크 #6 — 볼린저밴드 폭(BBW) 보조지표
**영향:** LOW
- 점수에 포함되지 않으므로 매매 영향 없음
- 단순 참고용 → 리스크 없음

## 구현 단계

### Phase 1 — 지표 구현 (1~2시간)
```
1. indicators.py에 ADX 함수 추가
   - true_range(), directional_movement(), adx() 구현
   - Wilder's smoothing (SMA가 아닌 RSI 방식의 평활화)
   
2. 볼린저밴드 폭(BBW) 함수 추가
   - BBW = (upper - lower) / middle * 100
   - 횡보장 감지용: BBW < 2% → 밴드 수축 → 변동성 확대 임박
```

### Phase 2 — 레짐 평가에 통합 (1시간)
```
3. regime.py에 _score_adx() 함수 추가
   - ADX 범위 0~100을 0~100점으로 매핑
   - ADX < 20: 추세 없음 (50점 중립)
   - ADX 20~40: 추세 있음 (50~80점)
   - ADX > 40: 강한 추세 (80~100점)
   
4. evaluate_regime()에 ADX 점수 추가
   - 가중치 5%, MA 정렬 30%로 조정
   - 거래량 15%로 상향

5. 볼린저밴드 폭 evidence에 추가
   - score에는 포함 안 함, reason/evidence에만 표시
```

### Phase 3 — 설정 + 시각화 (1시간)
```
6. core/config.py에 ADX 관련 설정 추가
   - regime_weight_adx: float = 5.0  # 신규
   - regime_weight_ma: 35.0 → 30.0  # 변경
   - regime_weight_volume: 10.0 → 15.0  # 변경

7. 대시보드 StrategyConfigPanel에 ADX 가중치 슬라이더 추가
   - "ADX 추세강도" 0~20% 범위

8. 레짐 응답에 변경 전/후 점수 비교 추가
   - evidence에 "adx_score", "bb_width" 추가
```

### Phase 4 — Dry-run + 검증 (7일)
```
9. dry-run(read-only) 모드 운영
   - 기존 레짐으로 매매 실행
   - 신규 레짐 점수는 DB에만 저장, 대시보드에 표시
   
10. 7일 후 점수 분포 비교 리포트
    - 기존 vs 신규 점수 차이 통계
    - 레짐 판정 변경 횟수 (Bull→Neutral 등)
    - 이상 징후 없으면 전환
```

## Phase 4 통과 기준

| 기준 | 목표 |
|------|------|
| 점수 차이 평균 | ±10점 이내 |
| 레짐 판정 변경률 | 20% 미만 (기존 Bull→Neutral 등) |
| 이상 스파이크 | ADX > 80 또는 점수 급변 없음 |
| 해석 가능성 | reason 문장에 ADX 설명 포함되어 자연스러움 |

## 롤백 방안

1. 설정값만 변경했으므로 대시보드에서 ADX 가중치=0, MA=35, 거래량=10으로 복원
2. 위험: 롤백해도 학습된 모드 전환 이력은 복원 안 됨
3. 대응: dry-run 기간 동안만 신규 레짐 적용, 실제 매매는 항상 기존 레짐 사용
