# 백테스트 "알 수 없는 오류" — 원인분석 + 수정계획서

> 일자: 2026-06-30
> 기준 커밋: ee5d0a3

---

## 1. 문제 증상

대시보드에서 "백테스트 실행" 클릭 → "실패: 알 수 없는 오류" 표시

## 2. 원인 분석

### 2.1 직접 원인

`dashboard/backend/api.py:570-571` — **API endpoint에 try/except 없음**

```python
result = await loop.run_in_executor(
    None, lambda: asyncio.run(run_backtest_async(cfg))
)
# ← 예외 발생 시 FastAPI 500 에러, frontend는 {detail: "..."}만 받음
return {"connected": True, "status": "completed", "result": result}
```

백테스트 내부 예외 → unhandled → FastAPI 500 응답 (`{"detail": "Internal Server Error"}`) 
→ frontend `resp.status` = undefined, `resp.message` = undefined
→ `"알 수 없는 오류"` 표시

### 2.2 백테스트 내부 예외 가능성

| # | 원인 | 영향 |
|---|------|------|
| 1 | `RealDataProvider` KIS 연결 실패 → fallback 처리 정상 | OK |
| 2 | ML `train_from_price_data` 데이터 부족 실패 | `model=None` → `BacktestEngine(model=None)` TypeError |
| 3 | `run_in_executor` + `asyncio.run()` 이중 async 컨텍스트 | 드물지만 이벤트루프 충돌 가능 |
| 4 | `_backtest_state` 공유변수 스레드 안전성 부족 (BE-12 락) | race condition 시 예외 |

### 2.3 추가 문제: 레짐 가중치 합이 100 아님

로깅에서 `레짐 가중치 합이 100이 아님: 105.0` 발견. 기본 config값 합이 100 초과.

```
regime_weight_ma: 30.0
regime_weight_rsi: 20.0
regime_weight_bollinger: 20.0
regime_weight_change_rate: 15.0
regime_weight_volume: 15.0  ← 원래 10이어야 하는데 15
regime_weight_adx: 5.0
합계: 105.0
```

Phase 2 TR-2+3에서 volume=10으로 수정했었으나, config 기본값이 15로 남아있음.

## 3. 수정 계획

### 3.1 API 오류 처리 추가 (api.py, 10분)

```python
# backtest_run() endpoint에 try/except 추가
try:
    result = await loop.run_in_executor(
        None, lambda: asyncio.run(run_backtest_async(cfg))
    )
    return {"connected": True, "status": "completed", "result": result}
except Exception as e:
    logger.exception(f"백테스트 실행 실패: {e}")
    return {"connected": True, "status": "error", "message": f"백테스트 실패: {e}"}
```

### 3.2 Frontend 오류 메시지 개선 (BacktestPanel.jsx, 5분)

상태="error" 응답 처리 및 에러 메시지 표시.

### 3.3 레짐 가중치 기본값 수정 (config.py, 2분)

`regime_weight_volume: 15.0` → `10.0` (Phase 2 TR-2+3 반영 누락)
