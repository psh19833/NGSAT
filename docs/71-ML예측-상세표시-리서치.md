## 1. 현황 분석

### 1.1 현재 동작 방식

`DiagnosisPanel.jsx`는 진단 API(`/api/diagnosis`)를 5초마다 폴링하여 화면에 렌더링한다.

**스크리너 종목 (현재 — 확장 가능)**
```jsx

// 클릭 시 expandedStock = i 로 설정 → 상세 ScoreBar 표시

```

**ML 예측 (현재 — 확장 불가)**
```jsx
predictions.map((p, i) => (
  <div className="flex items-center justify-between ...">
    <span>{p.name}({p.code})</span>
    <span>{(p.probability * 100).toFixed(0)}%</span>
  </div>
  // ← 클릭 이벤트 없음, 상세 정보 없음
))
```

### 1.2 API 응답 구조

현재 predictions 항목:
```json
{
  "code": "259960",
  "name": "크래프톤",
  "action": "hold",
  "probability": 0.372,
  "reason": "ML 예측: 홀드 (상승 확률 37.2%)..."

```

MLInference에서 생성되는 `evidence` 데이터가 **API 응답에 포함되지 않고 있음.**

### 1.3 사용 가능한 evidence 데이터

`predict_entry()`에서 생성:
```python
evidence = {
    "rise_probability": proba,        # 상승 확률 (0~1)
    "screening_score": candidate.score,# 스크리너 점수 (0~100)
    "buy_threshold": self._buy_threshold,
    "sell_threshold": self._sell_threshold,
    "patterns_detected": len(candidate.patterns),  # 감지 패턴 수
    "rsi": fv.features.get("rsi_14", 0),           # RSI(14)
    "macd_histogram": fv.features.get("macd_histogram", 0), # MACD
    "bollinger_position": fv.features.get("bollinger_position", 0), # 볼린저
}
```

`predict_minute_entry()`에서 생성:
```python
evidence = {
    "rise_probability": proba,
    "screening_score": candidate.score,
    "buy_threshold": self._buy_threshold,
    "mode": "short_term",
    "minute_rsi": mfv.features.get("m_rsi_14", 0),      # 분봉 RSI
    "minute_volume_spike": mfv.features.get("m_volume_spike_5", 0), # 거래량
    "minute_momentum_3": mfv.features.get("m_momentum_3", 0), # 모멘텀
}
```

일봉ML fallback 시에도 동일한 evidence 구조 사용 가능.

## 2. 수정 설계

### 2.1 Backend: evidence를 API 응답에 추가

**파일:** `live/entry_planner.py` — predictions 데이터에 evidence 포함

```python
# 현재 (L294-300)
result["predictions"].append({
    "code": pred.code, "name": pred.name,
    "action": pred.action.value,
    "probability": round(pred.rise_probability, 3),
    "reason": pred.reason,
})

# 수정 후
result["predictions"].append({
    "code": pred.code, "name": pred.name,
    "action": pred.action.value,
    "probability": round(pred.rise_probability, 3),
    "reason": pred.reason,
    "evidence": pred.evidence,  # ← 추가
})
```

**변경량:** +1줄

**리스크:** 🔴 ZERO — evidence는 MLPrediction dataclass에 이미 존재, API 응답에 추가만 하면 됨. 프론트엔드가 없는 필드는 무시하므로 하위 호환성 100%.

### 2.2 Frontend: ML 예측 아이템 확장/접기 + 상세 표시

**파일:** `dashboard/frontend/src/components/DiagnosisPanel.jsx`

**상태 추가:**
```jsx
const [expandedPred, setExpandedPred] = useState(null)  // ML 예측 확장
```

**ML 예측 리스트 수정 (기존 135-149행):**
```jsx
{predictions.map((p, i) => {
  const isExpanded = expandedPred === i
  const ev = p.evidence || {}
  return (
    <div key={i}>
      <div className="..." onClick={() => setExpandedPred(isExpanded ? null : i)}>
        <span>{p.action === 'buy' ? '🟢' : p.action === 'hold' ? '🟡' : '🔴'}</span>
        <span>{p.name}({p.code})</span>
        <span>{(p.probability * 100).toFixed(0)}%</span>
      </div>
      {isExpanded && (
        <div className="ml-5 pl-3 border-l-2 border-ngsat-border py-2 space-y-1.5 text-xs">
          <ScoreBar label="스크리너점수" raw={ev.screening_score} />
          <ScoreBar label="RSI" raw={ev.rsi} />
          <ScoreBar label="MACD" raw={ev.macd_histogram} />
          <ScoreBar label="볼린저위치" raw={ev.bollinger_position} />
          {ev.minute_rsi != null && (
            <ScoreBar label="분봉RSI" raw={ev.minute_rsi} />
          )}
          {ev.minute_volume_spike != null && (
            <ScoreBar label="분봉거래량" raw={ev.minute_volume_spike} />
          )}
          {ev.minute_momentum_3 != null && (
            <ScoreBar label="분봉모멘텀" raw={ev.minute_momentum_3} />
          )}
          <div className="pt-1 text-[10px] text-ngsat-muted border-t border-ngsat-border/50 mt-1">
            {p.reason}
          </div>
        </div>
      )}
    </div>
  )
})}
```

**변경량:** ~30줄 (스크리너 확장 패턴과 거의 동일)

**ScoreBar 재사용:** 기존 `ScoreBar` 컴포넌트는 모든 label을 자동 정규화하므로 그대로 재사용 가능. 신규 label(분봉RSI, 분봉거래량 등)은 자동으로 else 분기 처리됨.

## 3. ScoreBar 정규화 — label별 신규 매핑

현재 ScoreBar는 label 기반 정규화를 수행한다. 신규 label 추가:

| label | 정규화 로직 | 범위 |
|:------|:-----------|:----:|
| `스크리너점수` | `val` (0~100) | RSI 분기와 동일 |
| `MACD` | `(val + 5000) / 100` (0~100 clamp) | -5000~5000 → 0~100 |
| `볼린저위치` | `(val + 1) * 50` (0~100 clamp) | -1~1 → 0~100 |
| `분봉RSI` | `val` (0~100) | RSI 분기와 동일 |
| `분봉거래량` | `val * 20` (0~100 clamp) | 0~5 → 0~100 |
| `분봉모멘텀` | `(val + 5) * 10` (0~100 clamp) | -5~5 → 0~100 |

**또는** — 더 간단한 방법: 숫자는 그냥 0~50 사이 값으로 정규화하는 일반 분기로도 충분. `label === 'MACD'` 특수 분기만 추가하면 됨.

## 5. 리스크 검토

| 항목 | 등급 | 내용 |
|:-----|:----:|:------|
| evidence 필드 누락 | 🟢 LOW | `pred.evidence`가 None이면 `{}` fallback, 프론트에서 `ev.screening_score ?? '?'` 처리 |
| 하위 호환성 | 🔴 ZERO | 새로운 응답 필드는 기존 프론트에서 무시됨 |
| ScoreBar 미정의 label | 🟢 LOW | else 분기 `(val+1)*50`으로 안전 처리 |
| **종합** | **🟢 LOW** | **프론트엔드만 변경, 데이터 구조 변경 없음** |
