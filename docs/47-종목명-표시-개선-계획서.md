# 종목명 표시 개선 — 수정 계획서

> **문제**: 대시보드 스크리너/ML예측에 `종목189400(189400)` 형식으로 표시. `_code_to_name()` 딕셔너리가 약 40개 종목만 커버하고, 동적 유니버스에서 편입된 종목은 `종목{code}` fallback 처리
> **파일**: `docs/47-종목명-표시-개선-계획서.md`

---

## 문제 분석

### 현재 구조

```
코드 흐름:
  data_provider.load()
    → StockInfo(name=_code_to_name(code))    # 40개 하드코딩
  universe_manager._load_daily_data()
    → StockInfo(name=_code_to_name(code))    # 같은 딕셔너리 (P1에서 수정 완료)
  _code_to_name(code)                         # data/real_data_provider.py:492
    → names = {"005930": "삼성전자", ...}     # 약 40개만 커버
    → return names.get(code, f"종목{code}")   # 나머지는 "종목{code}" fallback
```

### 문제점

| 문제 | 설명 |
|:----|------|
| **딕셔너리 한계** | 하드코딩된 40개 종목만 커버. KOSPI 900종목 + KOSDAQ 1500종목 중 1.7% |
| **동적 유니버스** | 순위 API로 선정되는 종목은 딕셔너리에 없을 확률 99% |
| **ETF/ETN/스팩** | Q, 0, 1, 2로 시작하는 코드 다양 — 딕셔너리만으로 커버 불가 |
| **유지보수** | 새 종목 추가 시마다 _code_to_name() 수동 업데이트 필요 |

---

## 해결 방안

### A안 (권장): KIS 종목정보 API 활용

KIS `inquire_stock_basic` (CTPF1604R) 엔드포인트를 사용해 **실시간 종목명 조회**.

**이미 있는 코드**:
- `endpoints.py:153` — `inquire_stock_basic` 엔드포인트 정의 완료
- `mapper.py:235` — `parse_stock_info()` 파서 구현 완료
- 미구현: `adapter.py _get_stock_info(code)` 메서드 + 이름 조회 로직

**수정 항목**:

| # | 항목 | 파일 | 작업량 | 리스크 |
|:-:|------|------|:------:|:-----:|
| 1 | `KisAdapter.get_stock_info(code)` 구현 | `adapter.py` | ~10줄 | 🟢 LOW |
| 2 | `_code_to_name()` → API 기반 조회로 변경 | `real_data_provider.py` | ~15줄 | 🔵 MEDIUM |
| 3 | 응답 캐싱 (TTL 1일) | 신규 또는 `_code_to_name` 내부 | ~10줄 | 🟢 LOW |
| 4 | API 실패 시 기존 딕셔너리 fallback 유지 | `real_data_provider.py` | ~3줄 | 🟢 ZERO |

**동작 방식**:
```
_code_to_name(code):
  1. 캐시 조회 (code → name dict, TTL 1일)
  2. 캐시 미스 → KIS API 호출 (get_stock_info)
  3. API 성공 → 캐시 저장 + name 반환
  4. API 실패 → 하드코딩 딕셔너리 조회
  5. 딕셔너리에도 없음 → "종목{code}" fallback
```

**Rate Limit 영향**:
- 초기 로드 시 40종목 × 1회 API = 총 40회 (1회성)
- 이후 신규 편입 시에만 추가 호출 (하루 수십 회)
- 캐싱으로 중복 호출 없음
- Rate Limiter(C-1) 보호 적용

### B안: 대량 딕셔너리 구축

KIS `volume_rank` API 응답에 포함된 종목명을 수집하여 대량 딕셔너리 구축.

| 항목 | 평가 |
|:----|:----:|
| 개발량 | 적음 (수집로직 ~20줄) |
| 커버리지 | 상위 100종목만 가능 |
| 유지보수 | 매일 갱신 필요 |
| 한계 | 거래량 없는 종목(ETF/스팩) 누락 |

**비권장**: 거래량 순위 100종목만 커버 가능. 동적 유니버스 후보가 이 100종목에 국한되지 않음.

### C안: `_code_to_name()` 딕셔너리 수동 확장

현재 40개 → 200개로 수동 확장.

| 항목 | 평가 |
|:----|:----:|
| 개발량 | 많음 (200종목 수동 입력) |
| 정확도 | 높음 (수동 검증) |
| 유지보수 | **매월 수동 업데이트 필요** |
| 한계 | 2000종목 중 10% 커버 — 여전히 fallback 다수 |

**비권장**: 확장 불가능한 방식.

## 권장: A안 (KIS API 활용)

### 상세 구현

**1. KisAdapter.get_stock_info(code) 추가** (`adapter.py`)

```python
async def get_stock_info(self, code: str) -> str:
    """종목코드로 종목명 조회 (inquire_stock_basic)."""
    params = {
        "FID_COND_MRKT_DIV_CODE": "J" if code[:1].isdigit() else "Q",
        "FID_INPUT_ISCD": code,
    }
    resp = await self._http.get("inquire_stock_basic", params=params)
    if not resp.success:
        return ""
    from data.adapters.kis.mapper import parse_stock_info
    info = parse_stock_info(resp.data)
    return info.name
```

**2. _code_to_name() API 연동** (`real_data_provider.py`)

```python
_name_cache: dict[str, tuple[str, float]] = {}  # code -> (name, expiry_ts)
_NAME_CACHE_TTL = 86400  # 1일

async def _code_to_name(code: str, adapter=None) -> str:
    """종목코드 → 종목명 (API 캐시 + fallback)."""
    # 캐시 조회
    now = time.time()
    if code in _name_cache:
        name, expiry = _name_cache[code]
        if now < expiry:
            return name
    # 하드코딩 딕셔너리
    names = {"005930": "삼성전자", ...}  # 기존 40개 유지
    if code in names:
        _name_cache[code] = (names[code], now + _NAME_CACHE_TTL)
        return names[code]
    # KIS API 호출
    if adapter:
        try:
            name = await adapter.get_stock_info(code)
            if name:
                _name_cache[code] = (name, now + _NAME_CACHE_TTL)
                return name
        except Exception:
            pass
    return f"종목{code}"
```

**3. _load_daily_data() 동기화** (`universe_manager.py`)

현재 `_load_daily_data()`는 이미 `_code_to_name()` 사용 중 (P1 수정 완료). 
API 버전으로 변경 시 `adapter` 파라미터 전달 필요.

```python
from data.real_data_provider import _code_to_name
name = await _code_to_name(code, adapter)  # 비동기 호출
StockInfo(code=code, name=name, market=market)
```
