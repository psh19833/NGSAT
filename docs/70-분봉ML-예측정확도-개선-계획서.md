## 개요

분봉ML 모델의 예측 확률이 전 종목 0.4%~31.6%로 매우 낮게 유지되고 있다.
원인 분석 결과 3가지 문제가连环적으로 작용 중:

1. `parse_minute_history()` 정렬 누락 → 초기 모델이 뒤집힌 데이터로 학습 → AUC 0.0
2. 자동 재학습 데이터 부족(60개 < 70개 필요) → AUC 0.0 모델 영구 유지
3. 단타 모드에서 분봉ML만 예측, 일봉 모델(AUC 0.70) fallback 없음

## Problem 2: 분봉 모델 재학습

### 현황

- 현재 모델: `models/trained/minute_model.pkl`
- XGBClassifier, n_features=25, AUC=**0.0** (완전 반대 방향 학습)
- 자동 재학습이 60개 < 70개 데이터 조건으로 영구 실패
- 마지막 학습: 2026-07-06 11:03 (5일 전)

### 수정: 직접 재학습 실행

DB에 저장된 전체 분봉 데이터(7/8일 270개, 7/9일 누적 중)로 수동 재학습:

```python
# 1회성 재학습 스크립트 (scripts/retrain_minute_model.py)
from ml.training.trainer import train_from_minute_data
from data.repository import MinuteDataRepository
from data.db import get_engine
from sqlalchemy.orm import Session

engine = get_engine()
with Session(engine) as session:
    repo = MinuteDataRepository(session)
    codes = DEFAULT_CODES  # 30종목
    minute_prices_list = []
    minute_codes = []
    for code in codes:
        bars = repo.get_minute_bars(code)  # 전체 분봉 데이터
        if len(bars) >= 100:
            minute_prices_list.append(bars)
            minute_codes.append(code)

model, result = train_from_minute_data(
    minute_prices_list, minute_codes,
    model_type="xgboost",
    forward_minutes=10,
    forward_threshold=0.01,
)
if model.is_trained:
    from ml.training.persistence import save_model
    save_model(model, "models/trained/minute_model.pkl")
    print(f"재학습 완료: AUC={result.auc:.3f}")
```

### 기대 AUC

- DB 분봉 데이터: 7/8일 09:00~15:50 (270봉), 7/9일 누적 중
- 약 400봉 × 30종목 = 12,000개 샘플
- 특징: 25개 (RSI/MACD/볼린저/거래량/모멘텀/변동성 등)
- 예상 AUC: 0.55~0.65 (10분 앞 예측은 노이즈가 많음)

### 리스크

🟢 LOW — 모델만 교체, 시스템 재시작 필요 없음 (hot-reload)

## Problem 4: 단타 모드 분봉ML failover

### 현황

```python
# live/entry_planner.py — _evaluate_and_execute()
if ctx.is_short_term:
    minute_prices = ...
    if minute_prices and len(minute_prices) >= 30:
        pred = self._inference.predict_minute_entry(candidate, minute_prices)
        # AUC 0.0이어도 무조건 분봉ML 사용
    else:
        pred = self._inference.predict_entry(candidate, prices)
        # 일봉 모델(AUC 0.70)은 분봉 데이터만 있으면 호출 불가
```

### 수정

```python
# live/entry_planner.py
if ctx.is_short_term:
    minute_prices = ...
    if minute_prices and len(minute_prices) >= 30:
        pred = self._inference.predict_minute_entry(candidate, minute_prices)
        # 분봉ML 신뢰도가 낮으면 일봉ML로 fallback
        if pred and pred.rise_probability < 0.05 and self._inference.has_minute_model:
            daily_pred = self._inference.predict_entry(candidate, prices)
            if daily_pred and daily_pred.rise_probability > pred.rise_probability:
                pred = MLPrediction(
                    code=daily_pred.code, name=daily_pred.name,
                    rise_probability=daily_pred.rise_probability,
                    action=daily_pred.action,
                    reason=f"일봉ML fallback: {daily_pred.reason}",
                    evidence=daily_pred.evidence,
                )
    else:
        pred = self._inference.predict_entry(candidate, prices)
```

failover 조건:
- 분봉ML 확률 < 5% (매수 신호와 거리가 먼 경우)
- 일봉ML 확률 > 분봉ML 확률 (일봉이 더 낫다고 판단될 때만)
- 일봉ML 액션이 BUY일 때만

### 리스크

🟢 LOW — fallback 조건이 명확하고, 일봉 모델(AUC 0.70)이 더 신뢰할 수 있음

## 실행 계획

### Phase A: 즉시 적용 (리스크 ZERO, 5분)

### Phase B: 모델 재학습 (리스크 LOW, 10분)

## 실행 순서 (리스크 오름차순)

```
Phase A1 (1분) → Phase A2 (1분) → Phase C (15분) → Phase B (10분)
↓                    ↓                  ↓               ↓
정렬 수정         auto_retrain       failover        모델 재학습
(재시작 필요)    (재시작 필요)      (재시작 필요)   (hot-reload)
```

1️⃣ **A1**: parse_minute_history 정렬 — 근본 원인 해결
2️⃣ **A2**: auto_retrain 데이터 60→200 — 다음 재학습부터 정상 동작
3️⃣ **C1~C3**: failover 로직 추가 — 재학습 전에도 일봉 모델로 버틸 수 있도록
4️⃣ **B1~B4**: 모델 재학습 — AUC 0.0 → 예상 0.55~0.65
