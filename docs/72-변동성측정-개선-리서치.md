## 1. 현재 문제

`estimate_volatility_from_prices()`는 실제로 **CV(변동계수)** 를 계산하지만, 임계값은 **ATR% 기준**으로 설정되어 있어 단위 불일치 발생.

| 구분 | 현재 측정값 | 임계값 |
|:-----|:----------:|:------:|
| 측정 방식 | CV = σ/μ × 100 | `mode_high_volatility_atr_pct` |
| KOSPI 기준 | 3~8% | 1.5~2.0% |
| 결과 | `5.3% >= 2.0%` → **항상 고변동** | 중립+중간변동 구간 소멸 |

---

## 2. 비교 검토: 3가지 수정 방안

### 방안 A: ATR%로 변경 (추천)

`estimate_volatility_from_prices()`를 실제 ATR 계산으로 변경.

```python
def estimate_volatility_from_prices(closes, highs, lows, period=14):
    """ATR% 계산 — True Range의 이동평균 / 종가."""
    tr = np.maximum(highs - lows,
        np.maximum(abs(highs - np.roll(closes, 1)),
                   abs(lows - np.roll(closes, 1))))
    tr[0] = highs[0] - lows[0]
    atr = np.mean(tr[-period:])  # SMA of TR
    return atr / closes[-1] * 100
```

| 측면 | 평가 |
|:-----|:------|
| **유지보수** | ✅ 함수명(`estimate_volatility`), 파라미터명(`atr_pct`), 임계값(`1.5%~2.0%`) 모두 **일관성 확보**. 설정값 재조정 불필요. |
| **거래전략** | ✅ ATR%는 업계 표준 변동성 지표. 실제 가격 변동폭을 직관적으로 반영. KOSPI ATR%는 보통 0.8~2.5% 수준으로 임계값과 정합. |
| **데이터** | ✅ index_prices에 high/low/close 모두 존재 — 추가 데이터 수집 불필요. |
| **리스크** | 🔴 ZERO — 함수 내부만 변경, 외부 인터페이스 동일 |
| **공수** | ~10분 |

### 방안 B: CV 유지 + 임계값 재조정 + 파라미터명 변경

임계값을 CV 기준에 맞게 상향 조정하고, 설정 파라미터명을 변경.

| 변경 항목 | 현재 | 수정 |
|:----------|:----:|:----:|
| `mode_high_volatility_atr_pct` | 2.0 | `mode_high_volatility_pct` = **4.0** |
| `mode_low_volatility_atr_pct` | 0.7 | `mode_low_volatility_pct` = **2.0** |

| 측면 | 평가 |
|:-----|:------|
| **유지보수** | ⚠️ 파라미터명 `atr_pct`는 그대로 두면 혼란 가중. 변경 시 DB/환경변수/설정화면 동기화 필요. |
| **거래전략** | ⚠️ CV는 σ/μ 비율로 해석이 어려움. ATR%만큼 직관적이지 않음. |
| **리스크** | 🟢 LOW — CV 유지로 함수 로직 변경 없음. 단, 임계값 산출 근거 문서화 필요. |
| **공수** | ~15분 (파라미터 rename + DB ConfigService 매핑 수정) |

### 방안 C: 개별 종목 ATR 집계로 대체

스크리너가 이미 계산한 종목별 ATR%의 중앙값을 사용.

```python
# mode_selector.py
def select_mode(regime, atr_pct=None, stock_atr_values=None, ...):
    if stock_atr_values:
        vol = np.median(stock_atr_values)  # 종목 ATR 중앙값
```

| 측면 | 평가 |
|:-----|:------|
| **유지보수** | ❌ stock_atr_values를 전달받기 위해 호출 체인(orchestrator → entry_planner) 수정 필요. 의존성 증가. |
| **거래전략** | ❌ 유니버스 종목 ATR은 시장 전체 변동성보다 **개별 종목 변동성**을 반영. 종목 구성에 따라 값이 크게 달라짐. |
| **리스크** | 🟡 MED — 호출 체인 변경으로 회귀 위험 |
| **공수** | ~30분 |

## 4. 수정 제안 코드

**파일:** `strategy/mode_selector.py` — `estimate_volatility_from_prices()`만 교체

```python
def estimate_volatility_from_prices(
    closes: Sequence[float],
    highs: Sequence[float] | None = None,
    lows: Sequence[float] | None = None,
    period: int = 14,
) -> float:
    """ATR%로 변동성 추정.

    실제 True Range 기반 ATR을 계산하여 일관된 변동성 측정 제공.
    high/low가 없으면 표준편차 기반으로 폴백.

    Args:
        closes: 종가 리스트.
        highs: 고가 리스트 (True Range 계산용).
        lows: 저가 리스트.
        period: ATR 기간 (기본 14).

    Returns:
        ATR(%). 데이터 부족 시 0.5 반환.
    """
    closes_arr = np.asarray(closes, dtype=float)
    if len(closes_arr) < period + 1:
        return 0.5

    if highs is None or lows is None or len(highs) < period + 1 or len(lows) < period + 1:
        # Fallback: 표준편차 기반 (high/low 없을 때)
        recent = closes_arr[-period:]
        std = float(np.std(recent))
        mean = float(np.mean(recent))
        return (std / mean * 100) if mean > 0 else 0.5

    highs_arr = np.asarray(highs, dtype=float)
    lows_arr = np.asarray(lows, dtype=float)

    # True Range = max(H-L, |H-prevC|, |L-prevC|)
    prev_close = np.roll(closes_arr, 1)
    prev_close[0] = closes_arr[0]
    tr = np.maximum(highs_arr - lows_arr,
                    np.maximum(np.abs(highs_arr - prev_close),
                               np.abs(lows_arr - prev_close)))

    # ATR = SMA of True Range over period
    atr = float(np.mean(tr[-period:]))
    current_price = float(closes_arr[-1])
    if current_price <= 0:
        return 0.5
    return atr / current_price * 100
```

**변경 불필요:** `.env`, `core/config.py`, `mode_selector.py`의 `select_mode()`, `entry_planner.py`의 호출부 — 모두 현재 상태 유지.
