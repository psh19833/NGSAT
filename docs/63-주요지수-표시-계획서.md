# 대시보드 주요지수 표시 — 구현 계획서

**목적**: KOSPI, KOSDAQ 등 주요 지수를 대시보드에서 항상 볼 수 있도록 추가

---

## 1. 현재 데이터 가용성

### 백엔드에서 이미 제공 중인 데이터

| 데이터 | 출처 | 형식 |
|:-------|:------|:------|
| KOSPI 등락률 | `regime API` → `evidence.intraday_correction` | ±% |
| KOSPI 장중보정 | `regime API` → `reason` 문자열에 포함 | 텍스트 |
| 레짐 점수 | `regime API` → `score` | 0~100 |
| 시장 상태 | `regime API` → `regime` | bull/neutral/bear |

### 추가로 필요한 데이터

| 데이터 | 필요 API | 비고 |
|:-------|:---------|:------|
| KOSPI 현재가 | 신규 API 또는 기존 확장 | 실시간 필요 |
| KOSDAQ 현재가/등락률 | 신규 API | KIS `get_index_price()`에 KOSPI/KOSDAQ 모두 포함? |
| 전일 대비 등락폭 | 신규 API | 증감포인트 |

**검토 결과**: KIS `get_index_price()`가 KOSPI 데이터만 반환할 가능성이 높음. KOSDAQ 지수는 별도 API 필요할 수 있음.

### 제안 배치 A: RegimeCard 통합 (권장)

RegimeCard 내부에 KOSPI 등락률을 추가 표시. RegimeCard가 이미 시장 상태를 표시하므로 지수 정보를 함께 보여주는 것이 자연스러움.

```
┌─────────────────────┐
│    📈 중립장 (41점)  │
│  KOSPI -4.7% (-250P) │  ← 추가
│  KOSDAQ -3.2% (-80P) │  ← 추가
│  장중보정 -5점 적용   │
└─────────────────────┘
```

**변경량**: RegimeCard.jsx ~10줄  
**장점**: 정보의 연속성, 레이아웃 변경 없음, 추가 API 1~2개면 충분  
**단점**: RegimeCard가 다소 복잡해짐

### 제안 배치 B: Top Row를 4칸으로 확장

```
┌──────────┬──────────┬──────────┬──────────┐
│  Status  │  Regime  │  Indices │ Control  │
│ running  │ 중립장   │ KOSPI -5%│ start    │
└──────────┴──────────┴──────────┴──────────┘
```

**변경량**: App.jsx ~5줄 + IndexCard.jsx 신규 ~60줄  
**장점**: 전용 영역, 시인성 좋음  
**단점**: 4칸 그리드는 3칸보다 각 칸이 좁아짐, 모바일에서 더 좁아짐

### 제안 배치 C: 헤더 영역 (항상 표시)

헤더(sidebar 위)에 작게 지수 표시줄 추가. 모든 탭에서 항상 보임.

```
🚀 NGSAT  │  KOSPI 2,450(-4.7%)  │  KOSDAQ 780(-3.2%)  │  running
```

**변경량**: App.jsx ~10줄  
**장점**: 항상 표시, 모든 탭에서 가시성  
**단점**: 헤더가 복잡해짐, 모바일에서 너무 김

## 4. 구현 상세

### 4.1 백엔드: 지수 API 추가 또는 기존 확장

**옵션 1**: `regime` API 응답에 index 데이터 추가 (권장)

```python
# live/orchestrator.py — _last_diagnosis or regime 응답
{
  "regime": "neutral",
  "score": 41.0,
  "indices": {
    "kospi": {"price": 2450.3, "change_pct": -4.7, "change_pts": -120.5},
    "kosdaq": {"price": 782.5, "change_pct": -3.2, "change_pts": -25.8},
  }
}
```

**변경량**: `orchestrator.py` ~10줄 (진단 저장 시 index 데이터 수집)

**옵션 2**: 신규 `/api/indices` 엔드포인트

```python
@app.get("/api/indices")
async def get_indices():
    kospi = await broker.get_index_price("KOSPI")
    kosdaq = await broker.get_index_price("KOSDAQ")
    return {"kospi": ..., "kosdaq": ...}
```

**변경량**: `main.py` ~15줄 + `adapter.py` ~5줄

**권장: 옵션 1** — API 호출 최소화 (기존 regime API에 포함)

### 4.2 프론트엔드: RegimeCard 확장

**파일**: `RegimeCard.jsx` ~10줄 추가

```jsx
// 현재
export default function RegimeCard({ regime }) {
  return (
    <div className="ngsat-card p-6">
      <h3 className="text-sm text-ngsat-muted mb-4">시장 레짐</h3>
      ...
    </div>
  )
}

// 변경 — 지수 정보 표시 추가
export default function RegimeCard({ regime, indices }) {
  return (
    <div className="ngsat-card p-6">
      <h3 className="text-sm text-ngsat-muted mb-4">시장 레짐</h3>
      ...
      {indices && (
        <div className="mt-3 pt-3 border-t border-ngsat-border text-xs space-y-1">
          <div className="flex justify-between">
            <span>KOSPI</span>
            <span className={indices.kospi.change_pct >= 0 ? 'text-ngsat-green' : 'text-ngsat-red'}>
              {indices.kospi.price.toFixed(1)} ({indices.kospi.change_pct:+.1f}%)
            </span>
          </div>
          <div className="flex justify-between">
            <span>KOSDAQ</span>
            <span className={indices.kosdaq.change_pct >= 0 ? 'text-ngsat-green' : 'text-ngsat-red'}>
              {indices.kosdaq.price.toFixed(1)} ({indices.kosdaq.change_pct:+.1f}%)
            </span>
          </div>
        </div>
      )}
    </div>
  )
}
```

## 5. 리스크 분석

| # | 리스크 | 확률 | 영향 | 대응 |
|:-:|:-------|:----:|:----:|:------|
| R1 | KOSPI/KOSDAQ 실시간 데이터 API 미제공 | 중 | 높음 | KIS `get_index_price()`가 KOSPI만 지원할 수 있음. 이 경우 KOSPI만 표시 |
| R2 | API 호출 증가로 Rate Limit 소진 | 중 | 중 | `regime API`에 포함(옵션1)하면 추가 API 호출 없음. 옵션2는 1회 추가 |
| R3 | 지수 데이터 갱신 주기와 대시보드 5초 폴링 불일치 | 낮음 | 낮음 | 5초마다 갱신, 지수는 1분 단위로 변동하므로 문제 없음 |
| R4 | 장중 지수 데이터 없는 경우 (휴일/장 전) | 낮음 | 낮음 | `null` 체크로 미표시 처리 |
