## 1. 현황 분석

### 현재 데이터 흐름

```
KIS API ──→ RealDataProvider (하드코딩 250일) ──→ 모델 학습 (run_backtest)
                                                  └──→ main.py 합성데이터 폴백 (250일)

저장된 모델.pkl ──→ run_live (재학습 없이 로드만, ml_auto_retrain=false)
```

### 핵심 문제점

| 문제 | 위치 | 영향 |
|------|------|------|
| 학습 기간 250일 고정 | `real_data_provider.py:109` `start = end - timedelta(days=250)` | 사용자가 기간 설정 불가 |
| 합성 데이터 폴백 | `main.py:87-88` 실패 시 250일 하드코딩 | 실 데이터 우선 정책 미흡 |
| live 모드 재학습 미사용 | `main.py:148-154` 저장된 모델만 로드 | `--train` 없이 실 데이터 학습 불가 |
| ml_auto_retrain 기본 false | `core/config.py:122` | 장 마감 후 자동 재학습 비활성 |

### 이미 해결된 부분

- **ml_model_type** config → 기본값 gradient_boosting (이번 세션에서 수정)
- **auto_retrain + auto_tune 통합** → Optuna 20회 주기 튜닝 (이번 세션에서 수정)
- **모델 5종 주석 + 설명** (이번 세션에서 수정)

## 2. 요구사항

1. **ML 학습 기간을 사용자가 설정할 수 있어야 함**
   - 기본값: 250일 (현행 유지)
   - 설정 범위: 60 ~ 750일
   - .env / 대시보드에서 변경 가능

2. **실 데이터(KIS API)를 우선 사용해야 함**
   - 합성 데이터는 KIS API 장애 시에만 폴백
   - 폴백 시에도 사용자 설정 기간 존중

3. **live 모드에서도 실 데이터 학습 가능해야 함**
   - `--train` 플래그 또는 `ml_auto_retrain=true` 설정 시
   - 기동 시 RealDataProvider로 40종목 실 데이터 로드 → 모델 학습

## 3. 설계 방안

### 3A. Config 추가 (LOW RISK, ✅ 추천)

**StrategyConfig**에 필드 2개 추가:

```python
ml_training_days: int = 250      # 학습 기간 (일)
ml_training_data_source: str = "kis"  # "kis" | "synthetic" (기본: kis)
```

**환경변수:**
```
NGSAT_ML_TRAINING_DAYS=250
NGSAT_ML_TRAINING_DATA_SOURCE=kis
```

**변경 파일:** `core/config.py`

### 3B. RealDataProvider 학습 기간 인자 추가 (LOW RISK, ✅ 추천)

```python
# data/real_data_provider.py
class RealDataProvider:
    def __init__(self, codes=None, training_days: int = 250):
        self._training_days = training_days
        ...
```

- `load()` 메서드에서 `timedelta(days=self._training_days)` 사용
- `_fetch_index()`에서도 동일 기간 사용

**변경 파일:** `data/real_data_provider.py`

### 3C. main.py 실 데이터 학습 파이프라인 개선 (MEDIUM RISK)

**run_backtest():**
- RealDataProvider에 `training_days` 전달
- 합성 폴백 시 `n_days=training_days` 사용

**run_live() with --train:**
- `--train` 플래그 있을 때 Provider로 데이터 로드 → `train_from_price_data()` 호출 → 저장

**변경 파일:** `main.py`

### 3D. live 모드 자동 재학습 (ml_auto_retrain) 보강 (LOW)

- Orchestrator가 Provider 접근 가능하게 주입
- `ml_auto_retrain=true` + `ml_training_days` 설정 시 사이클마다 재학습
- 실 데이터로만 재학습 (synthetic 폴백 시 skip)

**변경 파일:** `live/orchestrator.py`

## 4. 변경 영향도

| 파일 | 변경 내용 | 영향 범위 |
|------|----------|-----------|
| `core/config.py` | 2개 필드 추가 | 전역 config |
| `data/real_data_provider.py` | 생성자 + load() 파라미터 변경 | 2개 호출 지점 |
| `main.py` | backtest/live 모두 training_days 전달 | ~3개 지점 |

**호환성:**

- Orchestrator 수정 → main.py에서만 호출, 영향 제한적

## 6. 리스크 검토

| 항목 | 리스크 | 대응 |
|------|--------|------|
| training_days 증가 시 KIS API 호출 증가 | MEDIUM | Rate Limit 대비 50ms 간격 이미 적용 |
| live 모드 학습 시간 증가 (40종목×250일) | LOW | startup 시 1회, 백그라운드 실행 |
| auto_retrain 사이클 지연 가능성 | LOW | 학습 중에도 현재 모델로 계속 매매 |
| 모델 재학습 시 AUC 하락 가능성 | LOW | `auto_retrain()`이 AUC 비교 후 더 나쁘면 기존 유지 |
