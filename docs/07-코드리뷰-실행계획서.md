### 재리뷰 발견사항

Phase 1 수정 과정에서 발견한 추가 이슈:

1. **orchestrator 진단 dict 필드 불일치** — `CycleResult`에 없는 `regime_score`/`regime_label`을 참조하려던 코드를 발견하고 원래의 `result.regime.value` + `getattr(self._last_regime, 'score', 0)` 패턴으로 복원 완료
2. **auto_tune() `_create_model()` 필요 없음** — 기획서에서 별도 메서드 분리를 가정했으나, 실제 수정은 inline model 생성으로 충분 (모델 종류가 4종으로 확정적이고 중복 로직이 objective 함수와 동일하므로)
3. **단타 ML threshold 실제값 확인** — config 주석은 +0.5%로 적혀 있었지만 `minute_builder.py`의 실제 `forward_threshold=0.01`(1%)로 이미 설정되어 있었음. config 타겟 주석을 +1.0%로 정정하고 손절만 조정

## Phase 2: 설계 결함 개선 (다음 순위, 1~2일)

### 2-1. DB 의존성 주입 및 생명주기 정비
- **파일:** `live/orchestrator.py` (__init__)
- **문제점 3가지:**
  1. SQLite 경로 하드코딩 — 설정의 PostgreSQL URL 무시
  2. `Session` 인스턴스 변수 보관 — 스레드 안전성 위험
  3. `commit()` 호출 시 예외 롤백 없음
- **수정안 A (단기):** 설정에서 DB URL 읽도록 변경 + `with session.begin()` 패턴
- **수정안 B (장기):** DB 세션 팩토리를 orchestrator 외부에서 주입받도록 설계 변경
- **난이도:** ★★★☆☆ (1~2일)

### 2-2. 글로벌 config → DI로 전환
- **파일:** `strategy/regime.py`, `strategy/screener.py`, `strategy/mode_selector.py`
- **문제:** 3개 모듈이 공유 `_strategy_config` 글로벌 변수 사용, 런타임 교체 시 경쟁 조건
- **수정:** 각 함수 인자로 config를 명시적 전달. 단기: ImmutableConfig로 교체
- **난이도:** ★★★☆☆ (1일)
- **참고:** mode_selector의 모듈 상수 문제는 Phase 1에서 해결 완료. 남은 것은 글로벌 패턴 자체의 재설계

### 2-3. 포지션 집중 리스크 제어
- **파일:** `live/orchestrator.py` (_calculate_quantity)
- **문제:** 포지션 당 10%지만 최대 종목 수, 업종 한도, 단일 종목 최대 비중 없음
- **수정:** Config에 `max_holdings`, `max_sector_exposure_pct` 필드 추가 → 진입 시 체크
- **난이도:** ★★☆☆☆ (1일)

### 2-4. 예외 처리 설계 재검토
- **파일:** `live/orchestrator.py` (_fetch_minute_prices)
- **문제:** `except (NotImplementedError, Exception)` — NotImplementedError가 Exception의 하위 클래스이므로 중복
- **수정:** 구체적 예외 타입 명시 + `logger.warning` 기록
- **난이도:** ★☆☆☆☆ (30분)
- **참고:** screener 예외 삼킴 패턴은 리뷰 오독이었음 (실제로 `except Exception: continue` 패턴 존재하지 않음)

### 2-5. dashboard .env 직접 수정 개선
- **파일:** `dashboard/backend/api.py`
- **문제:** PUT `/api/strategy/config`가 .env 파일 직접 수정 → 동시 쓰기 손상 위험
- **수정안 A (단기):** 파일 락(`fcntl.flock`) 추가
- **수정안 B (장기):** 설정 저장소(DB 또는 별도 config 서비스)로 이관
- **난이도:** ★★☆☆☆ (1일)

## Phase 3: 코드 품질 개선 (여유 있을 때, 1~2일)

### 3-1. EMA 표준화
- **파일:** `strategy/indicators.py`
- **문제:** `result[0] = arr[0]` — 표준 EMA는 SMA를 시드로 사용
- **수정:** 초기값을 `np.mean(arr[:period])`로 대체 (또는 주석으로 명시)
- **난이도:** ★☆☆☆☆ (30분)

### 3-2. RSI 인덱스 정렬
- **파일:** `strategy/indicators.py`
- **문제:** `gains[i-1]`이 `i=period`에서 이미 SMA에 포함된 값을 중복 계산
- **수정:** 루프를 `range(period+1, len(arr))`로 시작
- **난이도:** ★☆☆☆☆ (30분)

### 3-3. max_overflow 설정 로딩 추가
- **파일:** `core/config.py` (load_config)
- **문제:** DatabaseConfig에 max_overflow 필드 있지만 load_config()에서 로드 안 함
- **난이도:** ★☆☆☆☆ (10분)
