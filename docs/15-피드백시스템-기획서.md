# NGSAT 대시보드 사용자 피드백 시스템 기획서

## 현재 문제점
- 서버 재시작 버튼: 누르면 아무 반응 없음 (API는 정상 응답)
- 매매 시작/중단: 버튼만 바뀌고 사용자 알림 없음
- 오류 발생: 콘솔에만 로깅, 사용자 눈에는 안 보임
- 거래 체결: Telegram 알림은 가지만 대시보드에는 즉시 피드백 없음

## 제안: Toast Notification 시스템

### 1. 토스트 메시지 (즉시 구현 가능)

우측 하단에 3초간 나타났다 사라지는 알림:
- ✅ 성공: 초록 배경 — "매매 시작됨", "서버 재시작 완료"
- ℹ️ 정보: 파랑 배경 — "데이터 갱신 중..."
- ⚠️ 경고: 노랑 배경 — "KIS API 연결 불안정"
- ❌ 오류: 빨강 배경 — "주문 실패: 사유"

### 2. 트리거 포인트

| 액션 | 메시지 | 타입 | 우선순위 |
|------|--------|------|---------|
| 매매 시작 | "매매가 시작되었습니다" | ✅ success | 상 |
| 매매 중단 | "매매가 일시정지되었습니다" | ℹ️ info | 상 |
| 서버 재시작 | "서버가 재시작되었습니다" | ✅ success | 상 |
| 강제 매도 | "매도 주문이 전송되었습니다" | ℹ️ info | 중 |
| 거래 체결 | "한국전력 5주 매수 체결" | ✅ success | 중 |
| API 오류 | "KIS API 연결 오류" | ❌ error | 상 |
| 리스크 중단 | "일일 손실 한도 도달 - 매매 중단" | ❌ error | 상 |
| 모드 전환 | "스윙→단타 모드 전환" | ℹ️ info | 하 |

### 3. 상태 표시기 강화

사이드바 하단에 현재 상태를 한눈에:
- 🟢 초록: 정상 운영 중
- 🟡 노랑: 일시중지됨
- 🔴 빨강: 오류/리스크 중단
- ⚪ 회색: 연결 안 됨

### 4. WebSocket 연동 실시간 알림

이미 WebSocket(`/ws/realtime`)이 구현되어 있으므로, 서버에서 토스트 이벤트를 푸시:
```json
{ "type": "toast", "message": "...", "level": "success" }
```
