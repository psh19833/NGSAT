## C-1. KIS WebSocket 실시간 시세 — 사전 분석

### 현재 상황
- `RealDataProvider.refresh_prices()`가 10초마다 40개 종목 × REST API 호출
- Phase A에서 30사이클(5분)마다로 조정했으나, 여전히 REST API Rate Limit 위험存在
- 장중 실시간 가격 반영이 5분 단위로 지연됨

### KIS WebSocket API 구조 (공식 문서 기준)

#### WebSocket 엔드포인트
```
실전투자: wss://openapi.koreainvestment.com:21000
모의투자: wss://openapi.koreainvestment.com:21001
```

#### 인증 방식
1. REST API로 접속토큰 발급 (`/oauth2/Approval`)
2. 발급받은 `approval_key`를 WebSocket 연결 시 헤더로 전송
3. WebSocket 연결 유지 시간: 약 6시간 (토큰 만료 전 재발급 필요)

#### 지원 실시간 데이터 (국내주식)

| 서비스명 | TR_ID | 데이터 내용 | 활용도 |
|---------|-------|-----------|-------|
| 주식호가 | **H0UNCN0** | 실시간 호가 10단계 | ★★★ — 진입/청산 가격 판단 |
| 주식체결가 | **H0UCNT0** | 실시간 체결가·거래량 | ★★★★★ — 현재가 갱신 |
| 주식체결통보 | H0UCCN0 | 내 주문 체결 통보 | ★★★ — 주문 상태 추적 |
| 실시간체결통보 | HTS ID 기반 | 사용자 주문 체결 알림 | ★★ — 장애복구 |

#### 구현 전략

```python
# data/adapters/kis/websocket.py (신규)
import asyncio, json, logging
import websockets

class KisWebSocketClient:
    """KIS 실시간 시세 WebSocket 클라이언트."""
    
    WS_URL_REAL = "wss://openapi.koreainvestment.com:21000"
    WS_URL_DEMO = "wss://openapi.koreainvestment.com:21001"
    
    def __init__(self, base_url, app_key, app_secret):
        self._ws_url = self.WS_URL_DEMO if "demo" in base_url else self.WS_URL_REAL
        self._app_key = app_key
        self._app_secret = app_secret
        self._ws = None
        self._approval_key = None
        self._subscribed_codes: set[str] = set()
        self._on_price: callable = None  # async callback
    
    async def connect(self):
        """1. REST로 approval_key 발급 → 2. WebSocket 연결"""
        self._approval_key = await self._get_approval_key()
        self._ws = await websockets.connect(self._ws_url)
        # 재구독: 기존 구독 복원
        for code in self._subscribed_codes:
            await self._subscribe(code)
    
    async def subscribe(self, code: str):
        """종목코드 실시간 체결가 구독"""
        self._subscribed_codes.add(code)
        if self._ws:
            await self._subscribe(code)
    
    async def _subscribe(self, code: str):
        msg = {
            "header": {
                "approval_key": self._approval_key,
                "custtype": "P",
                "tr_type": "1",
                "content-type": "utf-8",
            },
            "body": {
                "input": {
                    "tr_id": "H0UCNT0",
                    "tr_key": code,
                }
            }
        }
        await self._ws.send(json.dumps(msg))
```

| 요소 | 분석 |

### 설계안

```python
# core/config_service.py (신규)
"""
ConfigService — DB 기반 설정 저장소.

데이터 흐름:
  기동 시: .env → StrategyConfig (현재 유지)
  대시보드 변경 시: ConfigService.set(key, value) → DB 저장 + 인메모리 캐시 갱신
  재시작 시: ConfigService.load() → DB → StrategyConfig (env보다 우선)
  
DB 테이블:
  system_config (
    key VARCHAR(100) PRIMARY KEY,
    value TEXT,
    updated_at TIMESTAMP
  )
  
.env와의 관계:
  - .env가 우선순위 1순위 (초기값)
  - DB가 우선순위 2순위 (런타임 변경값)
  - config.get(key) = db_value or env_value or default
"""
```
